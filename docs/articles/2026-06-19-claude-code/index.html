Claude Code v2.1.181 と課金変更の撤回 — /config key=value で運用を即チューニング

2026-06-17 公開の v2.1.181 は /config key=value 構文で対話・-p・リモートのどこからでも設定を即時変更できるようにし、sandbox.allowAppleEvents や Bun 1.4 同梱、行単位ストリーミングを追加。あわせて 6/15 に予定されていた Agent SDK 課金分離の撤回・凍結も整理し、自動化を従来どおりサブスク枠で回すための運用ポイントをまとめる。

📅 2026年6月19日 ⏱ 約3分 🔗 出典 3 件

なぜ今これが話題か

毎朝のスケジュールタスクや CI で Claude Code を回していると、効いてくるのは派手な新ウィジェットより運用の地ならしだ。2026-06-17 公開の v2.1.181 は操作性と基盤強化が中心で、設定変更の摩擦を一段下げる。あわせて 6/15 に予定されていた Agent SDK の課金分離が撤回・凍結された件も、自動化ユーザーには見逃せない。

何が新しいか

目玉は /config key=value 構文。対話・-p(ヘッドレス)・リモート操作のどこからでも任意の設定をその場で変更できる（例 /config thinking=false）。さらに macOS では sandbox 内コマンドが Apple Events を送れる sandbox.allowAppleEvents をオプトインで追加、同梱の Bun ランタイムは 1.4 に更新。長い段落のストリーミングが行単位で逐次表示され、思考中の API 切断時には自動リトライするようになった。

仕組みのポイント

/config key=value の価値は「スコープを問わない」点にある。これまで設定変更は対話セッションや設定ファイル編集に縛られがちだったが、ヘッドレス実行やリモート経由でも1行で即時に切り替えられる。スケジュールタスク内に /config を差し込めば、実行ごとに thinking のオン/オフやモデル設定を動的に振り分けられる。

課金変更の撤回

当初 Anthropic は Agent SDK・claude -p・サードパーティアプリの利用を Pro/Max/Team/Enterprise のサブスク枠から切り離し、別建ての月額ドルクレジット(繰り越しなし)へ移す計画だった。だが 6/15 にヘルプセンターで撤回を確認。当面これらは従来どおりサブスク上限から消費される。同社は「開発実態により合う形に作り直す」とし、将来の変更時は事前告知するとしている。

自分が試すなら

  • スケジュールタスクの先頭で /config を使い、重い処理だけ thinking を切ってコスト削減。
  • Apple Events 連携が要る macOS 自動化で sandbox.allowAppleEvents を限定的に有効化。
  • Bun 1.4 同梱で外部ランタイム依存が減るので、CI イメージを軽くできるか検証。

注意点

sandbox.allowAppleEvents はオプトイン＝既定で無効。アプリ間自動化はセキュリティ境界を緩めるので、対象アプリを絞って付与する。課金は「凍結」であって「無料化」ではない点も注意。サブスク上限は据え置きだが、Dynamic Workflows のような大規模並列はトークンを大きく消費するため /usage での監視は引き続き必須。

ひとことまとめ

v2.1.181 は派手さこそないが、/config key=value で運用の即応性が上がる実用アップデート。課金分離は当面凍結なので、自動化は従来どおりサブスク枠で回せる。まずは /config thinking=false をスケジュールタスクに仕込んで体感するのが手軽だ。
