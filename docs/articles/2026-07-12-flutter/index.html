Flutter 2026折り返しレビュー — 3.44とロードマップ『Finishing the Job』を読む

上半期は3.41→3.44を予定通り出荷、次の安定版は8月。material_uiのSDK分離、Impellerの120fps路線、Riverpod 3.0既定という下半期の軸を整理する。

📅 2026年7月12日 ⏱ 約3分 🔗 出典 4 件

2026年も折り返し。公式ロードマップ『Finishing the Job』は年4回以上の安定リリースをコミットし、上半期は3.41（2月）→3.44（5月・Google I/O 2026）を予定通り出荷、次の安定版は8月に控える。現行はFlutter 3.44.0 / Dart 3.12.2。名前の通り「やりかけを終わらせる」年で、柱はAndroidのImpeller移行完遂・WebのWasm既定化・GenUIとエージェント型アプリ・Full-Stack Dart・AI前提の開発体験の5本だ。

構造的に大きいのはmaterial_uiのpub.dev実体化だ。package:flutter/material.dartのライブラリを収める公式スタンドアロンパッケージで、分離後はSDKの3ヶ月サイクルに縛られず独自ペースで更新でき、アプリサイズも縮小する。M3の新機能がSDKリリースを待たず届く体制になる。Dart 3.12はprivate named parametersが正式化し、primary constructorsが実験的に入った。3.44移行ではSwift Package ManagerがiOS/macOS既定（CocoaPods置換）、UIScene lifecycle必須化対応が要チェック。採用事例としては2026年型Toyota RAV4が挙がる。

描画面の焦点は「60fps達成」から「120fps安定維持」へ移った。ImpellerはAOTシェーダーコンパイルでコンパイル起因のジャンクを排除し、複雑シーンの平均ラスタライズを約50%短縮。カスタムシェーダーは事前コンパイルと好相性だが、fragment shaderの重い分岐は避けmix/step/clampの数学演算に置き換え、実行4〜5ms以内が120fps維持の目安になる。フレーム予算はラスタライズ4ms未満で60fps、8.3ms未満で120fps。DevToolsのImpeller専用ビューがdraw callのバッチ処理を可視化し、テクスチャのメモリ過多を即ハイライトしてくれるので、当て推量の最適化から卒業できる。計測はprofileモード必須、Timeline Viewでrasterスレッドのスパイクを追う。

状態管理の基本線は変わらず、新規はRiverpod 3.0既定（unified Ref・指数バックオフの自動リトライ・実験的オフライン永続化）、大規模チームやfintech/healthcareの監査要件はBloc 9.0。実践則は「最小の状態スライスを最下層でwatch」「リストアイテムは分離ウィジェット化して必要最小限だけ渡す」。

ひとことまとめ: 下半期はWasm既定化とImpeller完遂が本丸。手元では8月安定版に備えてSPM移行とprofileモード計測の習慣化を進めておきたい。
