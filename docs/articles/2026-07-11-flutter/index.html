Flutter 3.44系の地雷ウォッチ — VS 2026非対応・iOS 26デバッグハング・APNGオーバーフロー

アップグレード前に踏むべき既知問題が3つ明確になった。アニメーションの指標は「60fps達成」から「120fps安定維持」へ、状態管理の教訓は「1つ選んで一貫し、テストを書く」。

📅 2026年7月11日 ⏱ 約3分 🔗 出典 4 件

今日の主役はFlutter 3.44系の既知問題ウォッチだ。アップグレード前に確認すべき地雷が3つはっきりした。(1) Visual Studio 2026はWindowsデスクトップアプリのコンパイル非対応で、ビルド環境はVS 2022を維持する。(2) iOS 26実機をIDEからデバッグすると、アプリはインストールされるが起動時に白画面のままハングする。(3) animated PNG（APNG）処理に整数オーバーフローの可能性が報告され、修正が進行中。安定運用の基準はFlutter 3.44.0 / Dart 3.12.2（次の安定版は2026年8月予定）。

3.44の柱は「framework packagesのSDK分離・Appleツーリング近代化・AI支援開発への準備・デスクトップ再編」。Material/Cupertinoは独立パッケージmaterial_ui/cupertino_uiへ移行中で、SDKの3ヶ月サイクルに縛られずpub.devで随時更新される。MCP serverとコーディングエージェントが起動中のアプリを自動検出・接続し、AIのUI編集で自動hot reloadするAgentic Hot Reloadも要注目だ。Dart 3.10のdot shorthands（型が明らかならprefix省略）は成熟期に入り、公式サンプルも移行中。

アニメーションの焦点は「60fpsを満たす」から「高リフレッシュ端末で120fpsを安定維持」へ移った。ImpellerはAOT（ビルド時）シェーダーコンパイルで重処理を起動前に済ませ、アニメ発火時には命令がGPU待機済み — シェーダーコンパイルジャンクを排除し、複雑シーンの平均ラスタライズ時間を約50%短縮した。フレーム予算は60fps=16ms、90〜120Hzでは8〜11ms。使い分けはまずimplicit（AnimatedContainer等）、精密なchoreographyはexplicit（AnimationController、dispose必須）、重い演出はRive（0.12+のeventsでハプティクスやナビゲーションを正確にトリガー）へオフロード。

状態管理の今日の教訓は「1つ選んで一貫し、テストを書く」。よくテストされたBlocアプリは、未テストのRiverpodアプリより保守しやすい — ライブラリはインフラで、テストが安全網だ。基本線は変わらず、新規はRiverpod 3.0（Mutations・200msから倍々で最大6.4sの指数バックオフ自動リトライ・画面外listenerの自動pause）、大規模・監査要件はBloc 9.0。パフォーマンスはAutomaticKeepAliveClientMixinでスクロールリストのstateを保持し、KeyedSubtree/ValueKeyでdiffを効率化、計測は必ずprofileモードで行う。

ひとことまとめ: 3.44へ上げる前にVS 2026・iOS 26・APNGの3点を確認。走り出したら次の指標は120fpsの安定維持。
