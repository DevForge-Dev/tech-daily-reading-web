Claude Code

既定は「Manual」へ — v2.1.200の権限転換とArtifacts、拡張4層の地図

7/7のClaude Codeは“主導権をユーザーへ戻す”アップデート。v2.1.200で既定の権限モードがManualになり、AskUserQuestionは勝手に進まなくなった。あわせてセッション成果をライブWebページ化するArtifactsが登場し、拡張はhooks/subagents/skills/slashの4層で整理が定着。採用面では日次インストール29M・公開コミットの約4%と“ChatGPTの瞬間”が続く。

📅 2026年7月7日 ⏱ 約4分 🔗 出典 5 件

なぜ今これが話題か

ここ数日のClaude Codeは自律実行の速さを競ってきたが、7/7の焦点はむしろ逆で、「重要な判断は人に戻す」方向への調整だ。エージェントが長時間・並列で動くほど、意図しない実行や無言の続行が事故につながる。v2.1.200はその安全側の既定変更をまとめて入れてきた。

何が新しいか

Releasebotによると、v2.1.200では CLI・--help・VS Code・JetBrains を通じた既定の権限モードが「Manual」になった。--permission-mode manual と "defaultMode": "manual" はいずれも従来の "default" のエイリアスとして受理される（Manualラベルの表示はv2.1.200以降が必要）。さらにAskUserQuestionのダイアログが既定で自動継続しなくなり（askUserQuestionTimeoutが"never"）、放置しても勝手に選択が進まない。アイドルで進めたい人は /config からタイムアウトへオプトインする。加えてバックグラウンドセッションの安定化、MCPサーバー信頼性、サンドボックスのネットワーク許可ダイアログ改善も含む。

もう一つの目玉：Artifacts

公式changelogによると、Claude Codeにセッションの作業内容をライブWebページにするArtifactsが加わった。PRのウォークスルー、インシデント対応ページ、ダッシュボード、チェックリストなどを、使い捨てのログではなく「開くたびに最新化される資産」としてチームに共有できる。Team/Enterpriseでは組織内限定の非公開共有とバージョン履歴がベータで提供される。

拡張の地図（4層）

技術記事側は拡張機構の整理で一致している。slashコマンド＝プロンプトの雛形、skill＝補助ファイルを伴う知識/プレイブック（呼ばれるまで本文がロードされずコスト小）、subagent＝隔離した並列ワーカー、hook＝コードでルールを強制。Hooksは2026年に全30種イベント・5種ハンドラー（command/prompt/agent/http等）へ拡大し、設定は管理者ポリシー>プロジェクト>ユーザー>プラグイン/スキルの優先順で評価される。rm -rf防御や.env保護は .claude/settings.json にコミット、個人実験は settings.local.json が定石だ。

自分が試すなら

まず既定のManualで数タスク回し、確認が増えて煩わしい定型だけを許可ルールで緩める運用に切り替える。放置バッチには /config でアイドル継続をオプトイン。トラブル時は前バージョンで入った --safe-mode で全カスタマイズを無効化し、/cd でキャッシュを壊さず作業ディレクトリを移して切り分ける。

注意点

Manual既定は安全側だが確認回数が増える。CIや無人実行では明示的に権限モードとタイムアウトを設定しないと止まりっぱなしになり得る。採用の勢いは続き、日次インストールは17.7M→29M、2026年2月時点で公開GitHubコミットの約4%（SemiAnalysisは年末20%超と予測）、Claude Code単体でラン・レート25億ドル超。国内でも楽天がリリースを24→5日、マネーフォワードはエンジニア約80%が利用と定着が進む。

ひとことまとめ

既定はManualへ、問いは勝手に進まない。成果はArtifactsで“生きたページ”に、拡張はslash/skill/subagent/hookの4層で組む——それが7/7時点の定石だ。
