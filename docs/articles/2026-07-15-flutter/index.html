Flutterの『AIで作る』が手順化 — MCP Server×Agentic Hot Reload、土台の3.44も描画改善

Gemini Code Assist・Gemini CLI・Dart＆Flutter MCP ServerでアプリにAI機能を組み込む手順が公式ドキュメント化。エージェントが起動中アプリを検出してUI編集をhot reloadするAgentic Hot Reloadと噛み合う。安定版はFlutter 3.44.0/Dart 3.12.2、Impellerの描画も改善。

📅 2026年7月15日 ⏱ 約3分 🔗 出典 3 件

Flutterの「AIで作る」が、ふわっとした期待から公式の手順書の段階に入った。目玉はCreate with AIの公式ガイドで、あわせて安定版3.44の描画改善や状態管理の実務知見も更新されている。

何が新しいか

公式ドキュメントが、Gemini Code Assist・Gemini CLI・Dart＆Flutter MCP Serverを使ってアプリにAI機能を組み込む手順を整理した。肝はAgentic Hot Reloadとの噛み合わせで、MCP Server経由でコーディングエージェントが起動中のアプリを自動検出・接続し、UI編集を人手なしでhot reloadできる。AIロジックはGenkit Dart（Google/Anthropic/OpenAI系を統一APIで扱う）でサーバ側Dartに退避でき、APIキーをクライアントから外せるのが実務的に重要だ。安定版はFlutter 3.44.0 / Dart 3.12.2（次の機能リリースは2026年8月予定）。

土台の改善

描画エンジンImpellerはVulkan周りが進み、キャッシュのメモリ管理改善とフレームドロップ時のGPU/CPU同期の効率化でスタッターを削減。円のレンダリングがsigned-distance function（SDF）ベースになりエイリアスが消え、perspective matrixの補正で影や3D風Transformが正確になった。フレーム予算は60fps=16ms / 120fps=8.3msで、constの徹底とRepaintBoundaryの戦略配置が基本線のまま効く。

使いどころ・状態管理の注記

自分が試すなら、まずDart＆Flutter MCP Serverを繋いでAgentic Hot Reloadで小さなUI調整を丸ごとエージェントに任せ、AI呼び出しはGenkit Dartでサーバ側に寄せてキーを守る。状態管理は据え置きで、新規はRiverpod 3.x（async-first）、大規模／監査はBloc 9.x。Bloc 9はsearch-as-you-typeや連打をbloc_concurrencyのrestartable/droppable/throttleで宣言的にさばける点が今も強い。注意点は、AI前提の手順はまだ発展途上でSDK/APIの更新が速いこと。ナレッジの日付を都度確認したい。

ひとことまとめ: 「MCPで起動中アプリに繋ぎ、Agentic Hot Reloadで編集、キーはGenkit Dartでサーバへ」。AI開発ループが手順として固まってきた。
