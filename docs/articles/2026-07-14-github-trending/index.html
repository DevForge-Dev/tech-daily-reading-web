今日のトレンド上位は、AIエージェント系が席巻する中で「クラウドに預けず手元で完結させたい」という揺り戻しが共通して見える3本。純粋なクリエイター向けアプリ、研究室発の投資エージェント、そしてエージェント専用のOfficeスイートという顔ぶれだ。それぞれ「解決したい課題」と「提供する機能」を整理する。

OpenCut-app/OpenCut（TypeScript・⭐66k）

解決したい課題：CapCutの規約変更・有料化への反発を背景に、「手元で完結し、動画をクラウドに預けず、透かしも入らない」動画編集への需要が再燃している。既存の無料ツールは透かしやアップロード必須が多く、プライバシーと自由度の両立が難しかった。

提供する機能：ブラウザ／デスクトップで動くオープンソースの動画編集ツール。タイムライン編集・トリミングなど基本機能をブラウザ完結で提供し、動画をクラウドにアップロードしない。透かしなし・完全オープンソースで自己ホストも可能。本日Trendshift日次2位に浮上し、AIエージェント系が上位を占める中で純粋なクリエイター向けアプリが急上昇した点が注目された。

HKUDS/Vibe-Trading（Python・⭐22k）

解決したい課題：個人の投資リサーチは、市場データの収集・分析・戦略立案・検証と工程が多く専門的で、手作業のコストが高い。LLMを金融ワークフローに落とし込む実用的な枠組みが求められていた。

提供する機能：自然言語で指示すると相場分析から戦略立案・シミュレーションまで行う個人向けAIトレーディングエージェント。マルチエージェント／ワークフロー自動化で市場データ収集・分析・戦略検討を補助する。香港大学のデータ系研究室(HKUDS)発で手法が公開されカスタマイズしやすい。直近7日で約+3.5k★と急伸し、「Vibe Coding」ならぬ「Vibe Trading」のネーミングも拡散を後押しした。※投資判断は自己責任で。自動化はあくまでリサーチ補助と捉えるのが安全。

iOfficeAI/OfficeCLI（C#・⭐16k）

解決したい課題：AIエージェントにWord/Excel/PowerPointを操作させたいが、Officeのインストールが前提だと環境構築が重く、CIやサーバー上での自動化がしにくい。

提供する機能：AIエージェントがOffice文書を読み書き・自動化するための単一バイナリのOfficeスイート。Officeインストール不要でWord/Excel/PowerPointの読み取り・編集・生成を実行でき、無料・オープンソースでCIへの組み込みもしやすい。直近7日で約+7.6k★と非常に高い増加率で、「AIエージェント専用のOfficeスイート」という切り口の新しさが受けている。

自分が試すなら：DevForge文脈ではまずOfficeCLIを、帳票・ドキュメント生成の自動化に組み込みたい。文書処理をエージェントから完結できれば、Office依存の重い環境構築を外せる。
