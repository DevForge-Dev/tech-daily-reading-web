Claude Code

Claude Code v2.1.187 — sandbox.credentials で認証情報を遮断、記憶を整える「Dreaming」、レート上限2倍

サンドボックス実行に外部認証を任せるほど、信用できないコマンドが ~/.aws/credentials や ~/.ssh を読めてしまう危険が増す。6月23日の v2.1.187 は sandbox.credentials でこの読み取りを既定で遮断できるようにした。あわせて過去セッションを振り返ってメモリを再編成するリサーチプレビュー「Dreaming」、Claude Code のレート上限2倍と並列サブエージェント委譲の強化も到来している。

📅 2026年6月25日 ⏱ 約3分 🔗 出典 3 件

なぜ今これが話題か

エージェントに自動実行を任せるほど、サンドボックス内のコマンドが何を読めるかが安全性の核心になる。既定では ~/.aws/credentials や ~/.ssh/、秘密の環境変数までプロセスから見えてしまい、信頼できないツールやスクリプトを動かす場面では情報漏れの経路になりうる。6月23日リリースの Claude Code v2.1.187 は、ここを設定でふさげるようにした。

何が新しいか

目玉は sandbox.credentials 設定。サンドボックス実行されるコマンドが認証ファイルや秘密の環境変数を読み取れないように遮断できる。あわせて、モデルピッカーに組織が構成したモデル制限が反映され、全画面モードのセレクトメニューでマウスクリック選択に対応。--resume が「No conversation found」で失敗する不具合の修正、--json-schema とワークフローエージェントの構造化出力の改善も入った。別系統では Anthropic が Dreaming（Dreams） をリサーチプレビューで提供開始。過去セッションを定期的に振り返り、繰り返す失敗・共有ワークフロー・チームの嗜好を抽出してメモリを自動でキュレーションする。さらに Claude Code のレート上限が2倍に引き上げられ（反映済み）、リードエージェントが並列サブエージェントへ委譲（各々が独自のモデル・プロンプト・ツール）する構成も強化された。

Dreaming はどう効くか

Dream は既存のメモリストアと過去セッションのトランスクリプトを読み込み、重複を統合・陳腐化エントリを置換・新知見を付与した再編成済みメモリを出力する。実行間でエージェントが賢くなることを狙うスケジュール処理だ。

自分が試すなら

  • settings.json に sandbox.credentials を設定し、サンドボックス内から認証ファイルが読めないことを確認する。
  • 組織のモデル制限を入れて、ピッカーに反映されるか・意図しないモデルが選べないかを検証する。
  • レート上限2倍を前提に、重い調査を並列サブエージェントへ委譲する構成を試す。

注意点

Dreaming は Claude Managed Agents 向けのリサーチプレビューで、挙動や提供範囲は変わりうる。sandbox.credentials は「サンドボックス実行されるコマンド」に効く設定なので、サンドボックス外で動く処理の権限管理は別途必要。レート上限2倍でも、自動化の暴走によるコスト増は別問題として上限・予算アラートで管理したい。

ひとことまとめ

v2.1.187 は「安全な自動実行」を一段進めた更新。まず sandbox.credentials で足元の漏えい経路を封じ、Dreaming とレート上限2倍で長時間タスクの質と量を伸ばすのが筋が良い。
