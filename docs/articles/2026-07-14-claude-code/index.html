Claude Code に Artifacts 正式導入 — セッションの成果を「更新され続けるライブWebページ」に

セッション作業をライブ共有Webページ(Artifact)として発行し、同一URLで自動更新できる機能が正式導入。同週のv2.1.202〜207ではautoモードがBedrock/Vertex/Foundryで既定に、Opus 4.8が各クラウド基盤の既定モデルへ。/configには動的ワークフロー規模の設定も加わった。

📅 2026年7月14日 ⏱ 約3分 🔗 出典 3 件

Claude Codeの設定・運用まわりが一段まとまった週だった。目玉はArtifactsの正式導入で、あわせて週次アップデート（v2.1.202〜v2.1.207）が基盤モデルと既定モードを大きく動かし、/configには新しい制御項目が加わった。単発の便利機能というより、「エージェントの成果をどう共有し、どう操縦するか」という運用面の底上げがまとめて来た格好だ。

何が新しいか

Artifactsは、セッションでの作業内容をライブなWebページに変換して共有する仕組み。PRウォークスルー、インシデントのタイムライン、各種ダッシュボード、リリースチェックリストなどを、作業の進行に合わせて同一URL上で自動更新できる。公開はできず、認証済みの組織メンバーのみ閲覧可能。バージョン履歴からいつでも過去の状態に復元できるのも実務向きだ。提供はTeam/Enterprise向けのベータで、Pro/Maxにも解放が進んでいる。

同週の週次アップデートでは、autoモードがBedrock・Vertex AI・Foundryで既定利用可能になり、Opus 4.8がこれら各クラウド基盤上のClaude Platformの既定モデルに昇格した。/reviewコマンド体系の刷新、/doctorによるCLAUDE.md簡素化提案も入っている。

/config と細かな改善

/configには「Dynamic workflow size」設定が加わり、動的ワークフローで動くエージェント数（small/medium/large）を制御できるようになった。/cdには/add-dir同様のディレクトリパス補完が付き、/commit-push-prはリポジトリ設定済みのpushリモートへのgit pushを自動許可する。長いリスト・表・コードブロックのストリーミング中に端末がフリーズ／入力が遅延する問題も修正された。

実務での使いどころと注意点

自分が試すなら、まずインシデント対応や長丁場の調査セッションをArtifact化して「同じリンクを見れば常に最新」という状態を作りたい。手元でも、まさにこのデイリー読み物のように、日次で追記されるライブページとして機能する。注意点は、Artifactsが組織内限定＝公開用途には使えないこと、そしてautoモードとOpus 4.8既定化でコスト特性が変わるため、重い定期タスクを回している場合は消費ペースを見直しておくとよい。

ひとことまとめ: 「成果をライブページで共有(Artifacts)」「基盤はauto＋Opus 4.8が既定」「規模と権限は/configで調整」——共有・モデル・制御が同時に底上げされた週。
