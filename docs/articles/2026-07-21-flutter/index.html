Flutter 3.44 深掘り——Web hot reload 既定化、Widget Previews、Riverpod 3.0 既定、Impeller 2.0

安定版は Flutter 3.44.0 / Dart 3.12（次の機能リリースは 8月予定）。Web hot reload が stable・既定化し状態を保ったまま反復可能に。@Preview の Widget Previews で単一ウィジェットを分離プレビュー。状態管理は新規=Riverpod 3.0 既定、監査/大規模=Bloc 9.0。Impeller 2.0 はシェーダーを AOT 化しジャンクを排除、Skia 比で約17%低い電力ドレイン。

この日のFlutterは開発ループとレンダリングの底上げが主役だ。安定版はFlutter 3.44.0 / Dart 3.12（Google I/O 2026・次の機能リリースは8月予定）。「静かだが深い」972コミット/178コントリビュータのリリースで、日々の反復とパフォーマンスの土台が更新された。

何が新しいか

目玉はWeb hot reloadのstable・既定化。ウィジェットツリーがリロードを跨いで保持され、StatefulWidgetの状態を維持したまま反復できる——Webの開発ループがモバイル並みになった。加えて@Preview注釈のWidget Previewsでアプリ全体を起動せず単一ウィジェットを分離プレビューでき、VS Code/IntelliJ統合でUI反復が速い。Agentic Hot ReloadはDart & Flutter MCP Server経由でエージェントが自動リロードする。Material/Cupertinoはコア凍結→将来material_ui/cupertino_uiへ独立バージョニングし、SDKを待たずM3新機能が届く。

状態管理とパフォーマンス

状態管理は新規プロジェクト＝Riverpod 3.0を既定（compile-time safety・unified Ref・Mutations・自動リトライ、現行3.3.x）、監査/大規模＝Bloc 9.0（event tracing・監査証跡、現行9.2.x）が基本線。2026ベンチのwidget rebuildはBloc 2.3ms / Riverpod 2.0msでほぼ互角、テストはRiverpodが30〜40%少ない行数。「1つ選んで一貫し、テストを書く」が選定より重要だ。描画側はImpeller 2.0が全シェーダーをAOTコンパイルして初回ジャンクを排除し、アニメ多用ワークロードでSkia比 約17%低い電力ドレイン。iOS＋モダンAndroid（API29+）で既定、Desktopへ拡大中だ。

自分が試すなら

まずWeb案件でhot reloadの既定化を体感し、共通ウィジェットは@Previewで分離プレビューして反復を速める。新規は迷わずRiverpod 3.0で、feature単位のClean Architecture＋go_routerを乗せる。既存Providerからはscreen-by-screenで移行できる（共存可）。パフォーマンスはDevToolsのPerformanceタブで>16msの長フレームを特定し、const徹底・build分割・RepaintBoundaryで再ビルドを削る。

注意点

Impellerで差が出たら--no-enable-impellerで切り分ける。アニメは「まずimplicit」を試し、explicit（AnimationController）は精密choreography・ループ・ジェスチャに限定してdispose必須。計測はprofileモードでないと数値が当てにならない。

ひとことまとめ

「反復はhot reload＋Previewで速く、描画はImpeller 2.0で滑らかに、状態はRiverpod 3.0を既定に」。3.44は派手さより日常の生産性を底上げする回だ。
