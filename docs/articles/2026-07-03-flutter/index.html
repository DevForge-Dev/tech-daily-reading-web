2026ロードマップ「Finishing the Job」— Android Impeller完了・Wasm-safe interop・Web WASM既定

今日の主役は公開された Flutter/Dart 2026ロードマップ「Finishing the Job」。新機能の追加より「やり残しの完遂」が主軸で、AndroidのImpeller移行完了(Android10+でレガシーSkia撤去しVulkan一本化)が中核。レガシーJS interopを廃止し Wasm-safe interop(dart:js_interop)へ、WebはWebAssemblyが既定化(ロード時間40%短縮・メモリ30%削減)。安定版は Flutter 3.44.0 / Dart 3.12.2。

📅 2026年7月3日 ⏱ 約3分 🔗 出典 3 件

なぜ今これが話題か

Flutter の2026年は「派手な新機能」ではなくやり残しの完遂がテーマだ。公開されたロードマップ「Finishing the Job」は、長年並行運用してきたレンダリングやWeb基盤の移行を"終わらせる"方向を鮮明にした。安定版は Flutter 3.44.0 / Dart 3.12.2、次の安定版は2026年8月予定(年4回リリース)。基盤が固まる局面なので、既存プロジェクトの棚卸しに直結する話が多い。

何が新しいか

中核はAndroidのImpeller移行完了。Android 10以上ではレガシーSkiaを撤去し、Vulkan一本化(Pure Impeller)になる。シェーダーをビルド時(AOT)にコンパイルするため、実行時のシェーダーコンパイル由来のジャンクが消え、決定論的フレームで 60/90/120fps が安定する。Web側は二段構え。レガシーJS interop(dart:html/package:js)を廃止し、Wasm-safe interop(dart:js_interop)へ移行、そしてWebはWebAssembly(Wasm)が既定になる。テストではロード時間40%短縮・実行時メモリ30%削減、DevToolsもWASM既定でTimelineが高頻度フレームをジャンクなく扱う。さらに Android 17 や新iOSへの day-zero 対応、言語面では Primary Constructors と Augmentations を2026年内に出荷予定。

実務での使いどころ

効くのはWebプロジェクトの棚卸しだ。dart:html に直接依存するプラグインは Wasm で動かないため、早めの洗い出しが要る。まず flutter build web --wasm の dry-run を走らせ、非対応依存を特定するところから始めるのが定石。Android側は Impeller 前提でシェーダー由来のジャンクが減るので、以前コンパイルジャンクを疑っていた箇所を再計測する価値がある。

注意点

「完遂」ロードマップは撤去を伴うのが肝。Android10+ のレガシーSkia撤去やレガシーJS interop廃止は、古い依存を抱えたままだと更新時に壊れうる。移行は依存の棚卸し→dry-run→段階置換の順で、一気に上げない。数値(-40%/-30%)は公式テスト値で、実アプリでは構成次第なので profile モードで自分の値を測ること。新言語機能(Primary Constructors 等)は2026年内予定で、今すぐ本番前提にはしない。

ひとことまとめ

2026のFlutterは「増やす」より「終わらせる」年。Impeller一本化とWeb WASM既定は性能に効くが、撤去を伴うぶん依存の棚卸しが先。まず --wasm の dry-run で自分のプロジェクトの現在地を確認しよう。
