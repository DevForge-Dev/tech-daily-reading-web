Material/Cupertino「凍結」の号砲 — 3.44とImpellerのSDF円描画

安定版はFlutter 3.44 / Dart 3.12.2。今日の主役は3.44でのMaterial/Cupertinoライブラリの“コード凍結”だ。これがコア内での最後の更新で、今後はstandaloneの material_ui / cupertino_ui へ。描画ではImpellerが円をSDFベースにし、ギザつきのないクリーンな円を描けるようになった。

なぜ今これが話題か

安定版はFlutter 3.44 / Dart 3.12.2（次期安定は2026年8月予定）。今日の主役は、3.44でのMaterial／Cupertinoライブラリの「コード凍結（frozen）」だ。これがコアフレームワーク内での最後の更新セットで、UIの提供モデルが変わる号砲になる。

何が新しい/重要か

凍結時点でMaterial 3コンポーネントは完全カバー済み。デザインシステム準拠のためのband-aid回避策や自作ウィジェットはもう不要だ。今後はstandalone packagesのmaterial_ui／cupertino_uiへ移行し、SDKリリースを待たずデザインシステムを独自cadenceで更新でき、アプリサイズも縮小する。描画ではImpellerが円のレンダリングをsigned-distance functions（SDF）ベースに更新し、以前エイリアス（ギザつき）が出ていたケースでもクリーンな円を描ける。角丸・ボーダー・アイコン等の円要素が多いUIで効く。perspective matrixの扱いも改善し、シャドウと3D風transformの見た目が正確になった。

描画基盤とアーキテクチャ

パフォーマンス面ではPure Impeller VulkanがモダンAndroidで既定かつ唯一の選択肢となり、全シェーダーとreflectionをビルド時にオフラインコンパイルして実行時のシェーダーコンパイルジャンクを排除する。DevToolsはWasmGCへ既定コンパイルされ軽快になり、Impeller専用ビューでdraw callのバッチ化を可視化できる。状態管理は新規＝Riverpod 3.3.1＋コード生成、規制/監査/複雑並行＝Bloc 9.2.xが基本線だ。

自分が試すなら

本番の凍結面は3.44 / 3.12.2に据えるのが安全。円やドロップシャドウの多い画面をprofileモードで再計測し、SDF化とperspective補正の効果を確認する。将来のパッケージ分離に備え、依存とimport経路の変更を見越してバージョン固定と更新計画を持っておく。

注意点

凍結後はコアからMaterialへのAPI追加が止まるため、新機能は将来のmaterial_ui側で追う前提になる。Rive等はImpellerで描画差が出る場合があり、--no-enable-impellerでSkiaにフォールバックして切り分ける。experimentalなPrimary Constructors/Augmentationsを本番の土台にしない。

ひとことまとめ

Material/Cupertinoは3.44で凍結し、UIはパッケージ独立更新の時代へ。描画はSDF円＋Vulkan既定で品質と決定論が上がる。今は3.44で堅く作りつつ分離の波に備えたい。
