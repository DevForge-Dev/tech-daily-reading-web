Week28で内蔵ブラウザ＋“直す”/doctor、/fork はバックグラウンド化・暴走ガードも／v2.1.212 と Microsoft研究のPR+24%

デスクトップ版にサンドボックス内蔵ブラウザ（Cmd/Ctrl+Shift+B）、/doctor は診断から自動修正へ。/fork はバックグラウンドセッション複製、旧サブエージェントは /subtask に分離し、WebSearch/サブエージェントに各200の上限。v2.1.212 で足回りを固め、Microsoft の査読研究はマージPR約24%増を実証。

📅 2026年7月18日 ⏱ 約2分 🔗 出典 4 件

今週のClaude Codeは「作業環境そのものをアプリに寄せる」方向の更新が続いた。ターミナルの黒い画面から、ブラウザ内蔵・自己診断・セッション分割まで、日々回す人ほど効く土台整備が主役だ。

何が新しいか

Week28（v2.1.202〜2.1.206）でデスクトップ版にサンドボックス化された内蔵ブラウザが入った。ドキュメントやデザイン、ローカルの dev サーバープレビューを開いて読み・クリックでき、起動は Cmd+Shift+B（Windowsは Ctrl+Shift+B）。外部サイトへの操作は安全性分類器がレビューする。あわせて /doctor が読み取り専用レポートから“診断して修正まで”へ進化——未使用の skill/MCP/プラグインをコンテキストコストと対比し、ローカルとコミット済み CLAUDE.md の重複排除、遅い hook の検出まで行う。

安定化とエビデンス

リリースは v2.1.212。Web検索・MCP・worktree・リモートセッションの信頼性修正が中心で、/fork・/resume・/background フローとガードレール強化の延長線上にある。事例面では、2026年7月公開の Microsoft Research 査読論文が、Claude Code 導入者はマージ済みPRが約24%増で効果が4ヶ月持続と報告。一方でトークン支出は組織規模で年数百万ドル級になり得るとし、予算ガバナンスの重要性も指摘した。

自分が試すなら

まず claude update → /doctor を回し、肥大した CLAUDE.md 重複や未使用 skill/MCP を掃除してコンテキスト単価を下げる。重いリサーチは /subtask で隔離し、暴走上限を環境変数で自分のワークロードに合わせる。デスクトップ内蔵ブラウザはローカルプレビュー確認に使うと画面往復が減る。

注意点

内蔵ブラウザの外部サイト操作は分類器レビュー前提で万能ではない。上限200はワークロード次第で早々に当たるので、長時間ジョブは CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION 等で明示調整を。24%増は特定環境の観測値で、CLAUDE.md/hook 整備なしに同じ伸びは出ない。

ひとことまとめ

「更新して /doctor で掃除、/subtask で隔離、上限で暴走を止める」週。環境を整えるほどコストと安定性の両方が効いてくる。
