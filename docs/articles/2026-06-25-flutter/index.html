Flutter のフルスタックAI化 — Genkit Dart プレビューと GenUI（A2UI）、エコシステムは月150万開発者へ

Flutter が「描画フレームワーク」から「AIを織り込んだフルスタック基盤」へ踏み出している。オープンソースのプレビューとして Genkit Dart が登場し、サーバ〜クライアントを横断するエージェント型 Dart/Flutter アプリを構築できる。クライアントの GenUI SDK は A2UI（Agent-to-UI）プロトコル上で、AIが実ウィジェットをその場で構成する。pub.dev は直近30日で13億DL超、月間アクティブ開発者は150万人（前年比+50%）。

📅 2026年6月25日 ⏱ 約3分 🔗 出典 3 件

なぜ今これが話題か

これまで Flutter は「描画とUI」の領域が中心だったが、生成AIをアプリの内側に常駐させる需要が高まっている。Google I/O 2026（2026-05-18）以降、安定版は Flutter 3.44.0 / Dart 3.12。年4回の安定リリースを継続しつつ、今回はAIとエコシステムの拡大が主役になった。

何が新しいか

第一に Genkit Dart がオープンソースのプレビューで登場。サーバからクライアントまで横断して、AIを組み込んだフルスタックのエージェント型 Dart/Flutter アプリを構築できる。第二に、クライアントの GenUI SDK が A2UI（Agent-to-UI）プロトコル上に構築され、AIエージェントが実 Flutter ウィジェットをその場で構成する。ストア更新や事前定義レイアウトなしに、完全インタラクティブな画面を動的生成できるのが新しい。Agentic Hot Reload は Dart & Flutter MCP Server 経由で起動中アプリを自動ホットリロードする。エコシステム指標も強く、pub.dev は直近30日で13億パッケージDL超、Flutter は両アプリストアで2番目に人気のモバイルSDK、月間アクティブ開発者は150万人（前年比+50%）。採用例に 2026 Toyota RAV4 のインフォテインメント、LG webOS SDK for Flutter（スマートTV向け）が挙がる。

仕組み（A2UI の流れ）

従来の「事前に組んだ画面にデータを流す」設計に対し、GenUI/A2UI はエージェントの応答そのものがUI構成になる。レイアウトを固定せず、対話の文脈に応じて必要なウィジェットを生成する発想だ。

自分が試すなら

  • Genkit Dart のプレビューで、サーバ側ロジックとFlutterクライアントを1スタックに通す最小サンプルを作る。
  • GenUI SDK で「問い合わせフォームをAIが生成」など、レイアウト固定なしの動的画面を1画面だけ試す。
  • Dart & Flutter MCP Server ＋ Agentic Hot Reload で、編集→反映ループの速さを体感する。

注意点

Genkit Dart・GenUI はいずれもプレビュー段階で、APIやプロトコルは変わりうる。動的UIは便利だが、レイアウトの予測可能性・アクセシビリティ・テストが難しくなりやすい。本番は GenUI を一部画面に限定し、堅い部分は従来のウィジェット＋状態管理（新規は Riverpod 3.0 が既定）で固めるのが無難。

ひとことまとめ

Flutter は 3.44/Dart 3.12 を土台に、Genkit Dart と GenUI でAIフルスタックへ広がりつつある。まずはプレビューを小さく触り、動的UIの効きどころと落とし穴を自分の手で測っておきたい。
