Claude Code、6月の地殻変動 — 課金が「対話」と「自動化」で分かれ、/cd・Tool(param:value)・mcp login で実務が固まる

2026年6月15日、AnthropicはAgent SDK・ヘッドレス実行・GitHub Actionsをサブスク枠から切り離し、自動化分をAPI料金の月次クレジットへ移した。同じ月、CLI側も /cd・パラメータ値ベースの権限・シェルからのMCP認証で実務装備が一段進む。対話補助から自動化基盤への移行を、課金と機能の両面から読む。

📅 2026年6月30日 ⏱ 約3分 🔗 出典 3 件

なぜ今これが話題か

Claude Code はこの半年で「対話的なコーディング補助」から「複数エージェントを束ねる自動化基盤」へと重心を移してきた。その地殻変動が会計に表れたのが2026年6月15日の課金変更だ。Anthropic は Agent SDK・ヘッドレス実行（claude -p）・Claude Code GitHub Actions・サードパーティ製エージェントを、これまでの月額サブスクリプションの利用制限から外し、API 料金で課金される別建ての月次クレジットへ移した。「人が画面の前で打つ利用」と「プログラムが回す利用」が、ようやく会計上はっきり分離されたことになる。

何が変わったか

同じ6月、CLI（v2.1.153〜v2.1.178）側も実務装備が進んだ。/cd（v2.1.169）はプロンプトキャッシュを壊さずにセッションの作業ディレクトリを移せる。権限は Tool(param:value) 構文が入り、ツール単位ではなく入力パラメータの値まで見て許可・拒否できる。モデルは Claude Opus 4.7 が Max/Team Premium の既定になり、最上位の xhigh エフォートも選べる。週次更新では対話メニュー無しでシェルから認証する claude mcp login/logout、シェルモードが npm test などのコマンド出力に自動で応答、/rewind が /clear 前まで遡及、背景サブエージェントの権限プロンプトをメインに表示——といった細かな改善が並んだ。

仕組みのポイント

分離のキモは「自動化はトークンを大量に焼く」前提が明示された点だ。入れ子サブエージェントやフォールバックチェーンを多用するほど、リトライと階層の分だけ消費が膨らむ。サブスクの一律制限に押し込めると、ヘビーな自動化が他の対話利用を圧迫してしまう。別建ての従量クレジットにすることで、自動化の実コストが可視化され、CI やバッチの設計を「いくら使うか」で考えられるようになる。Tool(param:value) 権限はこの流れと相性が良く、たとえば「Bash(rm:*) は拒否」のように危険な値だけを止める粒度の防御を組める。

自分が試すなら

  • このタスクのような毎朝の自動処理は claude -p 経路に乗るので、6/15以降はAPI月次クレジットの消費として別枠で見積もる。
  • 危険操作は Tool(param:value) でピンポイントに拒否ルール化し、許可は広く取りすぎない。
  • 長い調査セッションは /cd でキャッシュを保ったまま対象リポジトリへ移動して使い回す。

注意点

自動化を多用するなら、6/15の課金分離で想定外のAPI請求が出ていないかを最初に確認したい。ヘッドレスやGitHub Actionsを回している人ほど影響が大きい。エージェント単位のコスト按分・スコープ付き権限と合わせ、「どの自動化がいくら使ったか」を必ず監視する。

ひとことまとめ

6月のClaude Codeは、課金で「対話」と「自動化」を分け、CLIで /cd・Tool(param:value)・claude mcp login を足して実務基盤を固めた。自動化前提で使うなら、コストの見える化と値ベースの権限設計が主導権を握る。
