Flutter 3.44 のツールチェーン刷新 — Widget Preview に Inspector 内蔵・DevTools が WASM 既定でメモリ最大50%減

Flutter 3.44.0(2026-05-18)/Dart 3.12.1 の本丸は派手な新ウィジェットより開発体験(DX)の地殻変動。Widget Preview が Flutter Inspector を完全内蔵し、プレビュー検出を Dart Analysis Server ベースに刷新して IDE 全体のメモリを最大50%削減。DevTools は WASM 既定レンダラへ。静かだが効くツールチェーン刷新を読み解く。

📅 2026年6月19日 ⏱ 約3分 🔗 出典 3 件

なぜ今これが話題か

安定版は Flutter 3.44.0(2026-05-18) / Dart 3.12.1(2026-05-26)。3.45 は未リリースで現行は 3.44 系だ。3.44 の主役は新ウィジェットではなくツールチェーン(DX)の地殻変動。毎日コードを書く時間の「待ち」と「メモリ」に直接効くので、地味だが体感は大きい。

何が新しいか

  • Widget Preview が Flutter Inspector を完全内蔵：ウィジェットツリーの検査・render object 確認・プロパティ編集がプレビュー内で完結。
  • プレビュー検出を Dart Analysis Server ベースに刷新し flutter ツールのメモリを削減 → IDE 全体のメモリ使用を最大50%削減。Preview filtering(group/name/URI)で絞り込み可。
  • DevTools が WebAssembly(WASM) を既定レンダラに → 起動高速化、巨大ツリーのスクロールが滑らか、Timeline が高頻度フレームをジャンクなく処理。
  • 構造変化：Material/Cupertino を material_ui/cupertino_ui へ SDK から分離、SwiftPM 既定で CocoaPods 終焉・Rosetta 不要。
  • Dart 3.12：private named parameters 正式(先頭 _ を自動で外して公開名に)。

仕組みのポイント

メモリ削減の肝はプレビュー検出の担い手を替えたこと。従来 flutter ツールが担っていたプレビュー対象の探索を、IDE がすでに常駐させている Dart Analysis Server に寄せたため、重複した解析プロセスが減る。DevTools の WASM 化は、UI ロジックを JS ではなく WASM で実行することで起動とフレーム処理を底上げする狙いだ。

自分が試すなら

  • 3.44 へ上げ、Widget Preview 内の Inspector でツリー検査→プロパティ編集まで一筆書きで回す。
  • 大きめプロジェクトで Preview filtering を使い、対象を絞って起動を軽くする。
  • iOS は SwiftPM 既定化に伴い CocoaPods 依存を棚卸し。

注意点

Material/Cupertino の material_ui/cupertino_ui 分離は import 経路に影響しうるので、移行時は依存とパスを確認。SwiftPM 既定化は既存の CocoaPods 構成と衝突する場合があるため、CI のキャッシュ前提も見直す。新機能のメモリ計測は profile モードで行い、debug の数値で判断しない。

ひとことまとめ

3.44 は「速くなった・軽くなった」が主役。Widget Preview×Inspector の一体化と Analysis Server ベース化で IDE メモリ最大50%減、DevTools は WASM 既定。派手さはないが毎日の開発の摩擦を確実に削るアップデートだ。
