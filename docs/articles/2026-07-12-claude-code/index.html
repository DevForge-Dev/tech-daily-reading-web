Sonnet 5がClaude Codeの既定モデルに — 1Mトークン時代の始まりとFable 5最終日

v2.1.197でSonnet 5がデフォルト化。ネイティブ1Mトークンと8/31までの$2/$10プロモ、そして有料プラン込みのFable 5アクセスは今日7/12が最終日。

📅 2026年7月12日 ⏱ 約3分 🔗 出典 5 件

Claude Codeの既定モデルがv2.1.197でClaude Sonnet 5に切り替わった。目玉はネイティブ1Mトークンのコンテキストウィンドウで、8月31日までは入力$2/出力$10（Mtokあたり）のプロモーション価格が適用される。あわせて/configに「Dynamic workflow size」設定が追加され、動的ワークフローの規模を制御できるようになった。

もうひとつ、今日が区切りになるのがFable 5だ。有料プランに含まれるFable 5アクセスは当初7月7日期限だったが7月12日（今日）まで延長されており、期限後は$10/$50のクレジット課金へ移行する。週次上限の50%キャップは据え置き。「日常はSonnet 5の1Mコンテキスト、勝負どころだけFable 5を従量で」という使い分けが現実的な運用になる。

クラウド側も足並みが揃った。Bedrock・Vertex AI・Microsoft Foundry経由のClaude Codeでautoモードがデフォルトになり、BedrockのモデルはOpus 4.8へ更新。政府機関向けにはFedRAMP High認証のClaude for Government DesktopでClaude Code/Coworkが公開ベータ入りし、改ざん検知可能な監査ログや支出ガバナンスまで備える。

記事面ではTotalumのSkills完全ガイドが実用的だった。プロンプトのテンプレはスラッシュコマンド、ドメイン知識＋補助ファイルはSkill（SKILL.mdを置くだけ）、隔離した並列作業はSubagent、コードで確実に強制するルールはHook、外部システム接続はMCPという判断基準は、CLAUDE.md肥大化の処方箋としてそのまま使える。

事例面では、Anthropicが新規AI法人買い手の73%を獲得し年換算売上190億ドル、Claude Code単体でも年換算25億ドルという報道も出た。自分が試すなら、プロモ期間中に1Mコンテキストへ「リポジトリ全体＋関連ドキュメント丸ごと」を渡すレビューを回し、コストと精度の体感を掴んでおきたい。ただし巨大コンテキストは万能ではなく、Zennやビジネス+ITのTips記事が言う通り「不要な文脈を切り、参照ファイルは直接渡す」文脈設計は1M時代でも品質を左右する。

ひとことまとめ: 既定がSonnet 5・1Mトークンになり、工夫の軸は「文脈を削る」から「何を盛るか設計する」へ。Fable 5を試すなら今日が最終日。
