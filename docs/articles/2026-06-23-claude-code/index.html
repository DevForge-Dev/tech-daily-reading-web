Claude Code

Claude Code に Dynamic Workflows — 1タスクを最大1,000サブエージェントへ自動分解・並列実行、ultracode で単発起動

エージェントに大きな目標を丸ごと任せたいが、手作業でサブタスクへ割るのは面倒——その溝を埋めるのが公式発表の Dynamic Workflows。Claude 自身が目標をオーケストレーションスクリプトに落とし込み、分解→並列実行→検証→統合まで自動で進める。最大1,000サブエージェントを並列で動かせ、実行前にワークフローを検査できる。あわせて v2.1.178 で権限やセキュリティも強化された。

📅 2026年6月23日 ⏱ 約3分 🔗 出典 3 件

なぜ今これが話題か

エージェントに「リポジトリ全体を横断する大きな目標」を任せたいとき、これまでは人間が手作業でサブタスクへ割り、順番や並列度を設計する必要があった。Anthropic 公式が発表した Dynamic Workflows は、その分解と統括を Claude 自身に肩代わりさせる仕組みだ。リサーチプレビューとして、InfoQ など第三者メディアも取り上げている。

何が新しいか

核心は、Claude が複雑な目標を JavaScript のオーケストレーションスクリプトに落とし込み、サブタスクへの分解 → 並列実行 → 検証 → 統合までを自動で進める点。リサーチプレビューでは 最大1,000のサブエージェントを並列に動かせる。生成されたワークフローは実行前に内容を検査でき、再現性のある運用に一歩近づく。v2.1.154 以降で全有料プラン・API に加え Amazon Bedrock / Google Vertex / Microsoft Foundry でも使え、プロンプトに ultracode と書けばそのタスクだけをワークフローとして走らせられる。

同時に入ったセキュリティ強化（v2.1.178）

6月15日リリースの v2.1.178 では、権限ルールが入力値まで見て判定する Tool(param:value) 構文に対応。入れ子の .claude/skills は最も近い作業ディレクトリの定義が優先で読み込まれる。オートモードではサブエージェント起動前に分類器が評価し、レビューなしでブロック済みアクションを要求できる抜け穴を塞いだ。さらに公式 changelog によれば、ユーザーが指示していない git reset --hard / git clean -fd / git stash drop や、対象スタックを指定しない terraform/pulumi/cdk destroy は既定でブロックされる。

自分が試すなら

  • 大きめの調査・リファクタを ultracode で単発ワークフロー化し、生成スクリプトを実行前に必ず読む。
  • 並列度は最初から最大を狙わず、小さなサブタスク数で挙動とコストを確かめる。
  • 破壊的 git/terraform のブロック挙動を前提に、意図的な破棄は明示指定する運用へ。

注意点

最大1,000という数字は上限であって推奨ではない。並列が増えるほどコスト把握とデバッグが難しくなるので、まずは検査可能な小さなワークフローから。リサーチプレビュー段階のため、対象プラン・挙動は今後変わり得る点も踏まえたい。

ひとことまとめ

「分解と統括をエージェントに任せる」段階に入った。鍵は実行前にスクリプトを読めること。まずは ultracode で1タスクから試すのが安全だ。
