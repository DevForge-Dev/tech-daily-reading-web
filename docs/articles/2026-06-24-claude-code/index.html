Claude Code v2.1.186 — CLIから直接MCP認証（claude mcp login）、Artifacts搭載、ワークフロー起動語が ultracode に

MCPサーバーの認証はこれまで対話式の /mcp メニューを開く必要があり、SSH越しの自動化と相性が悪かった。6月22日の v2.1.186 は claude mcp login/logout でCLIから直接認証でき、--no-browser でSSH越しも完結する。あわせてセッションの作業をその場更新できる Artifacts、新モデル Fable 5 と Opus 4.8、起動語の ultracode 改名が入った。拡張の4要素（Skills/Hooks/Rules/Subagents）の公式整理も同時期に出ている。

📅 2026年6月24日 ⏱ 約3分 🔗 出典 4 件

なぜ今これが話題か

エージェントに長時間・自動の作業を任せるほど、外部ツール連携（MCP）の認証を人手の対話なしに通せるかが効いてくる。従来は /mcp メニューを開いて認証する必要があり、SSH越しのヘッドレス運用では詰まりがちだった。6月22日リリースの Claude Code v2.1.186 は、ここを正面から潰してきた。

何が新しいか

目玉は claude mcp login <name> / claude mcp logout <name>。対話メニューを開かずCLIから直接MCPサーバーを認証できる。--no-browser の stdin リダイレクトに対応し、SSH越しでも認証を完了できる。動的ワークフローの起動キーワードは workflow から ultracode に改名され、effortメニューから呼ぶと effort を xhigh に設定しつつ、ワークフロー化すべきかは Claude が自動判断する。さらに /workflows のエージェント詳細に f キーでステータス絞り込み、/plugin の Installed タブに Skills 欄が追加された。別系統では、セッションの作業をその場更新できる共有Webページ Artifacts 搭載、新モデル Fable 5（/model fable）と Opus 4.8、Bedrock/Vertex/Foundry でのオートモード拡大も入っている。

あわせて整理された拡張の4要素

同時期に Anthropic 公式ブログが Skills / Hooks / Rules / Subagents の使い分けを整理した。Hooks はライフサイクル上の決定的な自動化（25のライフサイクル点で発火し、PreToolUse はツール実行前の主要なセキュリティ関門）、Subagents は独立コンテキストでの並列委譲で冗長な出力を隔離し要約だけ返す、Skills はオンデマンドで読み込まれる再利用可能な指示パック。CLAUDE.md を肥大化させずにプロジェクト固有の振る舞いを教える設計指針だ。

自分が試すなら

  • リモート開発機で claude mcp login + --no-browser を使い、MCP認証をスクリプト化する。
  • 大きめの調査を ultracode で単発ワークフロー化し、effort=xhigh の挙動とコストを確かめる。
  • 繰り返す手順は Skills 化、止めたい操作は Hooks（PreToolUse）で防御、という棲み分けを試す。

注意点

Fable 5 / Opus 4.8 はモデル特性が異なるため、品質が重要なタスクは出力を見て選ぶ。Artifacts の組織内限定共有・バージョン履歴は Team/Enterprise のベータ扱い。respondToBashCommands など新しい自動応答挙動は settings.json で従来動作に戻せる点も把握しておきたい。

ひとことまとめ

v2.1.186 は「自動化の足回り」を磨いた更新。まずは claude mcp login と ultracode から触れると、ヘッドレス運用と大きめタスクの両方が一段楽になる。
