エージェントに長く任せるほど、認証のしづらさ・出力の取りこぼし・誤って消した会話といった「運用の摩擦」が効いてくる。6月第4週の週次リリース（v2.1.185〜193）と v2.1.190 の changelog は、claude mcp login/logout、!bash 出力への自動応答、/rewind による巻き戻し、/workflows のステータス絞り込みを投入。あわせて有料プランの利用枠が「対話」と「自動化」に分離された。派手な新機能より、長時間運用を止めない地ならしのアップデートだ。

なぜ今これが話題か

Claude Code を長く動かすほど、地味な運用の摩擦が効いてくる。MCPサーバーへの認証は対話メニューを開かないと通せない、シェル実行の結果を毎回こちらが促さないと解説が進まない、/clear で会話を消したらもう戻れない——こうした「止まる・取りこぼす・消える」を減らす運用系の改善が、6月第4週の週次リリース（v2.1.185〜193）と v2.1.190 の公式changelogでまとめて入った。

何が新しいか

目玉は4つ。claude mcp login / claude mcp logout は、対話的な /mcp メニューを開かずにシェルから直接設定済みMCPサーバーへ認証・資格情報クリアができ、--no-browser でSSH越しの認証完了にも対応する。シェルモード（!bash）はコマンド出力に自動で応答するようになり、npm test の結果を再プロンプトなしで解説する（respondToBashCommands で無効化可）。/rewind は /clear 実行前の会話状態を復元でき、誤クリアからの復旧手段になった。/workflows のエージェント詳細ビューでは f でステータス絞り込み、/plugin の Installed タブには Skills セクションが追加。さらにバックグラウンドのサブエージェントは権限プロンプトを自動拒否せずメインセッションに表示、アイドルなbg shellはメモリ圧迫時に自動回収される（CLAUDE_CODE_DISABLE_BG_SHELL_PRESSURE_REAP=1 で無効化）。

claude mcp login の動き

これまでMCP認証は /mcp の対話メニュー前提で、ヘッドレスやSSH環境では詰まりやすかった。新コマンドはシェルから完結し、ブラウザを開けない環境では --no-browser で表示されたURLを手元で開いてコードを貼り戻す方式に切り替わる。

自分が試すなら

  • リモート開発のSSHセッションで claude mcp login --no-browser を試し、ブラウザなしで認証が通るか確認する。
  • !bash でテストを流し、結果を再入力なしで解説してくれる流れを体験する。挙動が騒がしければ respondToBashCommands を切る。
  • 誤って /clear した直後に /rewind で戻れるか試し、復旧手段として頭に入れておく。

注意点

同じ6月15日からClaude有料プラン（Pro/Max/Team/Enterprise）の利用枠が「対話（Interactive）」と「自動化（Programmatic）」に分離された。Claude Code の対話利用は対話枠だが、スケジュール起動やAPI連携などの自動実行は別枠扱いになる。本タスクのような毎朝の自動実行を多用する場合、自分の使い方がどちらの枠に当たるかを把握しておかないと、思わぬ枠の消費につながる。!bash の自動応答も、副作用のあるコマンドでは流れに任せすぎないこと。

ひとことまとめ

今回は新機能の派手さより、認証・出力・復旧・運用の摩擦をまとめて削った地ならしの更新。claude mcp login でヘッドレス認証が通り、!bash 自動応答で手数が減り、/rewind が安全網になる。あわせて利用枠が対話/自動化に分かれた点は、自動実行を組む側として必ず押さえておきたい。
