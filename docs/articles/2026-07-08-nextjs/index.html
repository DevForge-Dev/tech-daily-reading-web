宣言的ルート遷移 — React <ViewTransition> × Next.js experimental.viewTransition

これまでMotion等が手作業でやっていた「mount/unmount管理・要素の位置追跡・タイミング調停」を、React公式の <ViewTransition> が宣言的に置き換える。next.config で experimental.viewTransition:true を立て、連続させたい要素に同じ name を付けるだけ。Next.jsのルート遷移は自動でTransition扱いになるので startViewTransition を自分で呼ぶ必要がない。enter/exit/update/share の4発火タイプと落とし穴を押さえる。

📅 2026年7月8日 ⏱ 約4分 🔗 出典 3 件

なぜ今これが話題か

6/15の回では「素のView Transitions API」「next-view-transitions」「React実験API」の3方式を俯瞰した。今日はそのうちReact公式の <ViewTransition> コンポーネントを、Next.js 16.2 の experimental.viewTransition と組み合わせる方式に絞る。従来ライブラリが担っていたライフサイクル管理や位置追跡を、「連続させたい要素に name を付ける」だけの宣言に落とせるのが要点だ。

何が新しいか

有効化は next.config.ts に experimental: { viewTransition: true } を書くだけ。<ViewTransition> はReact の Canary / Experimental 限定APIだが、App Router は React canary を使うため react@canary を自前で入れる必要はなく import { ViewTransition } from 'react' で使える。主要3ブラウザすべてが View Transitions API 対応で、未対応でもアプリは通常動作しアニメが付かないだけ（graceful degradation）。重要なのは発火条件で、素の setState では発火せず、Transition・Suspense・useDeferredValue に包まれた更新でのみアクティブ化する。Next.jsのルート遷移はこれに該当するので遷移では自動発火する。

仕組みのポイント

Reactは最寄りDOMノードのインラインスタイルに view-transition-name を（アニメ参加の瞬間だけ）付与し、startViewTransition を自動で呼ぶ。だから自前で startViewTransition を走らせると割り込んで中断する——View TransitionはReact側に一元化する。カスタマイズは name 直書きの疑似セレクタではなくView Transition Class推奨で、enter/exit/update/share/default 各propにクラス名を渡すとReactが子要素へそのクラスを付ける。

// 共有要素モーフ：グリッドと詳細で同じ name
<ViewTransition name={`photo-${id}`}><Image .../></ViewTransition>

/* CSS はクラスで受ける */
::view-transition-group(.slide-in) { /* ... */ }

実務での使いどころ

サムネ→ヒーロー画像のモーフ（同一 name）、Suspenseスケルトン→本体のリビール（fallbackに exit・本体に enter・default="none"）、<Link transitionTypes={['nav-forward']}> による方向付きスライド、同一ルート内の key 差し替えクロスフェード——の4パターンが定石。まずは共有要素モーフから試すのが費用対効果が高い。

注意点・落とし穴

enter/exitは先頭配置のときだけ発火（上に<div>があると起きない）。name はアプリ全体で同時に1つだけ——リストで同名を量産するとエラーになるため item-${id} で一意化する。戻る/進む（popstate）はView Transitionをスキップするので方向スライドは戻るボタンでは鳴らない。prefers-reduced-motion はReactが自動で切らない——CSSで明示的に無効化する必要がある。

ひとことまとめ

フラグを立て、連続させたい要素に name を付け、カスタムは View Transition Class で。startViewTransition はReactに任せ、name一意化・先頭配置・reduced-motionだけ手当てすれば、宣言的にルート遷移が動く。
