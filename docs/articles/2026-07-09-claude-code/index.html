Claude Code v2.1.203/204 — サブエージェント既定バックグラウンド化と自動化運用の地固め

スラッシュ・スキルの5連読み込みとサブエージェントの既定バックグラウンド実行で日常のスループットが向上。ヘッドレスのフック安定化やログイン期限警告など、CI・自動化まわりの足元も固まった。

📅 2026年7月9日 ⏱ 約3分 🔗 出典 5 件

Claude Code は小刻みなリリースを高頻度で重ねるスタイルだが、今回の一連の更新は「エージェントを並列に、自動化を安定に」という方向がはっきり見える。派手な新機能より、毎日使うワークフローの摩擦を削る改善群だ。

まず公式 changelog から。/skill-a /skill-b のようにスラッシュ・スキルを連ねて呼ぶと、従来は先頭しか読まれなかったところ、先頭から最大5つまで読み込むようになった。さらにサブエージェントが既定でバックグラウンド実行になり、実行中も本流の作業を続けられ、完了時に通知が届く。探索・レビューのような重い並列作業を「投げて待たずに進む」形が標準になった。

バージョン別では、7/7 の v2.1.203 がログイン有効期限が近づくと警告を出して再認証を促す機能、手動権限モード時のフッター一時停止バッジ（⏸）、セッションの追加作業ディレクトリを MCP の roots/list に反映して変更時に notifications/roots/list_changed を送る対応、macOS の低メモリ誤検知でセッション切替が15〜20秒止まる不具合の修正を含む。7/8 の v2.1.204 は、ヘッドレス実行の SessionStart フック中にイベントがストリームされず、フック実行途中でワーカーが「アイドル」と誤判定されて回収（idle-reap）される不具合を修正した。CI/CD で長めの初期化フックを走らせる構成には効く修正だ。

周辺では、DEV Community に hooks の2026年版完全ガイド（すぐ使える20以上の実例つき）が出た。hooks はプロンプト指示と違い毎回決定論的に発火するので、Write/Edit 後の prettier 強制、本番 push のブロック、シェルコマンドの監査ログといった「取りこぼしが許されない」処理に向く。ニュース面では、週間利用上限の50%引き上げ（7/13まで・既存プラン対象）、Claude Cowork のクラウド化と M365 書き込みツール追加、そして「Anthropic の新規本番コードの約80%を Claude が執筆」という数字が話題になった。

自分が試すなら、まず CI で使っているバージョンを v2.1.204 以降へ上げて SessionStart フックの安定性を確認し、次に重めのレビュー用サブエージェントをバックグアウンド前提のフローに組み直す。注意点は、サブエージェントの完了を暗黙に待っていた既存の手順が既定変更で崩れないか、通知の受け方を含めて一度見直すこと。

ひとことまとめ: 新機能より「並列で速く、無人でも落ちない」を積み上げた回。自動化運用者ほど恩恵が大きい。
