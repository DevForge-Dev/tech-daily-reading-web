Dart 3.12の新構文を実務へ＋Riverpod 3.0 Mutations／自動リトライ＋Impellerで120fps

Dart 3.12のdot shorthands・private named parameters・primary constructors(experimental)で記述を圧縮。安定版はFlutter 3.44.0/Dart 3.12.2、次は2026年8月。Riverpod 3.0のMutations(Idle/Pending/Success/Error)＋指数バックオフ自動リトライ(200ms→6.4s)で書き込みUXを強化。Impellerはラスタライズ約50%短縮で120fps、Skia比約17%省電力。

📅 2026年7月16日 ⏱ 約3分 🔗 出典 4 件

今日のFlutterは言語・状態管理・描画の三方向で「2026年の実務ライン」が見えた日。Dart 3.12の新構文、Riverpod 3.0の書き込みUX、Impellerの120fpsを順に押さえる。

Dart 3.12：冗長さを削る新構文

目玉は dot shorthands。型が文脈から明らかなら .value と書け、enum・static メンバー・named constructor に対応して型名の反復を減らせる。private named parameters は先頭 _ を自動で外して公開名にでき、private フィールドを名前付きで直接初期化できる。primary constructors はクラス宣言を圧縮する（experimental、primary-constructors フラグ）。安定版は Flutter 3.44.0 / Dart 3.12.2、次の機能リリースは 2026年8月予定（年4回：2/5/8/11月）。

// dot shorthands（型が文脈から明らか）
Color c = .blue;            // = Color.blue
enum Mode { light, dark }
Mode m = .dark;             // = Mode.dark
// primary constructors はまだ実験的。本番は様子見

Riverpod 3.0：書き込みUXのレジリエンス

新機能 Mutations は「Login」「Post Comment」等のアクションを定義すると Idle/Pending/Success/Error のライフサイクルを UI へ自動公開し、ローディング/エラー表示を定型化する。さらに自動リトライがネットワーク失敗時に指数バックオフ（200ms→倍々で最大6.4s）で再試行し、一時的な glitch を恒久エラー画面にしない。opt-in のオフライン永続化（stale-while-revalidate, riverpod_sqflite）も入った。現行版は Riverpod 3.3.x。新規の既定は Riverpod、監査/大規模は Bloc 9.x という役割分担は不変だ。

Impeller：120fpsと電力効率

Impeller は iOS＋モダンAndroid（API 29+）で既定。全シェーダーをビルド時（AOT）にオフラインコンパイルして実行時のシェーダージャンクを排除し、フレームラスタライズ約50%高速で 120fps を安定維持、アニメ多用時は Skia比 約17%低い電力ドレイン。フレーム予算はラスタライズ <8.3ms＝120fps。計測は必ず profile モードで、DevTools の Impeller ビューで overdraw を潰す。

自分が試すなら / 注意点

まず Riverpod の Mutations を1つの書き込み画面に入れて、ローディング/リトライの定型を体感するのが早い。Dart 3.12 は dot shorthands から採り入れ、primary constructors は構文が変わり得るので本番は保留。Rive を多用するなら Factory.rive と大きな .riv のプリロード（build内でロードしない）を徹底する。

ひとことまとめ

言語は短く、状態管理は粘り強く、描画は120fps。3つとも「2026年の既定」を今日更新した。
