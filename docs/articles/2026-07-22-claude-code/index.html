v2.1.215〜216——/verify・/code-review を手動呼び出しへ、ripgrep 標準化、長時間セッションの二乗遅延を解消

v2.1.215 で /verify・/code-review の自発実行をやめ、必要なときに明示的に呼ぶ方式へ。ripgrep をコードベース検索の標準ツールとして明記。v2.1.216 は sandbox.filesystem.disabled を追加し、長時間セッションで正規化コストがターン数の二乗で膨らむストール、OAuth 失効後の HTTP 401 誤検知を修正。直前の v2.1.207〜212 では /fork のバックグラウンド化と WebSearch/サブエージェント上限も入った。

📅 2026年7月22日 ⏱ 約3分 🔗 出典 2 件

この日のClaude Codeは自律実行の安全弁と検索基盤の整理が主役だ。v2.1.215〜216で、暴走しがちな自動実行を手動呼び出しへ引き戻し、長時間セッションの構造的な遅延を潰した。号令一下で数十分走らせる運用が当たり前になったからこそ、"止める・速くする"側の更新が効く。

何が新しいか

v2.1.215の目玉は/verify・/code-reviewの自動実行停止。これまでClaudeが自発的に走らせていたのをやめ、必要なときにユーザーが明示的に呼ぶ方式へ変えた。あわせてripgrepをコードベース検索の標準ツールとしてドキュメント上で明記し、検索を高速・一貫にした。v2.1.216ではsandbox.filesystem.disabledを追加——ネットワーク下り制御は保ったままファイルシステム隔離だけ外せる。さらに長時間セッションでメッセージ正規化コストがターン数の二乗で膨らみ数秒ストールする問題と、OAuthトークン失効/ローテーション後にautoモードがHTTP 401を誤検知してコマンドを拒否する不具合を修正した。

背景——/fork のバックグラウンド化と暴走ガード

直前のv2.1.207〜212では、/forkが会話を新しいバックグラウンドセッションへコピーし、メイン作業を止めずに分岐できるようになった。暴走ループ対策としてWebSearch呼び出しとサブエージェント生成にセッション単位の上限（既定200）を導入（CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION等で調整可）。2分を超えるMCP呼び出しは自動でバックグラウンドへ移り、経過時間のライブカウンタで「止まって見える」問題も解消した。

自分が試すなら

自動実行停止は黙って効くが、CIやレビュー運用で/verifyを回していたなら明示呼び出しへ書き換えが要る。長時間の定期タスクは二乗遅延修正で体感が改善するはず。sandbox.filesystem.disabledはネットワーク制御を残しつつFS隔離が邪魔な回に限定して使う。

注意点

明示呼び出し化で「勝手にレビューしてくれる」前提のワークフローは黙って素通りになる——呼び忘れに注意。サブエージェント/WebSearch上限200は重い調査タスクだと途中で頭打ちになるので、必要なら環境変数で引き上げる。

ひとことまとめ

「自動をやめて明示へ、検索はripgrepへ、長時間は二乗遅延を解消」。派手さはないが、毎日エージェントを回す人ほど効く"守り"の回だ。
