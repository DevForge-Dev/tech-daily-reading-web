CSS linear()でスプリング/バウンス — JSライブラリなしで物理アニメーションを書く

cubic-bezier()では原理的に不可能だったバネの揺り戻しを、純CSSのlinear()タイミング関数で表現できる。Baseline済みでhydrationゼロ、Server Componentのまま物理系の質感を出せるが、durationとのペア管理と中断への弱さが設計上の勘所。

📅 2026年7月10日 ⏱ 約3分 🔗 出典 5 件

スプリング（バネの揺り戻し）やバウンスは、これまでMotionやReact Springといったメインスレッド駆動のJSライブラリでしか作れなかった。cubic-bezier()は値が0〜1に縛られ、ターゲットを行き過ぎて戻る「オーバーシュート」を表現できないからだ。linear()関数はこの制約を外す。点列（stop）を直線でつなぐ折れ線でイージング曲線を近似する仕組みで、1を超える値を書けるため物理シミュレーションの記録をそのまま流し込める。2023年12月に主要3ブラウザが揃いBaseline済み（Chrome/Edge 113+・Firefox 112+・Safari 17.2+）、2026年は実戦投入フェーズだ。

構文はlinear(0, 0.25 75%, 1)のようにstop列と時間%で書くが、手書きは非現実的。Jake ArchibaldとAdam Argyleのlinear() easing generatorやEasing Wizard、Motionのspring→CSS出力で40〜50点を生成するのが前提になる。Chrome DevTools 114+にはビジュアル編集エディタもある。

Next.jsでの旨みは純CSSなので'use client'が不要なこと。Server Componentのままclassを付けるだけで、hydrationゼロ・バンドル増ゼロでINPに効く。実装はapp/globals.cssのトークンに集約する。

:root {
  /* フォールバック（擬似スプリングbezier） */
  --ease-spring: cubic-bezier(0.3, 1.4, 0.4, 1);
  --ease-spring-time: 800ms;

  @supports (animation-timing-function: linear(0, 1)) {
    --ease-spring: linear(0, 0.32, 0.72, 1.03, 1.16,
      1.15, 1.09, 1.02, 0.98, 0.99, 1.0, 1);
    --ease-spring-time: 1200ms;
  }
}

ここで重要なのが、transition宣言を2行並べる古典的フォールバックはCSS変数経由では機能しないという点。ブラウザはvar()の構文しか検証しないため、@supportsで変数定義側を上書きするのが正解だ。Tailwind v4なら@themeに置けばease-springユーティリティが生える。

落とし穴は3つ。第一に生成されたlinear()はセットのdurationとペアでのみ正しい——durationだけ変えると別物の動きになる。第二に中断に弱い。遷移中に逆方向へ切り替わるとreversing shortening factorで慣性を無視して即反転するため、連打されるトグルUIはJSスプリングの方が自然で、linear()は出現・ドロップ・トーストなど片道で完走する演出に向く。第三は運用面で、100点超でも実測fps低下はなく大型スプリング3本でgzip後+1.3kB程度だが、コピペ濫用は膨らむのでデザイントークンに集約する。動きが大きいぶんprefers-reduced-motion: no-preferenceの中でのみ付与するのも必須だ。

ひとことまとめ: 「片道の演出はlinear()、中断される相互作用はJSスプリング」という線引きさえ守れば、JSゼロで物理の質感が手に入る。
