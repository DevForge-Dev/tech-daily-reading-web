Claude Code

Claude Code、自動モードが破壊的コマンドを既定ブロック — git reset --hard / terraform destroy を遮断、Claude Design も大型刷新

エージェントに自動実行を任せるほど、取り返しのつかない操作の事故が怖い。最新の公式changelogでは、自動（auto）モードが git reset --hard・git clean -fd・terraform destroy などの破壊的コマンドを既定でブロックするようになった。あわせてストール表示の文言・しきい値の調整、JetBrains/Kitty端末の不具合修正、6月17日の Claude Design 大型刷新（/design-sync・/design）も到来している。

📅 2026年6月26日 ⏱ 約3分 🔗 出典 3 件

なぜ今これが話題か

自律的に走る時間が延びるほど、エージェントが不可逆な操作を踏んだときの被害が大きくなる。git reset --hard でローカル変更を吹き飛ばす、terraform destroy でスタックを消す——こうした事故を「明示的に指示していない限り起こさない」よう、自動モードの安全装置が一段強化された。

何が新しいか

公式changelogによると、自動モードはローカルの変更破棄を明示的に指示していない場合、git reset --hard・git checkout -- .・git clean -fd・git stash drop といった破壊的gitコマンドを既定でブロックする。さらに当該セッションでエージェント自身が作成していないコミットへの git commit --amend を遮断し、terraform destroy・pulumi destroy・cdk destroy も特定スタックを指定して依頼した場合を除いて停止する。信頼性面では、ストリーミングが滞った際のヒントが「No response from API · Retrying…」から「Waiting for API response · will retry…」に変わり、発火が10秒の沈黙から20秒に緩和。JetBrains系IDE(2026.1+)端末のちらつきを同期出力で解消し、Kitty系端末(WezTerm・Ghostty・kitty)で Shift+非ASCII(例 Shift+ä→Ä)が欠落する不具合、2.1.169 混入の起動リグレッション(約120ms増)も修正された。

Claude Design の大型刷新

6月17日、Anthropic は Claude Design を大型刷新し、Claude Code とのデザイン⇔コード双方向連携を追加した。/design-sync で既存コードベースのデザインシステムを Claude Design に取り込み、/design で Claude Design 側の修正・追加を Claude Code に同期する。9プラットフォームへのコネクタやブランドコントロールも加わり、ブランド一貫性を保ったまま UI 実装まで一本の流れにつなぐ狙いだ。

自分が試すなら

  • 自動モードを使う既存リポジトリで、reset --hard 等が確かに止まるかを安全な検証用ブランチで確認する。
  • IaC を扱うなら、destroy 系がスタック指定なしで止まる挙動を前提に運用ルールを更新する。
  • UIのある案件で /design-sync→/design の往復を一度通し、コード化までの分断が減るか体感する。

注意点

ブロックは「明示的に指示していない場合」の安全弁であり、意図的に破棄したいときは明示すれば実行できる。逆に言えば、危険な操作を雑に依頼すると通ってしまう余地は残るので、自動モードでも重要ブランチの保護や予算・権限のスコープ管理は別途必要。Claude Design 連携は対応プラットフォームやプランの条件を確認しておきたい。

ひとことまとめ

今回は「自動実行の安全性」を底上げした更新。破壊的 git/IaC コマンドの既定ブロックでまず足元の事故を防ぎ、Claude Design 連携でデザイン→実装の流れを縮める、という二本立てで効いてくる。
