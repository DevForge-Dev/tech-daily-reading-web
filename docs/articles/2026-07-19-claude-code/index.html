Claude Code

Opus 4.8が既定モデルに——Dynamic Workflowsで数百の並列サブエージェント、Week29でartifactがMCP接続

Claude Opus 4.8 が Max/Team/Enterprise/API の既定モデルに。Dynamic Workflows は1セッションで計画→数百の並列サブエージェント起動→検証→報告までを自動化（1Mトークン既定・最大128k出力、価格は4.7据え置き）。Week29では公開artifactが閲覧者のMCPコネクタでライブデータ取得・アクション実行が可能に。Hooks/Subagents/Skillsの3層整理と日本企業の定量効果も追う。

📅 2026年7月19日 ⏱ 約3分 🔗 出典 4 件

この日のClaude Codeは基盤モデルの世代交代が主役だ。Anthropicが最上位のエージェント/複雑コーディング用モデルとして Claude Opus 4.8 を公開し、Max・Team Premium・Enterprise従量課金・API の既定モデルになった。Bedrock/Vertex/AWS でも既定が Opus 4.8 へ移る流れで、コーディングアシスタントの土台を一段引き上げるニュースとして各所が報じている。

何が新しいか

目玉は Claude Code 向けの Dynamic Workflows（Enterprise・Team・Max でリサーチプレビュー）。従来の単発補完を超え、Claudeが作業を計画し、1セッション内で数百の並列サブエージェントを起動、各出力を検証してから報告するところまでを自動で回す。コンテキストは既定1Mトークン、出力は最大128kトークン。価格は Opus 4.7 から据え置きで、入力$5／出力$25、fast mode $10/$50。

あわせて Week29（7/13〜17）では、公開した artifact が閲覧者それぞれのMCPコネクタを通じてライブデータ取得やアクション実行を行えるように。公開共有リンク、Team/Enterprise のエディタロール、Claude Tag 由来 artifact、スクリーンリーダーモードも加わった。

拡張の3レイヤーを整理する

実務ガイド系の記事では、Claude Codeの拡張性を hook / subagent / skill の3層で捉える枠組みが定着してきた。決定的にルールを効かせたいなら hook、隔離した並列作業は subagent、再利用可能なワークフローは skill、という住み分けだ。

自分が試すなら

まず既定が Opus 4.8 になったことを確認し、重いリサーチや横断調査を Dynamic Workflows（またはsubagent）に投げて検証まで任せる運用を試す。artifactのMCP接続は自分の定期タスク成果を「開くたびに最新化される画面」に育てる余地がある。日本企業の事例では楽天が新機能実装を最大80%短縮、みずほFGはグループ約3万人がClaudeを業務利用と報告され、テスト生成70%・ドキュメント更新80%削減など定量効果も出ている。

注意点

Dynamic Workflows は並列サブエージェントを多用するためトークン支出が跳ねやすい——subagentは単一スレッド比で約7倍という指摘もあり、Analytics APIでの利用・コスト可視化とセットで運用したい。リサーチプレビュー段階なのでプラン制限にも注意。

ひとことまとめ

「既定モデルがOpus 4.8になり、計画→並列→検証→報告を1セッションで回す時代」。効果は大きいがコスト観測を同時に握るのが要。
