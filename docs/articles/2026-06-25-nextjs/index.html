react-three-fiber を Next.js App Router で — 宣言的3DとSSR回避（client island）の作法

WebGL（three.js）をReactの宣言的世界に持ち込むのが react-three-fiber（R3F）。<mesh> や <ambientLight> をJSXで書くだけで three.js のオブジェクトツリーへマップされる。注目は R3F v9 が React 19 に対応して安定したこと。App Router（15/16・React 19）と素直に噛み合い、ヒーロー演出や製品ビューアで実運用に乗せやすくなった。鍵は three.js を SSR させず client island として切り出す組み込み手順だ。

なぜ今これが話題か

命令的な new THREE.Mesh(...) の生成・破棄を手で管理するのは骨が折れる。react-three-fiber（R3F）は <mesh> や <ambientLight> をJSXで書くと、three.js のオブジェクトツリーへ Reactの reconciler がそのままマップしてくれる。コンポーネント分割・props・状態・Suspense といった既存のReact資産で3Dを組めるのが利点だ。

何が新しいか

転機は R3F v9 が React 19 対応で安定したこと。@react-three/fiber v9系は React 19.0〜19.2 に対応（v8→v9 は React 19対応のメジャーで、Props→CanvasProps など型整理を含む）。ヘルパ集 @react-three/drei は v10系（OrbitControls / Environment / useGLTF / Html）が対になる現行標準。メジャーを混在させない（v9 fiber に v9 drei を当てる取り違えが定番のハマり）。Next.js App Router（15/16・React 19）と素直に噛み合い、「Reactで3D」が実運用に乗せやすくなった。

仕組み・組み込みの要点

three.js は window/document と WebGL コンテキストを前提とするためSSRで死ぬ。基本は SSRさせずクライアント島に切り出す。ところが Server Component から直接 dynamic(import, {ssr:false}) を書くと「ssr: false is not allowed ...」エラーになるので、'use client' の薄いラッパに閉じ込めるのが現行のお作法だ。

// SceneClient.tsx
'use client';
import dynamic from 'next/dynamic';
const Scene = dynamic(() => import('./Scene'), {
  ssr: false,
  loading: () => <div style={{height:400}} />, // CLS回避
});
export default function SceneClient(){ return <Scene/>; }

アニメーションは Reactのstateで回さない。毎フレームは useFrame((_,delta)=>{ ref.current.rotation.y += delta }) でレンダーループ上から直接 mutate する（stateで回すと毎フレーム再レンダリングして破綻）。delta を掛けるとフレームレート非依存。静的シーンは frameloop="demand" で変化時のみ描画、dpr={[1,2]} でピクセル比上限2にしてRetinaのGPU焼けを防ぐ。

自分が試すなら

  • ページはRSCのまま、3D部分だけ SceneClient として島化し、親に明示的な高さを与えて CLS を防ぐ。
  • frameloop="demand" ＋ prefers-reduced-motion で自動回転を止め、OrbitControls の手動操作だけ残す。
  • 重いGLTFは useGLTF＋<Suspense>＋useGLTF.preload() で遅延・先読み。

注意点

R3F は独立した reconciler ルートを作るため、Canvas内から useRouter 等のNextコンテキストが見えないことがある。ナビゲーションはCanvasの外で行うか、コンテキストを橋渡しする。'use client' の付け忘れはフックが使えずビルドエラーになる。

ひとことまとめ

R3F v9＋drei v10 で「Reactで3D」が安定段階に。client island ＋ ssr:false ラッパ ＋ useFrame の3点を押さえれば、App Router でも LCP/CLS を守りつつ宣言的に3Dを載せられる。
