Claude CodeにArtifacts登場 — セッション成果のライブ共有と公式「Steering」ガイド

作業成果をライブなWebページとして共有できるArtifactsが追加。同時に公式ブログがCLAUDE.md・Skills・Hooks・Subagentsの使い分け決定版ガイドを公開し、「指示の置き場所」を再設計する日になった。

📅 2026年7月11日 ⏱ 約2分 🔗 出典 4 件

Claude Codeの公式changelogにArtifactsの追加が載った。セッションで作った成果物を、その場で更新されるライブなWebページとして共有できる機能で、Team/Enterprise向けには組織内限定共有とバージョン履歴がベータ提供される。「作業の途中経過をURLで見せる」がCLIだけで完結するようになる大きめの機能追加だ。あわせて/cdのディレクトリパス補完、コードベースから導出できる内容を削ってCLAUDE.mdのスリム化を提案する/doctorチェック、/commit-push-prが設定済みリモートへのpushを自動許可する改善も入った。

公式「Steering Claude Code」ガイドの要点

同じタイミングでAnthropic公式ブログが、Claude Codeを狙い通りに動かすための機構の使い分けを整理した。基本線は3つ。(1) デプロイ手順やリリースチェックリストのような手続き的な指示はCLAUDE.mdではなくSkillsへ置く。(2) 100%確実に守らせたいルールは指示ではなくHooksで決定論的に強制する。(3) CLAUDE.mdはビルドコマンドやアーキテクチャノートなど常時必要な情報だけに絞って簡潔に保つ。/doctorのスリム化提案は、この方針をツール側から後押しする機能と読める。

運用面では、Team/Enterprise管理者がリモートからのセッション閲覧・操作（ステアリング）の前に端末認証（Trusted Devices）を必須化できるようになり、バックグラウンドエージェントの自動アップグレードも入った。また報道ベースでは、週次利用上限が7月13日まで一時的に+50%されている（対Codexの利用促進策との見方）。VentureBeatによれば、Anthropic自身の新規プロダクションコードの80%はすでにClaudeが書いているという。

自分が試すなら、まず/doctorを走らせてCLAUDE.mdの削減提案を受け、手続き的な記述をSkillsへ移す棚卸しから。注意点は、Artifactsの組織内共有・バージョン履歴はベータかつTeam/Enterprise限定なこと、上限+50%は期間限定なこと。

ひとことまとめ: 成果物はArtifactsでライブ共有、指示は「常時=CLAUDE.md / 手続き=Skill / 強制=Hook」へ再配置する日。
