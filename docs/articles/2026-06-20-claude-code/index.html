Claude Code v2.1.183 — 自動モードが破壊的 git/terraform を既定でブロック、成果は Artifacts でライブ共有

毎朝のスケジュールタスクや CI で自動モードを回すなら今回は外せない。2026-06-19 公開の v2.1.183 は、明示指示が無い限り git reset --hard・git clean -fd・terraform destroy などローカル変更や本番を壊すコマンドをブロックする安全強化が中心。あわせて Team/Enterprise 向けに、セッションの成果を共有可能なライブ HTML ページ化する Artifacts も発表された。

📅 2026年6月20日 ⏱ 約3分 🔗 出典 3 件

なぜ今これが話題か

自動モード（auto mode）でエージェントに作業を任せるほど怖いのが、取り返しのつかない操作だ。git reset --hard でローカル変更が消え、terraform destroy で本番インフラが飛ぶ——人間なら一拍置く操作を、エージェントが「掃除のつもり」で実行してしまうリスクは現実的だった。2026-06-19 公開の v2.1.183 は、この安全性を一段引き上げる。

何が新しいか

ユーザーが明示的に作業の破棄を指示していない限り、ローカル変更を失う破壊的コマンドをブロックする。対象は git reset --hard・git checkout -- .・git clean -fd・git stash drop など。さらに、エージェント自身がそのセッションで作っていないコミットへの git commit --amend も止め、スタックを明示しない terraform destroy・pulumi destroy・cdk destroy も実行しない。加えて非推奨/自動更新モデル使用時に -p 出力で警告を出し、JetBrains 系 IDE（2026.1+）ターミナルのちらつきも同期出力で修正された。

成果は Artifacts でライブ共有

同時期に Anthropic は Team/Enterprise 向けに Claude Code の Artifacts を発表した。セッションの成果を、ライブで動く共有可能な HTML ページに変換できる。ライブコードや複数データソースを差し込み、共有 URL 上でインタラクティブに表示でき、PR ウォークスルー・ダッシュボード・リリースチェックリストなどに活用できる。バージョン履歴からいつでも復元可能だ（この読み物自体も同じ Artifacts の仕組みで動いている）。

仕組みのポイント

肝は「意図の明示を条件にした分岐」。ブロックは一律禁止ではなく、ユーザーが破棄を明示したかどうかで判断する。だから通常の編集フローは止めずに、事故だけを止められる。Hook と違ってモデル側のガードレールなので、追加設定なしで効くのも利点だ。

自分が試すなら

  • スケジュールタスクや CI を v2.1.183 以降へ更新し、破壊的操作の事故耐性を底上げする。
  • 意図的に reset したい時は「ローカル変更を破棄して」と明示してから実行する。
  • terraform/pulumi/cdk は対象スタックを必ず明示する運用に揃える。

注意点

ブロックは「破棄の明示が無いとき」の保険であって万能ではない。意図的な破棄は明示すれば従来どおり通るため、曖昧な指示文がかえって事故を呼ぶ。プロンプトでは「破棄する/しない」を一文で明確にするのが安全だ。Artifacts は現状 Team/Enterprise 向けの提供である点にも注意。

ひとことまとめ

v2.1.183 は派手さより事故防止のアップデート。自動化を回す人ほど恩恵が大きい。破棄系コマンドは「明示したときだけ通る」と覚え、まずは実行環境のバージョンを上げておきたい。
