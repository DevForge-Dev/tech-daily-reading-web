AIエージェント駆動の開発ループ — Agentic Hot Reload と GenUI/A2UI、コア構造も再編（Flutter 3.44 / Dart 3.12）

Google I/O 2026 で示された方向は「AIエージェントが開発ループそのものに入る」こと。安定版 Flutter 3.44.0 / Dart 3.12 を土台に、コーディングエージェントが起動中アプリを自動検出して UI 変更を即ホットリロードする Agentic Hot Reload、AI が実 Widget で画面を動的生成する GenUI/A2UI が目玉だ。同時に Material/Cupertino のコア凍結や Android 10+ の Pure Impeller 化など構造面も動いた。

なぜ今これが話題か

2026 の Flutter は「AIエージェントを開発ループの内側に取り込む」方向にはっきり舵を切った。安定版は Flutter 3.44.0 / Dart 3.12（Google I/O 2026, 2026-05-18）で、現行は 3.44 系。年4回の安定リリースを継続しつつ、編集ループとUI生成のあり方を更新している。

何が新しいか

目玉は Agentic Hot Reload。コーディングエージェントが起動中アプリを自動検出・接続し、UI を変更すると Dart & Flutter MCP Server 経由で自動ホットリロードする。人がリロードを押す手間が消え、AI編集ループが標準化される。さらに GenUI は固定ウィジェットツリーの制約を解除し、AIエージェントが実 Flutter ウィジェットをその場で構成してユーザー意図に応じた画面を動的生成する（A2UI プロトコルで接続）。言語面の Dart 3.12 は dot shorthands（.value 省略記法）と private named parameters（先頭 _ を自動で外して公開名に）を正式サポートした。

コア構造の再編

地味だが重要なのが土台の整理だ。Material / Cupertino をコアで凍結し独立パッケージ（material_ui / cupertino_ui）へ移行、SwiftPM が iOS/macOS の既定依存管理に正式昇格、Android 10+ で Skia を撤去し Pure Impeller(Vulkan) が既定になった。Impeller はシェーダーをビルド時に AOT 事前コンパイルするためシェーダージャンクが減り、フレーム予算（60fps=16ms / 120fps=11ms）を守りやすい。

自分が試すなら

  • 対応エディタで Agentic Hot Reload を有効にし、AI編集→自動反映ループを小さなUI変更で体感する。
  • Dart 3.12 の dot shorthands / private named parameters を既存コードの一部に適用し読みやすさを比較。
  • 独立パッケージ化に備え、material_ui 等への import 整理を段階的に検討する。

注意点

GenUI/A2UI は登場直後で本番採用は時期尚早。Impeller 化で Rive 等の描画/性能に差が出ることがあり、差があれば --no-enable-impeller で Skia と比較し切り分ける。計測は必ず profile モードで。進化が速い領域なので、採用判断前に最新の digests/ の日付を確認したい。

ひとことまとめ

「エージェントが編集ループに入り、AIが画面を生成し、コアは独立パッケージへ」。まずは Agentic Hot Reload と Dart 3.12 の新記法から触れておくと流れに乗れる。
