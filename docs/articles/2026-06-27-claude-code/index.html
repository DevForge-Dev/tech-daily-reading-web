Claude Code

Claude Code 週次更新（v2.1.166〜176）— /cd で作業ディレクトリを途中変更、サブエージェント入れ子化、--safe-mode と fallbackModel、sandbox.credentials で秘密情報を遮断

エージェントに長く任せるほど、作業ディレクトリの固定・秘密情報の漏れ・モデル障害時の停止といった地味な引っかかりが効いてくる。6月第2週の週次リリース（v2.1.166〜176）と公式changelogは、/cd の途中ディレクトリ変更、サブエージェントの入れ子生成（最大5階層）、--safe-mode、fallbackModel、sandbox.credentials を一気に投入した。派手さより「止まりにくく・壊れにくく・漏れにくく」を底上げする更新だ。

📅 2026年6月27日 ⏱ 約3分 🔗 出典 3 件

なぜ今これが話題か

エージェントに長時間任せるほど、地味な運用の引っかかりが効いてくる。作業ディレクトリが固定で別プロジェクトに移れない、サンドボックス内のコマンドが認証情報を読めてしまう、モデル側が一時的に落ちるとセッションごと止まる——こうした「事故と中断」を減らす運用系の改善が、6月第2週の週次リリース（v2.1.166〜176）と公式changelogでまとめて入った。

何が新しいか

目玉は5つ。/cd は会話の途中で作業ディレクトリを移動でき、プロンプトキャッシュを再構築せずに別ディレクトリへ切り替えられる。サブエージェントの入れ子生成が可能になり、バックグラウンドの連鎖は最大5階層まで。--safe-mode は全カスタマイズ（MCP・hooks・skills 等）を無効化して起動し、不具合の切り分けを容易にする。fallbackModel は最大3モデルを順に試すフォールバック構成に対応。安全面では sandbox.credentials がサンドボックス実行中のコマンドから認証情報ファイルや秘密の環境変数の読取を遮断する。あわせて、大きなツリーで denyRead/allowRead のグロブが Bash ツール説明文を肥大化させ Linux でセッションが使えなくなる不具合、スピナーが「running tool」で固着する問題を修正し、ミッドストリーム切断時には生エラーを出さず部分応答を保持するようになった。なお v2.1.161 単体でも22項目の変更（権限挙動・MCP レジリエンス・OAuth・省メモリ化）が入っている。

fallbackModel の動き

fallbackModel は指定したモデルを左から順に試し、過負荷やエラーで失敗したら次へ自動で切り替える。長時間の自動実行中に主モデルが一時的に落ちても、セッションを止めずに継続できる。

自分が試すなら

  • モノレポで /cd を使い、パッケージ間をキャッシュ維持のまま移動できるか確認する。
  • 設定に fallbackModel を入れ、主モデル過負荷時に継続するか試す。
  • 機密を扱う環境で sandbox.credentials を有効化し、秘密が読めないことを検証する。
  • 不調時はまず --safe-mode で起動し、カスタマイズ起因かを切り分ける。

注意点

サブエージェントの入れ子は強力だが、5階層の連鎖はトークン消費とコストが膨らみやすい。Claude Code の年換算売上は25億ドル規模に達し利用が急増する一方、月次支出が跳ねる事例も報じられており、fallback や入れ子を使う前に予算・使用量の可視化を整えておきたい。--safe-mode は切り分け用で、常用すると必要な拡張まで無効化される点に注意。

ひとことまとめ

今回は「止まりにくく・壊れにくく・漏れにくく」を一段進めた運用アップデート。/cd と入れ子サブエージェントで作業の自由度を上げ、fallbackModel と部分応答保持で中断を減らし、sandbox.credentials で足元の安全を固める三方向の改善だ。
