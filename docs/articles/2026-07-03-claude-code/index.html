バックグラウンドエージェントが自走する日 — 通知フック・自動ドラフトPR・Exploreのモデル継承

7月3日のダイジェストは「自走する自動化基盤」への一歩。Claude in Chrome が一般提供(GA)になり、バックグラウンドのエージェントが入力待ち/完了時に通知フック(agent_needs_input / agent_completed)を発火、worktree での作業を終えると確認待ちで止まらずコミット・プッシュしてドラフトPRまで開く。内蔵 Explore は haiku ではなくメインモデル(opus上限)を継承し、/dataviz スキルも追加。裏で Sonnet 5 の新トークナイザ(約30%増)というコストの注意点も。

📅 2026年7月3日 ⏱ 約3分 🔗 出典 3 件

なぜ今これが話題か

この半年、Claude Code は「対話しながら書く相棒」から「無人で長く回す自動化基盤」へ重心を移してきた。7月3日の公式CHANGELOGはその方向をさらに一段進める。Claude in Chrome が一般提供(GA)になり、そしてバックグラウンドのエージェントが人の確認を待たずに作業を完遂できるようになった。あわせて組織デフォルトモデル・読みやすいセッション名・クリック可能な添付・agentsビュー改善といった、毎日の運用の手触りを底上げする更新群も入っている。

何が新しいか

核心は3つ。第一に、バックグラウンドセッションが状態遷移でNotificationフックを発火するようになった。入力が要るときは agent_needs_input、完了時は agent_completed が飛ぶので、通知やスクリプトで拾える。第二に、worktree でコード作業を終えたバックグラウンドエージェントは確認待ちで止まらず、コミット・プッシュしてドラフトPRを開く。第三に、内蔵の Explore エージェントが haiku ではなくメインセッションのモデル(opus上限)を継承するようになり、調査の質が上がった。加えて、チャートやダッシュボード設計を助ける /dataviz スキルとカラーパレット検証が追加された。

仕組みのポイント

ポイントは「エージェントが自分の状態を外へ知らせ、区切りで成果を確定する」ループができたこと。agent_needs_input と agent_completed は Notification フックの一種で、人はそれを購読すればよく、常時監視から解放される。完了時の自動 push→ドラフトPR は、worktree による隔離が前提なので main を直接汚さない。Explore のモデル継承は、調査サブエージェントが安価な haiku 固定ではなく親と同じ賢さで動くという変更で、設計・横断調査の精度に効く。

自分が試すなら

  • 毎朝の自動処理を worktree ＋バックグラウンドで走らせ、agent_completed 通知でドラフトPRの有無だけ確認する運用に寄せる。
  • 横断調査タスクは Explore に委譲し、モデル継承で精度が上がったか before/after を比べる。
  • ダッシュボード系の可視化は /dataviz に任せてカラーパレット検証まで通す。

注意点

裏で走る Sonnet 5 の新トークナイザは同じ入力でも約30%多くトークンを生成する。自走で回数が増えるほど請求は膨らむので、カテゴリ別 /usage で自動化枠の消費を棚卸ししたい。加えて Sonnet 5 は適応的思考が既定オンで、手動の拡張思考や非既定サンプリングは400エラーになる点も設定見直しが要る。自動PRは便利だが、レビュー無しでのマージは避け、ドラフト止まりを既定に。

ひとことまとめ

通知フック＋自動ドラフトPR＋Exploreのモデル継承で、Claude Code は「人を待たずに成果を残す」自走基盤へ近づいた。強力さの裏でトークンコストは正直に増えるので、監視と worktree 隔離が主導権を握る。
