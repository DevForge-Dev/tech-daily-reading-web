/doctor が /checkup に刷新 — 診断だけから「修正＋コンテキスト掃除」まで。Autoモードの安全網も強化

v2.1.202〜206系で /doctor が読み取り専用レポートから、問題を診断し修正まで行う『フルセットアップ点検』に（エイリアス /checkup）。未使用Skills/MCP・CLAUDE.md重複を掃除。Autoモードは記録改ざんブロック・未解決変数のrm -rf確認、自動更新はピークメモリ約400MB削減。Bedrock/Vertex/FoundryでAutoモードが既定（Opus 4.8）。

📅 2026年7月16日 ⏱ 約2分 🔗 出典 5 件

今週のClaude Codeは派手な新機能より運用の底上げが主役。目玉は /doctor の刷新で、これまで「レポートを出すだけ」だった診断コマンドが、問題を見つけたうえで修正まで実行する『フルセットアップ点検』に作り替えられた（エイリアスは /checkup）。

何が新しいか

v2.1.202〜206系で、/checkup（=/doctor）はインストールの健全性に加え、未使用のSkills・MCPサーバー・プラグインがコンテキスト予算をどれだけ食っているかを可視化し、複数の CLAUDE.md の重複排除・削減提案、遅い hooks の検出までやる。変更前に必ず所見を提示して確認を求める安全設計だ。あわせて Auto モードの安全性が強化され、セッション記録（トランスクリプト）の改ざんをブロック、文脈から解決できない変数に対する rm -rf は実行前に確認を挟むようになった。自動更新はバイナリをメモリに溜めずディスクへ直接ストリームする方式に変え、アップデータのピークメモリを約400MB削減している。

基盤とエコシステム

Amazon Bedrock・Google Vertex AI・Microsoft Foundry の各基盤で Auto モードが既定利用可能になり、これらの環境では Opus 4.8 がデフォルトモデルに。Artifacts も Pro/Max で解放され、/review コマンド体系の刷新も進んだ。

実務での使いどころ

毎日多数の定期タスクを走らせている分だけコンテキストのムダは積もりやすい。まず /checkup を週1で回し、使っていない Skills/MCP と肥大した CLAUDE.md を掃除してトークンを絞るのが効きそうだ。実際、Microsoft Research の査読論文（7/1公開）では Claude Code 採用者のマージ済みPRが約24%増（効果は4か月持続）と出た一方、トークン費用は年数百万ドル規模に達しコスト統制が課題になったと報告された。アルバータ州政府の事例では約50体のエージェント並列で4.66億行を20時間（従来約6年半相当）で走査——規模を出すほどコスト観測性が重要になる。

注意点

/checkup の削減提案は必ず確認してから適用する（CLAUDE.md の重要な規約を誤って削らないよう）。rm -rf の確認は「変数が未解決のとき」限定なので、解決済みパスの破壊的コマンドは従来通り自分でガードする。

ひとことまとめ

今週は「散らかったセットアップを畳んでコストを締める」週。/checkup を定期化し、Auto モードの安全ネットに乗るのが吉。
