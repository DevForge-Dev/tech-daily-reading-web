7/15の話題リポジトリ — CPUで動く音声クローンTTS・NASA製Wasm・自然言語でWeb操作

📅 2026年7月15日 ⏱ 約2分 🔗 出典 3 件

この日の急上昇3本を貫くのは「重い前提を外して軽くする」という方向性だ。GPUなしで動く音声合成、宇宙機のような極限環境で決定的に動くWasm、拡張機能もヘッドレスブラウザも要らないWebエージェント——いずれも導入の敷居を下げることで広がっている。

kyutai-labs/pocket-tts（Python・⭐約6.7k）

解決したい課題：高品質な音声合成はGPUや大きなモデルを前提にしがちで、オンデバイス／CPUだけで声のクローンまでやるのは難しかった。提供する機能：約1億パラメータと軽量ながらCPUだけで動き、音声クローンに対応するTTS。macOSはCore ML対応でオンデバイス完結する。Moshiで知られるKyutai Labs発で、公開3日で1kスター→現在約6.7kと急伸。オンデバイス音声生成の新星として拡散している。

nasa/spacewasm（Rust・⭐約749）

解決したい課題：宇宙機のような資源制約環境では、実行時のメモリ使用や挙動が決定的であることが必須だが、汎用のWasmランタイムはそこまで保証しない。提供する機能：Wasm 1.0に追加制約を課してメモリ使用と実行を決定的にし、バイナリを単一パスでストリーミング復号・実行する、NASA/JPL製の飛行対応インタプリタ。火星ローバーのコマンドシーケンス実行という具体的文脈が驚きを呼び、OSS公開自体が話題に。

alibaba/page-agent（TypeScript・⭐約22.7k）

解決したい課題：既存のWeb自動化は拡張機能やPython、ヘッドレスブラウザなどの重いインフラを要し、本番ページに載せにくかった。提供する機能：1本のスクリプトを読み込むだけでDOMを読み、自然言語コマンドでUIを操作できるin-page GUIエージェント。モデル非依存・ゼロインフラでローカルモデルも使える。ピーク時1日+1,100スター超で累計2.2万超。

まとめ

自分が試すなら、まずpage-agentを社内の管理画面に1行差してみたい（インフラ不要で撤退も容易）。pocket-ttsはオンデバイス通知の読み上げに、spacewasmは決定的サンドボックスの発想の参考に。共通するのは「前提を外して手元で動かせる」ことの強さだ。
