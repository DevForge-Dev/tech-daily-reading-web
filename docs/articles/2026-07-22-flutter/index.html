AI がコア開発ループへ——Agentic Hot Reload と Rive×GenUI、Riverpod 3.3.1 の DI overrides

Flutter 3.44 / Dart 3.12 のテーマ built by everyone, for everyone の下、Agentic Hot Reload・Agent Skills・GenUI・A2UI・Genkit Dart が「書く・デバッグ・出荷」の各段へ統合。描画は Pure Impeller Vulkan がモダン Android の既定に。状態管理は Riverpod 3.3.1 のコード生成＋DI overrides テストが主役で、規制/大規模は Bloc 9.1.1。GPU 目標は 4ms 未満=60fps / 8.3ms 未満=120fps。

この日のFlutterはAIがコア開発ループへ統合されたことが主役だ。Flutter 3.44 / Dart 3.12のテーマ"Everywhere, everyday, built by everyone, for everyone"の下、Agentic Hot Reload・Agent Skills・GenUI・A2UIプロトコル・Genkit Dartが「書く・デバッグ・出荷」の各段に組み込まれた。安定版はFlutter 3.44.0/Dart 3.12で、次の機能リリースは8月予定（年4回：2/5/8/11月）。

何が新しいか

Agentic Hot Reloadはコーディングエージェントが起動中アプリを自動検出・接続し、UI変更後に自動でhot reloadする（Dart & Flutter MCP Server経由・手動設定不要）。描画はPure Impeller VulkanがモダンAndroidの既定になり、初回シェーダーコンパイルのジャンクを排除。到達領域も広がり、2026 Toyota RAV4のマルチメディアがFlutter製、大画面向けにLG webOS SDKが近日提供。Dart 3.12はprivate named parameters・primary constructors（experimental）・dot shorthandsを追加した。

Rive×GenUI とレンダラ選択

アニメの深掘りはA2UIで動的生成されるUIにRiveを組み込むパターン。Riveのstate machineでアニメロジックを描画から分離し、デザイナーはFlutterコードに触れずloading stateを反復（実装側はboolean/number/triggerを繋ぐだけ）、eventsでハプティクス/サウンド/ナビゲーションを正確トリガーする。複雑グラフィックや多数同時表示はFactory.rive（Vector Featheringはこれのみ）、.rivは事前ロード＋RepaintBoundaryが定石だ。

状態管理とパフォーマンス

状態管理はRiverpod 3.3.1のコード生成とDI overridesテストが主役。@riverpod＋build_runnerでcompile-safeなproviderを生成し、BuildContext不要でテスト可能。テストの勝ち筋はDI overrides——ProviderContainer(overrides:)でツリー全体をモックせず任意providerだけ差し替える（addTearDown(container.dispose)）。選定は新規=Riverpod 3.3.1＋コード生成、規制/監査/大規模=Bloc 9.1.1。描画側はImpellerがshader jankをほぼ排除し60/120fpsをネイティブ級で実現、DevToolsのflame chartでジャンク関数を特定する。

自分が試すなら

まずAgentic Hot Reloadを試し、エージェント駆動のUI反復を体感したい。新規はRiverpod 3.3.1のコード生成でボイラープレートを削り、テストはDI overridesで書く。パフォーマンスはprofileモードでtimeline traceを取り、flame chartで犯人関数を突き止めてからconst徹底・最小スライスwatch・RepaintBoundaryで削る。

注意点

GPUスレッド時間の目標は<4ms=60fps／<8.3ms=120fps、計測は必ずprofileモード（debugは当てにならない）。カスタムfragment shaderは重い分岐を避けmix/step/clampへ。Material/Cupertinoは3.44でコア凍結され将来独立バージョニングへ移る点も把握しておく。

ひとことまとめ
