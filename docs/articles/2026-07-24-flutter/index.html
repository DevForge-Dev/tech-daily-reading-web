3.44 安定運用フェーズの実利——Riverpod 3.0 の4新機能とフレーム予算で足元を固める

いまは8月の機能リリースへの助走期。安定版は Flutter 3.44.0 / Dart 3.12（3.44.7 まで hotfix）で、次は2026年8月予定。派手な新機能より、Riverpod 3.0 の stale-while-revalidate／自動リトライ／Mutations／unified Ref と、不要リビルド撲滅＋フレーム予算（60fps=16ms）の地固めが今日の実利だ。

いまは8月の機能リリースへの助走。安定版は Flutter 3.44.0 / Dart 3.12（Google I/O 2026・以後 3.44.7 まで patch hotfix）で、次の機能リリースは2026年8月予定（年4回 2/5/8/11月）。iOS 26 / Xcode 26 / macOS 26 を正式サポートし、描画は Impeller が Android 既定、依存は SwiftPM が iOS/macOS 既定。派手さより「3.44 系で何を固めるか」の局面だ。

状態管理——Riverpod 3.0 の4つ

新規プロジェクトの de facto 標準になった Riverpod 3.0 の要点は4つ。stale-while-revalidate のオフライン永続化、失敗プロバイダの自動リトライ（指数バックオフ）、書き込みの中間状態を扱う Mutations API、Ref 型を統一する unified Ref。選ぶ理由は compile-time safety・少ないボイラープレート・BuildContext 不要・provider override でテスト容易。監査証跡が要る fintech/healthcare や大規模の一貫パターンは Bloc 9.0 が理想で、現行版は Riverpod 3.3.x / bloc 9.2.x。移行は screen-by-screen で共存できる。

パフォーマンス——不要リビルドとフレーム予算

最も多い性能問題は不要な再ビルド。プロパティ不変なら const でリビルドを完全スキップできる。計測は DevTools（Performance/Memory/Network）か、実行中に P キーで UI/GPU スレッドのフレーム時間をリアルタイム確認（profile モード必須）。フラッグシップは 90/120Hz が主流で、フレーム予算は下図の通り。予算を超えるとユーザーがジャンクを知覚する。

実務・まとめ

再ビルド削減は const 徹底・build 分割・最小スライス watch・RepaintBoundary の戦略配置。リストは ListView.builder＋固定 itemExtent、画像は cached_network_image＋width/height 明示。アニメは「まず implicit、連鎖は flutter_animate、精密は explicit（dispose 必須）、重い演出は Rive/Lottie」。Rive は build() 内でロードせず initState/FutureBuilder で事前ロードし、同一アニメは RiveFile インスタンスを再利用する。まとめると、3.44 は新機能待ちよりRiverpod 3.0＋フレーム予算＋const で足元を固める回だ。
