7/12の話題リポジトリ — 744B MoEを25GBで動かす・PostgresをRustで全面リライト・AI就活エージェント

「データセンター級を個人の手に」「不可能とされた書き直しを現実に」「エージェントを日常の実務に」——AIが前提を書き換える3本。

📅 2026年7月12日 ⏱ 約3分 🔗 出典 3 件

今日の3本に共通するのは、これまで「無理」で片付けられていた前提をAIとエンジニアリングの工夫でひっくり返している点だ。巨大モデルのローカル実行、レガシー資産の全面リライト、就活という個人タスクのエージェント化——スケールも言語もバラバラだが、どれも既存の限界線を一段ずらしている。

JustVugg/colibri（C・⭐3.5k、本日+約1.4k）

解決したい課題: 744BパラメータのMoEモデルGLM-5.2のような巨大モデルは、本来データセンター級のメモリを要求し、個人がローカルで動かす道は事実上閉ざされていた。ローカルLLM需要が高まる中、この「サイズの壁」が最大の不便だった。

提供する機能: 純C・依存ゼロの超軽量推論エンジン。MoEのエキスパート群をディスクから必要な分だけストリーミングする設計で、25GB RAMのコンシューマ機で744Bモデルを実行できる。Apache 2.0で導入しやすい。本日のTrendshift日次1位で、「小さなエンジンで巨大モデル」という発想の逆転が刺さった格好だ。

malisper/pgrust（Rust・⭐1.5k、本日+約670）

解決したい課題: PostgreSQLは数十年分のC資産であり、メモリ安全性や保守性のための全面リライトは工数的に「経済的に不可能」とされてきた。レガシーの安全なモダン化は業界全体の宿題だった。

提供する機能: PostgreSQLをRustで書き直し、本家のリグレッションテストを100%通過したデータベース実装（AGPLv3）。Hacker Newsでは「AIがソフトウェア書き直しの経済性を変える」文脈で議論されており、到達点そのものがAI時代の大規模リライトの実証例として注目されている。

MadsLorentzen/ai-job-search（TypeScript・⭐20.7k、本日+約1.5k）

解決したい課題: 就活は求人の適合判断・CV調整・カバーレター作成・面接準備と反復作業が多く、汎用チャットでは自分のプロフィールという文脈を保った伴走が難しい。

提供する機能: Claude Code上に構築された就活フレームワーク。フォークして自分のプロフィールを書き込むだけで、求人評価からCV調整・カバーレター生成・面接準備まで一貫支援する。MIT・フォーク6.2kと「実際に使われる」エージェントになっており、7/7のTrending 1位以降も勢いが続く。

まとめ: 自分が試すならまずcolibri。エキスパートストリーミングの実装は、MoEをエッジで動かす発想の教材として一級品だ。ai-job-searchは「SKILL.md群でエージェントを組む」設計例としてフォークして中身を読むだけでも価値がある。
