v2.1.213〜214 と Fable 5 標準搭載——会話を畳む EndConversation、長時間タスクの進捗ハートビート

v2.1.213〜214 で会話を明示終了する EndConversation ツールと、長時間ツール呼び出しの進捗ハートビート表示が追加。カスタムエージェント行にモデル名・effort 表示、メモリに ISO タイムスタンプも。並行して 7/20 から最上位モデル Claude Fable 5 が Max・Team Premium に上限50%枠で標準搭載され、Claude Code からも使えるようになった。

📅 2026年7月21日 ⏱ 約3分 🔗 出典 3 件

この日のClaude Codeはエージェント運用の「見える化」と最上位モデルの解禁が主役だ。v2.1.213〜214で長時間・自律実行まわりの体験が整い、同時にClaude Fable 5がサブスクの標準枠に載った。号令一下で数十分走るタスクを回す時代に、進捗の可視化とモデル選択肢の拡大が噛み合う。

何が新しいか

v2.1.213〜214の目玉はEndConversationツールと進捗ハートビート。前者はエージェントが「やることは終わった」と判断したら会話を明示的に畳めるツールで、無駄なターン継続を止める。後者は長時間実行されるツール呼び出しに対して定期的な進捗表示を出し、「固まったのか動いているのか」の不安を解消する。加えてカスタムエージェント行にモデル名とeffort（推論の強度）を表示するオプション、メモリファイルへのISO形式タイムスタンプ付与、OpenTelemetryログへの message.uuid / client_request_id 追加が入った。Linuxで pkill -f がセッションを巻き添えで落とす問題、Windowsプロキシ配下のストリーミング失敗、スケジュールタスクのプロンプト却下なども修正されている。

Fable 5 が標準枠に

7/20より最上位モデルClaude Fable 5がMaxとTeam Premiumに通常上限の50%枠で標準搭載された。Pro/Team Standardは従量課金（usage credits）でのアクセス継続に加え一度きりの$100クレジットが付く。Fableは需要予測が難しく段階的に枠を広げてきた経緯があり、上限50%という制約付きの解禁だ。Claude CodeからもFable 5を選べるため、複雑・長時間の自律タスクに強いモデルを土台に据えられる。

自分が試すなら

まず進捗ハートビートは黙っていても効くので、長時間MCP呼び出しを含む定期タスクの体感が改善するはず。EndConversationはエージェント定義で「完了条件を満たしたら畳む」運用に組み込むと、暴走ターンのコストを抑えられる。Fable 5はMax枠なら重い設計・リファクタの回に指定して費用対効果を見極めたい。

注意点

Fable 5は50%枠ゆえヘビーに回すとすぐ上限に触れる。The Decoderは「Pro/Team Standardが実質API価格へ誘導されている」と批判的に指摘しており、$100クレジットはあれど常用のコスト構造は変わりうる。モデル名・effort表示はデフォルト非表示のオプション扱いな点も把握しておく。

ひとことまとめ

「畳む・見せる・強いモデルを載せる」回。EndConversationと進捗ハートビートで長時間運用が締まり、Fable 5解禁で選択肢が増える——ただしコスト管理は別腹。
