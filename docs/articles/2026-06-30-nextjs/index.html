ツールチップに JS ライブラリはもう要らない — CSS Anchor Positioning が Baseline 2026 到達、App Router では RSC のまま動く

Floating UI / Popper.js が担ってきた『要素を別要素に追従させ、画面端で回り込ませる』処理が、2026年初頭に主要3ブラウザ出揃いで純CSS化した。position-anchor / anchor() / position-area / position-try-fallbacks。Popover API と組めば 'use client' もハイドレーションも不要で、INPとバンドルに効く。

📅 2026年6月30日 ⏱ 約4分 🔗 出典 3 件

なぜ今これが話題か

ツールチップ・ポップオーバー・ドロップダウンを「トリガーに追従させ、画面端では自動で反対側へ回り込ませる」処理は、長らく Floating UI / Popper.js など JS ライブラリの仕事だった。スクロールやリサイズのたびに getBoundingClientRect() を読んで top/left を書き戻す——典型的なレイアウトスラッシング源だ。CSS Anchor Positioning はこれをブラウザネイティブのCSSだけで完結させる仕様で、Firefox 147（2026年1月13日）でデフォルト有効化され主要3ブラウザが揃い、Baseline 2026 に到達した。「ツールチップにJSライブラリは要らない」と言える段階だ。

何が新しいか

中核は3点セット。アンカー側に anchor-name: --tip;、配置側（position: absolute/fixed 必須）に position-anchor: --tip;、そして配置の指定だ。配置は position-area でアンカー中心の3×3グリッドにキーワードで置く（bottom center 等）のが最も手軽。anchor() 関数で辺を参照したり、anchor-size(width) でトリガー幅にドロップダウンを揃えるのも定番だ。画面端では position-try-fallbacks: flip-block, flip-inline; がはみ出しを検知して収まる位置へ反転（Floating UIの flip() 相当）、position-visibility: anchors-visible; はアンカーが画面外に出たらポップオーバーも隠す。

仕組みのポイント（App Router）

App Router 的に効くのは 純CSSで完結＝'use client' 不要、Server Component のまま使えること。さらに Popover API（popover / popovertarget）と組めば、開閉・Escクローズ・外側クリックで閉じる・top-layer重なり（z-index地獄回避）まで宣言的HTMLで賄え、ハイドレーションもクライアントJSもゼロでリッチなオーバーレイが組める。INP とバンドルサイズに直接効く。

自分が試すなら

  • 軽量な自前ツールチップ／メニューを、まず position-area + Popover API で置き換える（shadcn/Radixは当面Floating UIのまま）。
  • 未対応ブラウザ向けに @supports (anchor-name: --x) で素の配置をフォールバックにして段階導入する。

注意点

リスト内で同じ anchor-name: --tip を量産すると、各 position-anchor が DOM上「最後の」アンカーに解決され全部同じ場所に出る。useId() で一意な dashed-ident を生成して回避する（または anchor-scope）。Chrome 125→129で inset-area→position-area 等の改名があり、古い記事のコピペは無言で効かない。ホバー型ツールチップの完全宣言化はまだ Chrome 限定なので注意。

ひとことまとめ

anchor-name＋position-area＋position-try-fallbacks の3点で、ツールチップ配置はJS無し・RSCのままになった。Popover API と組んで、軽いオーバーレイから置き換えるのが現実的な第一歩。
