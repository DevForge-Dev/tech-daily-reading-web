なぜ今これが話題か

エージェントに長時間の作業や複数プロジェクトの横断を任せるほど、「セッションを切らずにどこまで自律で走らせられるか」が効いてくる。2026年6月のClaude Codeアップデートは、まさにこの自律実行まわりをまとめて強化した。

何が新しいか

目玉は新コマンド /cd。プロンプトキャッシュを壊さずに作業ディレクトリを移動でき、移動先プロジェクトの保存領域を使うため --resume / --continue もそこで見つかる。Agentツールはサブエージェントが自身の子サブエージェントを起動できるようになり、バックグラウンドの連鎖は最大5段ネストでキャップされる。さらに最大3モデルを順に試す fallbackModel、壊れた設定を切り分ける safe mode、セッション終了後に走る post-session hook が加わった。運用系では /usage がスキル・サブエージェント・プラグイン・MCP別に消費を内訳表示し、/code-review が正確性バグを検出・報告、/plugin list がメニューを開かずインストール済みプラグインを一覧する。

仕組みのポイント

/cd の肝は「キャッシュを壊さない」こと。通常ディレクトリを変えるとプロンプトキャッシュが無効化されコスト/レイテンシが悪化するが、/cd はキャッシュを保ったまま文脈を移す。ネストするサブエージェントは、親が全体を統括し子へ分割タスクを投げる構造を最大5段まで安全弁付きで許可し、暴走的な無限ネストを防ぐ。fallbackModel は1つ目が混雑/失敗しても2つ目→3つ目へ自動で切り替え、長時間バッチの途中停止を減らす。

自分が試すなら

  • モノレポやマルチプロジェクト作業で /cd を使い、セッションとキャッシュを維持したまま行き来する。
  • 大きな調査タスクを親→複数の子エージェントへ分解し、5段以内で設計する。
  • 夜間バッチに fallbackModel を設定し、可用性低下時の自動フォールバックを用意する。

注意点

ネストは便利だが深いほどデバッグとコスト把握が難しくなる。5段は上限であって推奨ではなく、まずは2〜3段で十分なことが多い。fallbackModel は品質の異なるモデルへ落ちる可能性があるため、出力品質が重要なタスクではフォールバック先も吟味したい。

ひとことまとめ

今回は「自律で長く走らせる」ための足回り強化。まずは /cd と fallbackModel から入れると、日々の自動化が一段安定する。
