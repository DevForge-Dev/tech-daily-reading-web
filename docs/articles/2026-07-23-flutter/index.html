Flutter 3.44 実測——Impeller が Android 既定に昇格、mid-range で描画フレーム30–40%短縮

2年のプレビューを経て Impeller が Android の既定レンダラに昇格。コミュニティの実機テストで mid-range Android の描画フレーム時間が30–40%短縮した。安定版は Flutter 3.44.0 / Dart 3.12（Google I/O 2026）。Impeller は全シェーダーをビルド時(AOT)にコンパイルし、iOS=Metal / Android=Vulkan で GPU に直結、初回アニメの shader jank をゼロにする。

📅 2026年7月23日 ⏱ 約3分 🔗 出典 3 件

この日のFlutterは3.44 の実測レビューが主役。長年の宿題だった「初回アニメのカクつき（shader jank）」に、レンダラ刷新でまとまった答えが出た回だ。

何が新しいか

2年のプレビューを経て、Impeller が Android の既定レンダラに昇格した。GitHub のコミュニティ実機テストでは mid-range Android の描画フレーム時間が30–40%短縮。安定版は Flutter 3.44.0 / Dart 3.12（Google I/O 2026・2026-05-18、リリースは年4回で次は8月予定）。DevTools は WASM を既定にして起動・スクロール・Timeline が高速化、Widget Preview は Dart Analysis Server ベースの検出書き直しで flutter tool のメモリを最大50%削減。Dart 3.12 は private named parameters・primary constructors(experimental)・dot shorthands を追加した。

仕組み——AOTシェーダーコンパイル

従来はシェーダーを実行時にコンパイルするため、初回描画で数フレーム分の停止（jank）が起きていた。Impeller は全シェーダーをビルド時(AOT)にバイナリへ事前コンパイルし、iOS=Metal / Android=Vulkan で GPU に直結。アプリ起動前に重処理を終えるので、初回アニメの shader jank が原理的に消える。もし描画/性能差が出たら公式トラブルシュートに従い --no-enable-impeller で切り分けられる。

実務での使いどころ・注意点

状態管理は2026も三強：新規の既定は Riverpod 3.0（compile-time safety・組込みオフライン永続化・最小ボイラープレート）、監査/規制/大規模は Bloc 9.0（event traceability）。計測は必ず profile モードで、フレーム予算（60fps=16ms/90fps=11ms/120fps=8.3ms）を基準に。再ビルド削減は const 徹底・build 分割・最小スライス watch・RepaintBoundary、画像は cached_network_image＋width/height 明示が定石。Impeller は Android 10+ が必須なので、旧端末サポートが要るプロジェクトは切替の影響を確認する。

ひとことまとめ

「Impeller が Android 既定へ→フレーム30–40%短縮、shader jank は AOT で原理的に解消」。2026の安定スタックは Impeller ＋ Material 3 ＋ 堅い状態管理（Riverpod/Bloc）だ。
