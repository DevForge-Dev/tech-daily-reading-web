Claude Code

既定は「手動で確認」へ — v2.1.201までの安定化とChrome GA

7/5は派手な新機能より“既定の見直し”。権限モードの表記が「Manual」になり、AskUserQuestionは自動継続をやめた。背景ではv2.1.201までの安定化、Claude in ChromeのGA、/dataviz追加。拡張はhook/skill/subagentの使い分けが定説化し、Anthropicは新規本番コードの約8割をClaudeが書くと明かした。

📅 2026年7月5日 ⏱ 約2分 🔗 出典 4 件

なぜ今これが話題か

ここ数日のClaude Codeは、新機能の追加より「既定でどう振る舞うか」の作り直しが続いている。7/5は権限まわりの既定が安全側へ寄り、背景の安定化とツール連携のGAが重なった節目だ。

何が新しいか

まず権限。CLI/VS Code/JetBrainsで「default」表記が「Manual」に変わり、--permission-mode manualとdefaultMode: manualがdefaultと併せて受理される。加えてAskUserQuestionダイアログは既定で自動継続しなくなった（/configのアイドルタイムアウトでオプトイン復帰）。安定版はv2.1.201まで進み、スリープ/再開でバックグラウンドセッションが無言のまま止まる不具合を修正、Sonnet 5が会話途中のsystemロールを使わなくなりスクリーンリーダー対応も改善した。少し前のv2.1.197〜198ではClaude in ChromeがGA、チャート設計を助ける/dataviz、背景エージェントの自動commit/push/ドラフトPRが入っている。VentureBeatはAnthropicの新規本番コードの約8割をClaudeが執筆していると報じた。

実務での使いどころ

自動継続の廃止は暴走抑止に効く一方、無人の定期実行では確認待ちで止まりうる。定型自動化は/configでタイムアウトを明示し、肥大化したCLAUDE.mdはskill/hookへ切り出すのが安全だ。fastやChrome連携は反復の多い検証作業へ寄せてコストと速度を実測したい。

注意点

「Manual」は表記変更で、挙動の実体はこれまでのdefaultと同じ。既存スクリプトの--permission-mode defaultはそのまま動くが、無人運用ではAskUserQuestionが停止する点に注意。

ひとことまとめ

既定は“手動で確認”へ、連携はChrome GAで外へ。賢く自走しつつ最終判断は人に戻す方向の更新だ。
