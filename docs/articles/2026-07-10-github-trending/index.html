7/10の話題リポジトリ — エージェントに実務作法・ローカル会議AI・WiFi空間センシング

AIに実務の作法を教える、会議データを外に出さない、カメラなしで見守る——3本に共通するのは「便利さの裏で犠牲になっていたもの（品質・プライバシー）」を技術で取り戻す動き。

📅 2026年7月10日 ⏱ 約3分 🔗 出典 3 件

今日の3本は言語もジャンルもばらばらだが、「既存のやり方が犠牲にしてきたもの」を回収しにいく点で通じている。エージェントの出力品質、会議データのプライバシー、見守りにおけるカメラの侵襲性。それぞれの「課題」と「提供機能」を見ていく。

addyosmani/agent-skills（JavaScript・⭐75k）

解決したい課題: AIコーディングエージェントのコードは「動く」が、コードレビューの観点、パフォーマンス改善の定石、保守性への配慮といった実務品質（production-grade）のエンジニアリング作法を欠きがち。個々の開発者がプロンプトで毎回教え直す非効率が常態化していた。

提供する機能: そうした実務知を「スキル」としてパッケージ化したキュレーション集。Claude Codeなど主要エージェントにそのまま導入でき、レビューや性能改善の作法を装備させられる。Chrome開発で知られるGoogleのAddy Osmani氏が編んでいる信頼性も効いており、直近7日で+約6,700スター、本日のTrending 1位。エージェントスキルブームの本命だ。

Zackriya-Solutions/meetily（Rust・⭐22k）

解決したい課題: クラウド型の会議AIアシスタントは便利な反面、会議音声・議事内容という最も機密性の高いデータを外部に送信する。コンプライアンス上クラウド利用が難しい企業や個人には採用しづらかった。

提供する機能: Parakeet/Whisperによる従来比4倍速のライブ文字起こしから要約・ノート生成までを完全ローカルで実行する会議アシスタント。データは一切外に出ない。直近7日で+約8,400スターと3本中最も勢いがあり、クラウド型会議AIの対抗馬として注目されている。

ruvnet/RuView（⭐77k）

解決したい課題: 高齢者の見守りや在室検知はカメラやウェアラブルが前提で、プライバシーの抵抗感・装着の手間・導入コストが壁だった。

提供する機能: 家庭のWiFi電波をセンサー化し、壁越しに人の存在・呼吸・心拍・活動状態を検知する空間センシングシステム。ESP32のような安価なエッジデバイスと8KBの超軽量AIモデルで動き、Home Assistant・Apple Home・Google Home・Alexaに統合できる。カメラ不要の見守りという切り口で本日Trending 2位を維持。

まとめ: 自分が試すならまずagent-skills。Claude Codeに数個入れてレビュー品質の変化を見るだけで元が取れる。meetilyは社内議事録フローのローカル化検証に、RuViewはESP32が手元にあれば週末プロジェクトとして面白い。
