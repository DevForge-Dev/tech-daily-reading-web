Flutter 4.0の輪郭 — Impeller 2.0とデザインのデカップリング

安定版は Flutter 3.44 / Dart 3.12.2 で、次は2026年8月予定。今日の主役はその先の次期メジャー Flutter 4.0 の輪郭だ。Impeller 2.0 と Dart 4.x 統合を軸に mid-2026 が取り沙汰されるが、日程よりコアゴール（とりわけMaterial/Cupertinoのデカップリング）達成を優先する方針が示された。

📅 2026年7月5日 ⏱ 約2分 🔗 出典 3 件

なぜ今これが話題か

安定版はFlutter 3.44 / Dart 3.12.2で、次の安定版は2026年8月予定。今日の主役はその先、次期メジャー Flutter 4.0の輪郭だ。Flutterチームは日程より「コアゴール達成」を優先する方針を示した。

何が新しいか

4.0はImpeller 2.0レンダリングエンジンとDart 4.x統合が軸で、mid-2026 が取り沙汰される。その前提条件がデザインのデカップリング——Material/Cupertinoをコアから独立した versioned な pub.dev パッケージ（将来のmaterial_ui/cupertino_ui）へ切り出す作業が2026初頭から進行中だ。モジュール化でアプリサイズ縮小とSDKリリースを待たないUIの独立更新を狙う。言語面ではDart 3.12が private named parameters を初期化フォーマルで対応し、年内にPrimary Constructors／Augmentationsを出荷予定と、いずれもDart 4.x世代への布石だ。

実務での使いどころ

本番の凍結面は3.44 / 3.12.2に据えるのが安全。4.0系の機能はまだ experimental だ。状態管理は新規＝Riverpod 3.x、監査/規制＝Bloc 9.2 が基本線で、Impellerは既にiOS/モダンAndroidの安定既定エンジンになっている。

注意点

デカップリングでMaterial/Cupertinoがパッケージ化されると、依存やimport経路が変わりうる。移行を前提にバージョン固定と更新計画を持ちたい。experimentalな4.0系を本番の土台にしない。

ひとことまとめ

4.0は「速さ（Impeller 2.0）」と「軽さ・独立更新（デカップリング）」の同時追求。今は3.44で堅く作りつつ、パッケージ分離の波に備える時期だ。
