Dart 3.12 Primary Constructors（実験プレビュー）と Impeller の WebGPU 移植 — 言語の簡潔化とツール/Web高速化（Flutter 3.44）

安定版 Flutter 3.44.0 / Dart 3.12 を土台に、今日の主役は「言語の簡潔化」と「ツール/Webの高速化」。Dart 3.12 の Primary Constructors は実験プレビューとして、クラス本体でコンストラクタを短く宣言できる。dot shorthands は正式サポート。描画では Impeller がもはや既定で、Android 10+ では Skia を撤去。DevTools は WasmGC 既定化で解析が高速になり、Impeller の WebGPU 移植でブラウザにもネイティブ級の性能が見えてきた。

なぜ今これが話題か

安定版は Flutter 3.44.0 / Dart 3.12（Google I/O 2026, 2026-05-18）で、年4回の安定リリースを継続中。今日の素材が示すテーマは2つ——言語の簡潔化と、ツール/Webの高速化だ。日々の書き味と計測体験に効く話なので、リリース情報として押さえておきたい。

何が新しいか

言語面の目玉は Dart 3.12 の Primary Constructors。「実験プレビュー」として提供され、クラス本体内で new / factory を使う短い構文でコンストラクタを宣言でき、本体が空のクラスはセミコロンで終えられる。有効化は primary-constructors フラグ。dot shorthands（.value 省略記法）は正式サポートされ、enum や静的メンバ参照の宣言が簡潔になった。構文はフィードバックで変わり得るため、Primary Constructors の本番全面採用は様子見が推奨だ。

// イメージ（実験プレビュー / 構文は変更され得る）
// 本体が空ならセミコロンで終了できる
// dot shorthands：型が決まる文脈で .value と書ける
final Color c = .blue;          // = Color.blue
return switch (s) { .idle => ... };

ツール/Webの高速化

描画の Impeller はもはや実験的ではなく既定。iOS とモダン Android（API 29+）の標準レンダラとして、シェーダーをビルド時に AOT 事前コンパイルすることで長年のシェーダージャンクをほぼ解消し、複雑シーンの平均フレームラスタライズ時間を約50%短縮した。さらに Android 10 以上でレガシー Skia バックエンドが正式撤去され、純 Impeller(Vulkan) に。アニメ負荷の高い処理で Skia 比 約17%のバッテリー消費低減も報告されている。DevTools は WasmGC に既定コンパイルされ、テレメトリ解析の遅延が 200ms 超短縮。ロードマップでは Impeller を WebGPU へ移植中で、ブラウザ上の Flutter にもネイティブ級グラフィック性能が見えてきた。

自分が試すなら / 注意点

  • dot shorthands を既存コードの enum/静的参照に当て、読みやすさを比較する。
  • Primary Constructors は primary-constructors フラグで小さなモデルクラスから試し、構文変更リスクを織り込む（本番全面採用は避ける）。
  • 性能は必ず profile モードで計測し、Performance Overlay の赤バーでジャンクを検出する。
Impeller 既定化で Rive 等の描画/性能に差が出る場合があり実機検証が必要。最頻出の性能問題は依然「不要な再ビルド」で、const / build() 分割 / Selector が基本線である点は変わらない。

ひとことまとめ

「言語は短く（dot shorthands は正式・Primary Constructors は様子見）、描画と計測は Impeller 起点で速く」。まずは dot shorthands と profile 計測から取り入れると流れに乗れる。
