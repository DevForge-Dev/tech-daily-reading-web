Flutter 3.44 / Dart 3.12.2 — flutter run のWeb構成ファイル、画面共有の機密保護、Dart 3.12 dot shorthands

安定版はFlutter 3.44.0 / Dart 3.12.2（docs最終更新2026-06-24）。今日の焦点は派手な新ウィジェットより開発体験（DX）とセキュリティの底上げだ。flutter run がWeb構成ファイルに対応し、Androidは画面共有時に機密UIを隠せるようになった。Dart 3.12のdot shorthandsと合わせて、地ならし系のアップデートを読む。

なぜ今これが話題か

安定版は Flutter 3.44.0 / Dart 3.12.2（docs最終更新 2026-06-24）。リリースは年4回（2月/5月/8月/11月）で、次の安定版は2026年8月予定。今回の焦点は新ウィジェットの追加ではなく、開発体験（DX）とセキュリティの底上げという地ならしにある。日々の flutter run やセキュアな画面まわりで効く、地味だが効用の大きい変更が入った。

何が新しい/重要か

第一に flutter run がWeb設定の構成ファイルに対応し、host / port / certification / header をまとめて指定できる。構成ファイルは proxy設定にも対応し、同一ホストの動的エンドポイントへ繋ぐWebクライアント開発が容易になった。第二に、Androidで画面共有・録画から指定UIを隠す機能。顧客情報など機密コンテンツを共有画面で保護できる。第三に新設の「Create with AI」ガイドが、Gemini Code Assist / Gemini CLI / Dart & Flutter MCP Server を使ったAI機能の組み込みを解説。Dart 3.12は dot shorthands と、initializing formals / primary constructor parameters でのprivate named parameters（先頭 _ を自動省略）をサポートする。

仕組みと使いどころ

構成ファイルは、これまでコマンドラインフラグに散らばっていた接続設定を一箇所に集約する。proxy対応により、ローカル開発でAPIゲートウェイや動的なバックエンドへ透過的に繋げる。画面共有保護は、決済・本人確認・顧客台帳のような映してはいけない画面を持つ業務アプリで効く。

自分が試すなら

Web開発では構成ファイルにproxyを書き、動的エンドポイント越しの開発フローを整える。機密画面を持つアプリではAndroidの画面共有保護を有効化し、共有/録画時に該当UIが隠れるか確認する。AI機能を足すなら「Create with AI」ガイドとDart & Flutter MCP Serverから入るとよい。

注意点

現行は3.44系で、ここでの情報は docs 2026-06-24時点。レンダリングは Impeller が 3.27以降 iOS/Android API29+ で既定（フレームラスタライズ約50%高速、120fpsを一貫維持）になっている点も合わせて押さえたい。計測は必ずprofileモードで行い、最大の敵である不要な再ビルドは const・build分割・Selector で抑える。

ひとことまとめ

3.44はDXとセキュリティの地ならし。flutter run のWeb構成ファイル（host/port/cert/header＋proxy）で接続設定が一元化し、画面共有保護が機密業務アプリを守る。Dart 3.12のdot shorthandsと合わせ、日々の開発の手触りを静かに改善する更新だ。
