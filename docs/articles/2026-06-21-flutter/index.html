Flutter、ネイティブ120fpsとデスクトップ拡張 — Impellerで1フレーム約8ms、macOSにpopup window・Windowsがスタイラス筆圧対応

フラッグシップ端末は90/120Hzが主流になり、滑らかさの基準が上がった。新描画エンジン Impeller は高リフレッシュ端末で1フレーム約8msのネイティブ120fpsを実現する。安定版は Flutter 3.44.0 / Dart 3.12。加えて macOS が popup window をサポートし、Windows がスタイラスの回転・筆圧に対応するなど、デスクトップ/クリエイティブ用途も一段前進した。

なぜ今これが話題か

フラッグシップ端末は 90/120Hz が主流になり、滑らかさの基準が上がった。Flutterの新描画エンジン Impeller は高リフレッシュ端末で1フレーム約8msのネイティブ120fpsを実現する。加えて 2026 はデスクトップ/クリエイティブ用途が一段前進した。

何が新しいか

安定版は Flutter 3.44.0 / Dart 3.12（Google I/O 2026, 2026-05-18）。Impeller は iOS / モダンAndroid（API 29+）の既定レンダラで、シェーダーをビルド時にAOT事前コンパイルし、最大の弱点だったシェーダーコンパイルジャンクを解消する。デスクトップ面では macOSでpopup windowをサポート（メニュー/ツールチップをOSネイティブのポップアップで表示）、Windowsがスタイラス入力に対応し回転と筆圧（pressure）の精密トラッキングを含む。Linux/Windowsのpopup windowは将来対応予定だ。

仕組みのポイント

120fpsの予算は1フレーム約8ms（60fps=16ms / 90fps=11ms）。Impeller はシェーダーを事前に用意し、Metal(iOS)/Vulkan(Android)の低レベル経路を直接叩くことで、初回フレームから安定して8msに収める。移行済みアプリはスクロール/遷移が滑らかになり、持続アニメでバッテリー効率も改善と報告されている。

自分が試すなら

  • 120Hz端末で重い CustomPaint/シェーダーアニメ画面を profileモード＋DevToolsで計測し、8ms以内に収まるか確認する。
  • macOSアプリのメニュー/ツールチップを popup window 化してネイティブ感を上げる。
  • お絵描き/署名系アプリで Windows スタイラスの筆圧・回転を取り込む。

注意点

性能計測は必ず profileモードで（debug VMは大幅に遅い）。Impellerでも不要な再ビルドは別問題として残るため、const化や Selector による局所再描画は依然有効。描画差が出たら --no-enable-impeller でSkiaと比較し切り分ける。popup windowは現状macOSのみで、Linux/Windowsは将来リリース予定だ。

ひとことまとめ

「8msに収める」が120fps時代の合言葉。Impellerでシェーダージャンクが消え、macOS popup window と Windows スタイラスでデスクトップ表現も前進した。まずは重い描画画面をprofileで測り直すのが近道だ。
