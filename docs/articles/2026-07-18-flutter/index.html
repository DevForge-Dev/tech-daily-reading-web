GenUI SDK と A2UI プロトコル — エージェントが“実 Flutter ウィジェット”をその場で組む（3.44 / Dart 3.12）

Google I/O 2026 最大の構造転換。オープンな A2UI（Agent-to-UI）の上で、AIエージェントが実ウィジェットをランタイム構成し完全インタラクティブな画面を生成。安定版は Flutter 3.44.0 / Dart 3.12.2、Impeller は Android既定で120fpsが実務射程に。

📅 2026年7月18日 ⏱ 約3分 🔗 出典 4 件

今日の主役は 「GenUI SDK と A2UI プロトコル」——Google I/O 2026 で最も構造的に重要な発表だ。事前定義レイアウトやアプリストア更新に頼らず、AIエージェントが実 Flutter ウィジェットをその場で構成し、ユーザーが達成したいことに応じて完全インタラクティブな画面をランタイム生成する。土台はオープンな A2UI（Agent-to-UI）プロトコルだ。

何が新しいか

3.44/3.12 は増分でなく3方向の構造転換：作り方（agentic/GenUI）、描画（Android Impeller-only＋HCPP）、構造（Material/Cupertino のデカップリング＝material_ui / cupertino_ui へ移行）。GenUI は Agentic Hot Reload（Dart MCP Server 経由でエージェントが起動中アプリを自動 hot reload）や Genkit Dart（Google/Anthropic/OpenAI 系をモデル非依存の統一APIで扱う）と噛み合い、AIロジックをサーバ側 Dart に退避できる。安定版は Flutter 3.44.0 / Dart 3.12.2、次の機能リリースは2026年8月予定（年4回：2/5/8/11月）。

描画とパフォーマンス

Impeller が iOS＋モダンAndroid（API 29+）で既定。シェーダーをビルド時（AOT）にコンパイルしてコンパイルジャンクを排除し、ラスタライズ約50%高速・120fps安定・アニメ多用でSkia比 約17%低い電力ドレイン。120fpsチェックリストは、const の積極活用、アニメ部を RepaintBoundary で囲む、ListView に固定 itemExtent。フレーム予算はラスタライズ<4ms=60fps／<8.3ms=120fps、計測は必ず profile モードで。

状態管理の基準線

迷ったら Riverpod 3.0 が既定（unified Ref・Mutations・指数バックオフ自動リトライ・オフライン永続化・画面外で listener 自動pause）。監査/大規模は Bloc 9.0（イベント対応の監査証跡、mounted 安全チェック）。2026ベンチの widget rebuild は Bloc 2.3ms / Riverpod 2.0ms、テスト行数は Riverpod が平均30〜40%少ない。現行は Riverpod 3.3.x / flutter_bloc 9.1.x。

注意点

GenUI/A2UI は登場したての基盤で、実運用パターンはこれから固まる。Material/Cupertino のコアは凍結済みで material_ui / cupertino_ui へ移行中——依存の棚卸しが要る。Impeller はミッドレンジAndroid（一部 Adreno GPU にクセ）で実機検証必須、fragment shader の重い分岐は避け mix/step/clamp へ。

ひとことまとめ

「画面はエージェントがその場で組む」時代の入口。まずは GenUI/Genkit Dart の概念を掴みつつ、Impeller 前提で const・RepaintBoundary・itemExtent の120fps基礎を固めておくのが吉。
