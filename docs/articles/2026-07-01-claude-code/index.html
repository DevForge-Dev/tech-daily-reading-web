Claude Code

拡張機構が『選び分け』の時代へ — hooks/subagents/skills と階層サブエージェント3段、6月の総仕上げ

7月1日のダイジェストは、6月の機能ラッシュを『品質・信頼性』で締める公式アップデートと、拡張機構(hooks/subagents/skills)の使い分けを説く実践記事が並んだ。階層的サブエージェント(最大3段)・dynamic workflows・Artifactsベータに、『新規本番コードの約80%をClaudeが生成』という数字まで。対話補助から自動化基盤への移行がはっきり形になってきた。

📅 2026年7月1日 ⏱ 約3分 🔗 出典 5 件

なぜ今これが話題か

この半年でClaude Codeは「対話的なコーディング補助」から「複数エージェントを束ねる自動化基盤」へ重心を移してきた。6月はその総仕上げの月で、7月1日の公式「What's new」は新機能ではなく品質・信頼性——組織単位の既定モデル、読みやすいセッション名、クリック可能な添付、MCP耐障害性とOAuth処理の改善、長時間セッションのCPU/メモリ低減——を前面に出した。派手さより「毎日安心して回せるか」に軸足が移った合図だ。

何が新しいか

SitePointの「June 2026: 10 New Features」は開発者が押さえるべき目玉として、親が子を、子がさらに子を生む階層的サブエージェント生成（最大3段）と、大規模課題向けのdynamic workflows、そしてセッション出力をその場で更新される共有ライブページに変えるArtifactsベータを挙げる（この読み物自体がArtifacts運用の一例）。加えてTeam/Enterprise向けにTrusted Devicesが入り、リモート操作前のデバイス検証を必須化できる。

仕組みのポイント

ofox.aiの完全ガイドは、hookは「コードでルールを強制する決定論的な仕組み」、subagentは「独立コンテキストで並列に動く専門ワーカー」、skillは「実際のドメインロジックや補助ファイルを伴う場面」で使う、と整理する。強力なのはsubagentだが、Nimbalystはトークン消費が単一スレッドの約7倍に達しうると釘を刺す。レート上限が近いときは、あえて単一スレッドの方が安く速い。

自分が試すなら

  • 毎朝の自動処理は「必ず走らせたい検査」をhookに、「重い調査」だけをsubagentに切り出し、多段化は本当に必要な所に限定する。
  • 繰り返す手順はskill化して再利用し、プロンプトを毎回書き直さない。
  • Artifactsベータで、日次の成果を再訪できるライブページに固定する。

注意点

subagentは便利だが約7倍のトークンを食う。階層3段×並列を無計画に使うと請求が跳ねる。6月15日の課金分離（自動化はAPI月次クレジット）と併せ、「どの自動化がいくら使ったか」を必ず監視したい。

ひとことまとめ

6月のClaude Codeは、hooks/subagents/skillsの「選び分け」と階層サブエージェント・dynamic workflows・Artifactsで自動化基盤を固めた。強力さの裏でコストは正直なので、用途に合った最小構成が主導権を握る。
