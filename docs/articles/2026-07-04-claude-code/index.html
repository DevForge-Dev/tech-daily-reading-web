Claude Code

デフォルトが切り替わった日 — Opus 4.8・Sonnet 5・Claude Code Artifacts

7/4のダイジェストは“既定値”の話。上位プランの既定がOpus 4.8になりeffortは全面で「高」、fastモードは3倍安。Claude CodeはSonnet 5が既定で100万トークン文脈＋割引価格。さらにセッションをライブ共有できるArtifactsが加わり、ステアリング指針も公式に整理された。

📅 2026年7月4日 ⏱ 約2分 🔗 出典 5 件

なぜ今これが話題か

ここ数日のClaude Codeは、新機能の追加より「既定でどう動くか」の再設定が続いている。モデル・価格・共有のデフォルトがまとめて更新され、意識せず使うだけで挙動が変わる。7/4はその節目だ。

何が新しいか

まずモデル。Opus 4.8がMax・Team Premium・Enterprise従量課金・APIの既定になり、Claude Code/APIを含む全サーフェスでeffortパラメータが既定「高」になった。高effortでもOpus 4.7の既定と同程度のトークン消費で性能が上がり、2.5倍速のfastモードは従来比3倍安。一方Claude Code本体はSonnet 5が既定で、100万トークンの文脈窓と8/31までの割引価格$2/$10 per Mtokが案内された。加えて組織デフォルトモデル・読みやすいセッション名・クリック可能な添付も入る。そしてClaude Code Artifacts——セッションの成果をライブ共有HTMLページとして公開し、進行に合わせ自動更新、公開ごとに新バージョンが作られ履歴から復元できる（Team/Enterpriseベータ・組織限定共有）。

ステアリングの整理

公式は拡張手段の使い分けも整理した。毎回必ず守らせたい規則はCLAUDE.md、必要時だけ読み込む手順はskill、決定論的に強制したい処理はhook、重い/並列/文脈分離が要る仕事はsubagentへ委譲——という対応づけだ。CLAUDE.mdを肥大させずに挙動を安定させる設計指針になる。利用実態の調査では平均週20時間使われ、エージェントが動くGitHubプロジェクトは半年で2倍超に増えたという。

自分が試すなら

  • fastモードが3倍安になったので、反復の多い自動処理をfastへ寄せてコストを実測する。
  • 毎朝の定型レポートをArtifact化し、再訪できるURLとして残す。
  • 肥大化したCLAUDE.mdをskill/hookへ切り出し、必須処理だけhookで強制する。

注意点

effort既定「高」は品質に効くが、自走の回数×高effortで請求は伸びる。カテゴリ別/usageで消費を棚卸ししたい。割引価格($2/$10)は8/31までの期限つきである点にも注意。

ひとことまとめ

モデル(Opus 4.8/Sonnet 5)・価格(fast3倍安/割引)・共有(Artifacts)の既定が同時に更新。“設定しなくても賢く速く安く、成果は共有できる”方向へ寄せる一日だった。
