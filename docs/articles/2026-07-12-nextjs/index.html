WebGL不要のカード演出 — CSS 3D Transformsでフリップ＆ポインタ連動チルト

perspective・preserve-3d・backface-visibilityの3点セットだけで、依存ゼロ・バンドル増ゼロの3Dカードが作れる。鍵は「どこをClient島にするか」と「flat化の罠」。

📅 2026年7月12日 ⏱ 約3分 🔗 出典 4 件

プロダクトカードや料金カードに奥行きを足したいとき、react-three-fiberを持ち出すのは大袈裟だ。CSSの3D変換はGPUコンポジタで動くため軽量で、カードフリップ・ポインタ追従チルト程度なら依存ゼロで済む。R3Fが「本物の3Dシーン」用なら、こちらは「UI部品にちょっとした奥行き」用の定番という住み分けになる。

柱は3つ。まずperspectiveはカメラ距離で、親に置くプロパティ版は子全員が消失点を共有し、transform: perspective(1000px)の関数版は要素ごとに個別の消失点を持つ。カードが並ぶグリッドでは親版だと端のカードが歪むため、単体演出は関数版が自然なことが多い。値は1000〜1200pxが自然、500pxで劇的、2000px以上で控えめ。次にtransform-style: preserve-3dで子を同じ3D空間に残し、最後にbackface-visibility: hiddenで回転中の鏡文字の裏写りを消す。translateZ()はperspectiveが無いと視覚上何も起きない点も忘れやすい。

最大の落とし穴がこの図の下段だ。overflow（visible以外）・opacityが1未満・filter・clip-path・maskなどをpreserve-3dの要素に付けると、CSS Transforms Level 2の仕様によりused valueがflatへ強制的に戻る。「角丸＋overflow:hiddenを足した瞬間フリップが壊れた」の正体はほぼこれで、角丸クリップは面側の要素へ逃がすのが正解。

.scene { perspective: 1000px; }
.card  { transform-style: preserve-3d; transition: transform .6s; }
.scene:hover .card,
.scene:focus-within .card { transform: rotateY(180deg); }
.face  { position: absolute; inset: 0; backface-visibility: hidden; }
.back  { transform: rotateY(180deg); }

Next.js（App Router）での組み込みは切り分けが気持ちいい。:hover/:focus-within駆動のフリップは純CSSなので'use client'不要、Server Componentのままhydrationゼロ。一方チルトはpointerイベント＋rAFが要るのでClient必須だが、カードの「枠」だけをClientにして中身はchildrenで差し込めばRSCの恩恵を保てる。ポインタ座標はstateに入れずel.style.setProperty('--rx', ...)とCSS変数経由で書けば再レンダー完全回避。rAFスロットリング、離脱時のリセット、そしてprefers-reduced-motionではフリップを即時切替・チルト無効化まで入れて完成形。

ひとことまとめ: フリップはServer Componentのまま純CSS、チルトだけ薄いClient島。壊れたらまず「preserve-3dをflat化するプロパティ」を疑う。
