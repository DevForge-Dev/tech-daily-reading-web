JSなしで enter/exit を完結させる — モダンCSSの4機能を実装目線で噛み砕く

@starting-style・transition-behavior: allow-discrete・keyframes内のdisplay切替・overlay。これらが揃い、モーダルやポップオーバーの出入りをCSSだけで描けるように。App Routerでの組み込みと落とし穴まで。

📅 2026年6月16日 ⏱ 約4分 🔗 出典 4 件

なぜ今これが話題か

「要素が現れる／消える」動き（モーダル、トースト、ドロップダウン、リストの追加/削除）は、長らく JS や setTimeout で display:none を遅延させる定番ハックを必要としてきた。CSS だけでは display:none への/からのトランジションができず、マウント直後の初期値からの enter も表現できなかったからだ。これを解く4機能が出揃い、JS無し・CSSのみで enter/exit を完結できるようになった。Server Component のまま（'use client' 不要で）使えるのが Next.js 的に大きい。

何が新しいか

鍵は4つ。@keyframes 内での display/content-visibility 切替（Chrome 116〜）、離散プロパティもトランジション対象にする transition-behavior: allow-discrete（Chrome 117〜）、マウント前の初期スタイルを定義する @starting-style（Chrome 117〜）、top-layer からの退場を exit 中も保持する overlay（Chromium 中心）。@starting-style＋allow-discrete＋keyframes の display 切替は Baseline Newly available（2024年8月、Chrome 117 / Safari 17.5 / Firefox 129）。overlay は2026年でも対応が限定的なので「より滑らかにする追加要素」として、無くても破綻しない設計にする。

仕組み（3点セット）

HTML側は <button popovertarget="m">開く</button> と <div id="m" popover class="menu">…</div> だけ。JS不要で開閉する。

App Routerでの使いどころ

これは純粋なCSS機能で、CSSファイルに 'use client' の概念は無い。app/globals.css や CSS Modules にそのまま書け、Server Component で描画される要素にも効く。state で開閉を制御する場合のみ、その開閉ロジックを持つコンポーネントが Client になる。ネイティブ <dialog>/popover を使えば popovertarget 属性だけで JS なしに開閉できる。

落とし穴

  • transition ショートハンドは transition-behavior を normal にリセットする。allow-discrete は必ずショートハンドの後に書く。
  • @starting-style は通常宣言の後ろ（前だと負ける、Chrome 130以降は順序が厳格化）。
  • 条件付きマウント（{open && <Modal/>}）だと exit アニメが効かない。DOM が即消えるため。CSSだけで exit を見せるなら要素は常時マウントして display/属性で出し入れする（<dialog>/popover が自然）。React state でアンマウントしたいなら exit は View Transitions API かライブラリ併用を検討。
  • UIの動きなので @media (prefers-reduced-motion: reduce) で transition: none を必ず用意する。

ひとことまとめ

enter/exit のためだけにライブラリを足す時代が終わりつつある。Server Component と相性が良く、まずはポップオーバーや <dialog> の出入りから CSS だけで置き換えてみるのが安全な第一歩。
