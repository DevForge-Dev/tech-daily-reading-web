AutoAnimate — 親要素に1行、DOM変化を滑らかにするゼロコンフィグ・アニメ

@formkit/auto-animate v0.9.0。親要素にrefを渡すだけで、直下の子の「追加・削除・並び替え」をFLIPで自動アニメーション。約2.5kbでreduced-motionを既定で尊重。App Routerではリスト部分だけをClient化すればRSCと共存できる。

📅 2026年7月14日 ⏱ 約3分 🔗 出典 3 件

リストやカンバン、フィルタ結果、アコーディオン——要素が入れ替わる瞬間の「カクつき」を、演出を作り込まずに解消したい。そのニッチに刺さるのが@formkit/auto-animateだ。FormKitチーム製・MIT・GitHub 13.8k★、最新はv0.9.0（2025-09-05）で2026年も安定版。ランタイムは約2.5kbとMotionやReact Springより1〜2桁小さい。

何が新しい／何が違うか

思想は「Prettier for motion」。既存マークアップに後付けできる点が肝で、コンポーネントを書き直さずサードパーティ製UIにもrefを刺すだけで効く。Motion/GSAPが「演出を作る」ライブラリなのに対し、AutoAnimateは「DOMの構造変化を説明する」道具で、用途がきれいにすみ分かる。

App Router での組み込み

DOMを直接触るクライアント専用ユーティリティなので、フックを使うコンポーネントは'use client'必須。ただしページ全体をClient化する必要はなく、リストを描く部分だけ切り出せばデータ取得やページはRSCのまま保てる。

'use client'
import { useAutoAnimate } from '@formkit/auto-animate/react'

export default function Todos({ list }) {
  const [parent] = useAutoAnimate()
  // key は index ではなく安定・一意な id を使う
  return <ul ref={parent}>{list.map(t => <li key={t.id}>{t.text}</li>)}</ul>
}

これだけでlistに要素を足す／消す／並び替えるとFLIPで滑らかに動く。楽観的更新（useOptimistic）で足した子も自動でアニメーションされる。

落とし穴

並び替えには安定・一意なkeyが必須。key={index}にすると「移動」ではなく「中身の差し替え」と見なされ移動アニメが鳴らない。Flexでflex-grow:1の子は幅確定が遅れて正しく測れないことがあり、明示的な幅で安定する。CSSトランジションの置き換えではなく、ホバー等の「スタイル変化」は従来どおりCSSで書く。数百件の同時リフローには不向きで、そこはMotion/GSAPへ。アクセシビリティ面ではprefers-reduced-motion: reduceを既定で尊重し、その環境では即時反映になる（追加コード不要）。

ひとことまとめ: CRUD系UIの「入れ替わる瞬間」を最小コストで滑らかにする後付けFLIP。演出を作りたいときはMotion、変化を説明したいときはAutoAnimate。
