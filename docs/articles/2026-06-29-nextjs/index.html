GSAPが完全無料化、App RouterではuseGSAPが正解 — 自動クリーンアップ・scope・contextSafe と書き直されたSplitText 3.13

2025年4月、Webflow買収を経てGSAPが商用込みで100%無料化。SplitTextやScrollTriggerなど旧プレミアムが全部使えるようになり、選定理由が変わった。App Routerで安全に使う鍵は @gsap/react の useGSAP()。SSR安全・自動revert・scopeの三点と、約半分のサイズに書き直されたSplitText 3.13を整理する。

📅 2026年6月29日 ⏱ 約4分 🔗 出典 4 件

なぜ今これが話題か

GSAPが2025年4月に100%無料化された。Webflowが2024年秋にGreenSockを買収し、これまで有料のClub GreenSock限定だったSplitText / MorphSVG / DrawSVG / ScrollTrigger / ScrollSmoother / Inertiaを含む全機能が、商用利用込みで無償になった。タイムラインベースの精密なシーケンス、スクラブ連動の本格スクロール、文字単位のテキスト演出が課金なしに使える。Motion（旧Framer Motion）が「宣言的に動きを与える」のに対し、GSAPは「命令的なタイムラインで時間軸を細かく組む」のが得意で、両者は併用できる。

何が新しい/重要か

App Routerでの正解は @gsap/react の useGSAP()。useLayoutEffect/useEffect のドロップイン置換で、内部の useIsomorphicLayoutEffect によりSSR安全。フック中に作られた全アニメ・ScrollTrigger・SplitTextを gsap.context() が記録し、アンマウント時に自動revertする。Strict Modeの二重実行やクライアントナビでの二重登録事故を吸収し、手動 kill() 行脚が要らない。さらにSplitText 3.13で全面書き直し（約7kb、サイズ半減）。mask:'lines' でマスクリビールを内蔵、aria 自動付与、autoSplit でフォント遅延ロードやリサイズ時に自動再分割する。

仕組みと最小コード

Reactでは document.querySelector ではなくrefかscope付きセレクタを使うのが鉄則。useGSAP({ scope: ref }) でセレクタ文字列をコンテナ配下に限定できる。

'use client';
import gsap from 'gsap';
import { useGSAP } from '@gsap/react';
gsap.registerPlugin(useGSAP);

useGSAP(() => {
  gsap.from('.box', { y: 40, opacity: 0, stagger: 0.1 });
}, { scope: ref });   // 自動クリーンアップ + スコープ

自分が試すなら

ルート/レイアウトはServer Componentのまま保ち、動かす島だけ 'use client' に切り出す。reduced-motion対応は gsap.matchMedia() が相性抜群で、(prefers-reduced-motion: reduce) ブロックに「動かさず即完成形」を書けばメディアクエリ変化に自動追従する。宣言的レイアウトはMotion、作り込んだ演出はGSAP、と併用する構成を試したい。

注意点

GSAPはDOM/windowに触るため 'use client' 必須（RSCのままは不可）。App Routerの遷移後に高さが変わると位置がズレるので、画像/フォント確定後に ScrollTrigger.refresh()。ScrollSmootherは重いので、軽量なLenis等も選択肢。プラグインは gsap.registerPlugin() で使用前に登録する。

ひとことまとめ

無料化でGSAPの全武器が解禁。App Routerでは useGSAP の「SSR安全・自動revert・scope」が事故を消し、SplitText 3.13がアクセシブルな映画的リビールを手組みなしで実現する。宣言的はMotion、精密演出はGSAPの併用が現実解だ。
