SVGフィルターで“溶ける・ゆらぐ” — Goo(feGaussianBlur+feColorMatrix)と液体feTurbulence

DOMを増やさず質感を足すSVGフィルター。重なる図形を溶かすGooはぼかし→アルファ急峻化の2段、液体ゆらぎはfeTurbulence→feDisplacementMap。定義も適用も純CSS/SVGで完結するのでRSCのまま置ける。ただしGooはChromium前提・フィルター領域は小さく、が鉄則。

📅 2026年7月4日 ⏱ 約2分 🔗 出典 4 件

なぜ今これが話題か

SVGのフィルタープリミティブは、CSSのfilter: url(#id)から参照するだけで、要素を足さずに「溶け合う」「にじむ」「液体のように歪む」質感を作れる。輪郭（パス描画/モーフ）とは別レイヤーの、ピクセルの合成・歪みを扱う手法だ。少ないマークアップで強い印象を出せて、しかもServer Componentのまま使えるのが今どきの旨味になる。

何が新しい/重要か

核は2つのパイプライン。Goo（溶け合い）はfeGaussianBlurで重なった図形のアルファをぼかし、feColorMatrixでアルファだけを急峻化（例 ...0 0 0 18 -7）——18倍して7を引くことで、ぼけたアルファがしきい値を跨いだ所で硬いエッジに“戻り”、近接図形がヌルっと橋渡しされて1つの塊に見える。液体ゆらぎはfeTurbulenceでPerlinノイズを生成し、feDisplacementMapのscale（10〜30程度）で元画像を歪ませる。

仕組み・実装のポイント

フィルター定義も適用も静的なCSS/SVGなので'use client'不要。定義は非表示の<svg width="0" height="0">にインラインし、適用側は.cluster { filter: url(#goo); }を当てるだけ。個々の子要素はCSSのtransform/opacityで動かす（合成スレッド）。アニメ手段は、宣言的なSMILは本番非推奨、CSSはフィルター属性値を直接動かしにくい、結局requestAnimationFrameでbaseFrequencyをsin波で微小オシレーションさせるJSが最も確実。動かす所だけ小さなClient島に切り出すのが定石だ。

注意点・落とし穴

最重要は対応差。Gooのアルファ急峻化トリックはChromium/Chromeで確実に効くが、Safari/Firefoxではただのぼかしに見える構成があり塊化しない。本番はChrome/Edgeで確認し、非対応時はフィルターを外してぼかし無しにフォールバックする。性能面ではフィルター領域を小さく保つのが鉄則（全画面に大判フィルターを敷いて毎フレーム再計算すると重い）。idはページ内で一意にし、定義はアートの近くにインラインで置く。

ひとことまとめ

純CSS/SVGで質感を足せてRSCと好相性。Gooは“ぼかし→アルファ急峻化”、液体は“ノイズ→変位”。効くブラウザとフィルター領域のサイズだけは最初に押さえる。
