Claude Code

Sonnet 5がClaude Codeの既定モデルに — 1Mコンテキスト標準化とv2.1.205の監査性強化

Claude Sonnet 5が既定モデルとなりネイティブ1Mトークンコンテキストが標準に。v2.1.205はautoモードのトランスクリプト改ざんブロックで無人運用の監査性を高め、/dataviz・/doctorと管理コンソール新タブが個人・組織の両面を底上げした。

📅 2026年7月10日 ⏱ 約3分 🔗 出典 6 件

今日のClaude Codeは「基盤モデル」「安全性」「可視化」の3層が同時に動いた一日だった。単発の機能追加ではなく、大規模利用を前提としたプラットフォーム整備が進んでいる印象だ。

最大のニュースはClaude Sonnet 5がClaude Codeの既定モデルになったこと。ネイティブで100万トークンのコンテキストウィンドウを備え、大規模コードベースを分割せずに読み込める。しかも8月31日まで$2/$10 per Mtokのプロモーション価格が適用されるため、移行のコスト障壁も低い。モノレポ横断のリファクタリングや調査タスクで「コンテキストに入り切らないから分割する」という段取り自体が不要になる変化だ。

安全性側では、7月8日公開のv2.1.205がautoモードにセッショントランスクリプトファイルの改ざんをブロックするルールを追加した。エージェントが自身の実行記録を書き換えられないようにする強化で、無人運用の監査証跡が守られる。あわせてJSONスキーマ出力、max-turns制限時のメッセージ処理、Windowsのworktree操作、バックグラウンドエージェントのステータス表示も修正された。

ツール面では、チャート・ダッシュボード設計をガイドする/datavizスキルと、セットアップを一括点検する/doctorの総合診断が追加。管理コンソールにはUsage/Valueの2タブが加わり、アクティブ開発者数・人気コマンド・コミット単価まで日次で可視化できるようになった。周辺記事では、ofox.aiの完全ガイドが「テンプレならスラッシュコマンド、ドメイン知識はスキル、隔離・並列はサブエージェント、強制はフック」という使い分け原則を提示しており、skills→hooks→subagentsの導入順は自分の整理としても採用したい。ニュース面ではCoworkのWeb・モバイル拡大、Epicで利用の過半数が非開発者という報道もあり、非エンジニア領域への浸透が数字で裏付けられた。

自分が試すなら、まず/doctorで環境を点検し、大きめのリポジトリでSonnet 5の1Mコンテキストに横断調査を投げてプロモ期間中にコスト感を掴む。注意点は既定モデル変更で従来のOpus前提のワークフローと挙動・コストが変わり得ること。CIで使うバージョンはv2.1.205以降に上げ、autoモードの監査強化を有効に使いたい。

ひとことまとめ: 1Mコンテキストが「特別なもの」から既定になった日。安全性と可視化も揃い、組織展開のフェーズが本格化している。
