openai/codex-plugin-cc（JavaScript・⭐23.1k）

解決したい課題: コーディングエージェントの乱立で、開発者はClaude CodeとCodexを行き来し、レビュー品質や会話コンテキストがツール間で分断される不便が生まれていた。OpenAIは競合Anthropic製のCLIの内側に公式に入り込むことで、この切り替えコスト自体を無くしにきている。

提供する機能: Claude Code内からCodex本体の/reviewと同品質のコードレビューを実行でき、会話コンテキストを引き継ぐCodexスレッドを作ってタスクを委譲できる。バックグラウンドのCodexジョブの一覧・結果確認・キャンセルにも対応。急上昇（本日+629）の理由は、この異例のクロスベンダー展開そのものだ。

elastic/elasticsearch（Java・⭐77.3k）

解決したい課題: RAGやAIエージェントには「全文検索とベクトル検索の両立」と大規模ベクトルインデックスの構築速度が必要だが、専用ベクトルDBと既存検索基盤の使い分けは運用負担が大きい。

提供する機能: ハイブリッド検索（全文＋ベクトル）をネイティブにサポートし、NVIDIA cuVS連携のGPUアクセラレーションでベクトルインデックス構築を最大12倍高速化。エンタープライズ実績が豊富な分散RESTful検索・分析エンジンとして、エージェントの検索/コンテキスト基盤に収まる。AI向け強化の連打が、老舗を本日のTrendingに押し戻した。

pytorch/pytorch（Python・⭐101.4k）

解決したい課題: 研究に必要な柔軟なモデル記述と、プロダクションに必要な学習・推論速度の両立という、深層学習フレームワークの根源的な課題。

提供する機能: 動的計算グラフによる柔軟な記述とGPUアクセラレーションを核に、torch.compileなどで高速化が進む。LLMを含む主要AI研究/プロダクションの事実上の標準基盤だ。総スターが10万を超えた節目でTrending入りし（本日+290）、成熟した大型OSSとしては異例のペースで伸び続けている。

まとめ: 自分が試すなら文句なしにcodex-plugin-cc。普段のClaude Codeセッションに入れて、Codexの/reviewとClaude自身のレビューの指摘差分を1週間比べてみたい。
