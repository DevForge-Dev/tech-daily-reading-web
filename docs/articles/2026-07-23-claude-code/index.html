v2.1.217——同時サブエージェントに既定20の上限・ネスト生成を既定オフ、暴走とコスト膨張への安全弁

7/21の v2.1.217 は、同時に走るサブエージェントを既定20で頭打ちにし、サブエージェントが入れ子でさらにサブを生む挙動を既定オフにした。--max-budget-usd がバックグラウンドのサブを止めない不具合も修正。号令一下で並列に走らせる運用が普通になった今、"広げすぎ・課金膨張"を抑える守りの回だ。あわせて絵文字ショートコード補完やログイン失効警告の前倒し(5→3日)も入った。

📅 2026年7月23日 ⏱ 約2分 🔗 出典 3 件

この日の主役はv2.1.217（7/21リリース）。エージェントを号令一下で並列に走らせる使い方が当たり前になった結果、いちばん怖いのは「サブエージェントが際限なく増えて時間と課金が膨らむ」こと。今回はそこにまとめて安全弁を入れた回だ。

何が新しいか

核は3点。まず同時実行サブエージェントに既定20の上限を設けた。次に、サブエージェントが入れ子（ネスト）でさらにサブを生む挙動を既定オフに変更（ネスト自体は最大5階層まで技術的には可能だが、暴発を防ぐため既定は無効）。そして--max-budget-usd がバックグラウンドのサブエージェントを停止できない不具合を修正し、予算上限が本当に効くようにした。UI面では絵文字ショートコード補完（:heart: 等）、CLAUDE.md/SKILL.md のパス系frontmatter値の扱い、フッターPRバッジのリンク化を改善。ログイン失効の警告は5日前→3日前に前倒しされた。

背景——週次アップデートの潮流

直近の週次まとめでは、Bedrock/Vertex/Foundry で auto モードがオプトイン不要の既定になり、これらの環境で Opus 4.8 がデフォルトモデルに。Artifacts が Pro/Max へ解放され、公開したアーティファクトが MCP コネクタを呼んでライブデータを表示できるようになった。/review の刷新、/doctor による肥大化 CLAUDE.md の簡素化提案、--forward-subagent-text（stream-json にサブの思考/テキストを含める）も入っている。

実務での使いどころ・注意点

設計指針は「広い探索（走査・発見・要約）はfork、重い深掘りはinline」。副作業を隔離コンテキストのサブに逃がすとメイン会話のコンテキスト氾濫を防げる。hooks は100%確実に発火する強制レイヤーとして、lint 自動実行や rm -rf の事前ブロックに使う。落とし穴は、これまで既定20超・ネスト前提で重い調査を回していた場合に途中で頭打ちになること——必要なら明示的に上限を上げる。

ひとことまとめ

「並列は既定20まで・ネストは既定オフ・予算上限は本当に効く」。派手さはないが、毎日エージェントを回す人ほど効く安全弁の回だ。
