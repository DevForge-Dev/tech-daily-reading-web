Flutter 3.44 移行実務 — iOSはSPM既定・AndroidはAGP 9、Web は WASM 既定へ前進、Impeller は WebGPU 移植中

Google I/O 2026 で出た Flutter 3.44.0 / Dart 3.12 への移行が今日の焦点。iOS/macOS は Swift Package Manager（SPM）が新規プロジェクトの既定依存マネージャに昇格して CocoaPods を置換し、Apple Silicon で Rosetta 不要に。Android は Android Gradle Plugin 9（AGP 9）対応と Kotlin 設定の組込み化へ。Web は WebAssembly が既定コンパイルターゲットへ前進し、Impeller を WebGPU へ移植中。

なぜ今これが話題か

安定版は Flutter 3.44.0 / Dart 3.12（Google I/O 2026, 2026-05-18）。年4回（2月/5月/8月/11月）の安定リリースが続き、次は2026年8月予定。今日の主役は新機能そのものより3.44 への移行実務で、iOS と Android の足回りが同時に変わるため、既存プロジェクトはバージョン整合と deprecated API の解消が要る。

何が変わるか（プラットフォーム別）

iOS/macOS では SPM が新規プロジェクトの既定依存マネージャに昇格して CocoaPods を置換、Apple Silicon で Rosetta 不要になる。Android は AGP 9 をサポートし、Kotlin 設定の組込み化へシフト。既存プロジェクトは Gradle/AGP/Kotlin のバージョン整合と非推奨APIの解消が必要だ。SDK 側では Material/Cupertino がコアで凍結され、将来 material_ui / cupertino_ui として独立バージョニングへ（2026年内完了見込み）。Dart 3.12 は dot shorthands・private named parameters が正式、primary constructors は実験プレビュー。Web は WebAssembly を既定コンパイルターゲットへ前進し、Impeller を WebGPU へ移植中でブラウザでもネイティブ級グラフィックを狙う。

性能の物差し（Impeller）

Impeller の核心はシェーダーのAOT（ビルド時）コンパイル。Skia の実行時(JIT)コンパイルが招いた初回ジャンクを排除し、複雑シーンの平均ラスタライズ時間を約50%短縮、高リフレッシュ端末で120fps を一貫維持、アニメ負荷時のバッテリーは Skia比 約17%減。フレーム予算は下図が目安だ。

自分が試すなら

  • 検証ブランチで flutter create し直し、iOS は SPM 既定・Android は AGP 9 でビルドが通るか確認する。
  • 既存アプリは Gradle/AGP/Kotlin を上げ、非推奨APIの警告をつぶしてから 3.44 へ。
  • 視覚/性能の不具合切り分けには --no-enable-impeller で一時的に Skia へフォールバックして比較する。

注意点

Material/Cupertino の独立パッケージ化は移行作業が2026年内に続くため、依存の置き換えが将来発生する。Dart 3.12 の primary constructors は実験プレビューで本番不可。Web の WASM/WebGPU 経路は前進中だが対応ブラウザ・プラグイン互換を要確認。状態管理は新規=Riverpod 3.0、監査/大規模=Bloc 9.0 の基本線は変わらない。

ひとことまとめ

3.44 は「足回りの世代交代」。iOSのSPM既定・AndroidのAGP 9・WebのWASM前進が同時に来るので、まず検証ブランチで素のビルドを通し、整合性と非推奨APIを片付けてから本体を上げるのが安全だ。
