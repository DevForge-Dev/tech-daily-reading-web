AI-Native な開発ループ — Agentic Hot Reload と Genkit Dart

2026のFlutterは AI/エージェント連携を開発ループの中核へ。目玉の Agentic Hot Reload は Dart & Flutter MCP Server を使い、エージェントがUIを書き換えるたびに手動設定なしで自動ホットリロードする。あわせて Genkit Dart がOSSで登場し、Gemini/Anthropic/OpenAI を統一APIで扱えるフルスタックAIアプリを any platform で構築できる。安定運用は Flutter 3.44.0 / Dart 3.12.2、Impeller が既定レンダラ。

📅 2026年7月8日 ⏱ 約3分 🔗 出典 3 件

なぜ今これが話題か

2026のFlutterは、コーディングエージェントを「補助」ではなく開発ループの中核に据える方向へ舵を切った。エージェントがUIコードを書いても、反映のたびに人がホットリロードを押していてはflowが途切れる。今日の焦点 Agentic Hot Reload は、まさにこの摩擦を消すために設計されている。

何が新しいか

Agentic Hot Reload は Dart & Flutter MCP Server を使い、手動設定なしで自動ホットリロードする。エージェントがUIを書き換えるたびに人手を介さず反映され、開発者とエージェントの両方をflowに保つ。あわせて Genkit Dart がOSSで登場——フルスタック・AI駆動のエージェント型Dart/Flutterアプリを any platform で構築でき、モデル戦争に左右されず Gemini/Anthropic/OpenAI 系を統一APIで扱える。AIロジックは genkit_shelf でUI非依存のDartバックエンドへ退避でき、APIキーをクライアントから外せる。クライアント側の GenUI SDK は A2UI（Agent-to-UI）プロトコル上で、許可済みウィジェットカタログからランタイムに画面を組み立てる（experimental/alpha）。言語は Dart 3.12.2 で private named parameters と dot shorthands（.value 省略）、primary constructors を experimental サポート。

実務での使いどころ・自分が試すなら

まず Dart & Flutter MCP Server を入れ、コーディングエージェントとつないで Agentic Hot Reload の反映速度を体感する。AI機能を持つアプリなら、推論ロジックを最初から genkit_shelf のDartバックエンドに置き、APIキーをクライアントに埋めない設計にしておくと後の監査が楽。状態管理は従来どおり新規はRiverpod 3.x、大規模/監査はBloc 9.xが基本線で、AI連携でもこの選定は変わらない。

注意点・落とし穴

GenUI SDK と A2UI はexperimental/alphaで本番投入は時期尚早。primary constructors も experimental フラグ段階。Impeller は既定だが Platform View / Texture を多用する画面は実機計測必須で、Rive等で描画差が出たら --no-enable-impeller で Skia にフォールバックして切り分ける。フレーム予算は 60fps=16ms / 120fps=8.3ms、最大の敵は依然として不要な再ビルド。

ひとことまとめ

エージェントが書いて即反映（Agentic Hot Reload）、AIロジックはGenkit Dartでサーバへ、鍵はクライアントから外す。安定運用は 3.44.0 / Dart 3.12.2・Impeller既定・状態管理はRiverpod/Blocの二択、が7/8の地図。
