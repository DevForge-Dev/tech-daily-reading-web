height:auto へアニメーション — interpolate-size と calc-size() で純CSS開閉、Server Componentのまま

『開くと高さが auto に伸びるアコーディオン』は要求が多いのに CSS だけでは長年できなかった代表格。interpolate-size: allow-keywords と calc-size() が、auto などの内在サイズキーワードそのものを補間可能にする。JS計測もハックも不要で、Server Component のまま hydration ゼロで動く。プログレッシブエンハンスメントの好例だが Chromium 129+ 限定なので、特徴検出とフォールバックが要る。

なぜ今これが話題か

height: 0 → auto は補間対象にならない——この一点のために、開閉アニメは長らく回避策の巣窟だった。JS で scrollHeight を測って px を当てる、grid-template-rows: 0fr → 1fr、max-height に大きめの値を当てる（イージングが不自然・中身が超えると切れる）。どれも一長一短で、純CSSでの『素直な auto への補間』は不可能だった。interpolate-size: allow-keywords と calc-size()（Chrome/Edge/Opera 129+、2024-09〜）は、auto などの内在サイズキーワードそのものを補間可能にするネイティブ機能で、この積年の課題を正面から解く。

何が新しいか

interpolate-size: allow-keywords は、<length> と内在サイズキーワード（auto/min-content/max-content/fit-content 等）間の補間をオプトインで有効化する。既定は numeric-only（後方互換のため）。継承するので :root { interpolate-size: allow-keywords; } と一度書けば文書全体で効く。一方 calc-size() は calc() に似るが、ちょうど1つの内在サイズキーワードに対する演算を許す関数で、calc-size(auto, size) のように書く。size はそのキーワードの計算後サイズを表し、返り値は補間可能。calc-size() を値に使うと、その値だけに局所的にオプトインが効く。

仕組みのポイント

ネイティブ <details> なら開閉ロジックも JS 不要で、::details-content（中身を包む疑似要素）に高さ補間を当てる。display が切り替わる要素は、高さだけでなく transition-behavior: allow-discrete と @starting-style を併用して初めて開閉とも滑らかになる。純CSSなので 'use client' 不要、SSRで初期の開/閉を確定でき、ちらつかない。

/* app/globals.css */
:root { interpolate-size: allow-keywords; }
details::details-content {
  height: 0; overflow: hidden;
  transition: height 300ms ease, content-visibility 300ms allow-discrete;
}
details[open]::details-content {
  height: calc-size(auto, size); /* この値だけで補間オプトイン */
}

注意点

Chromium 129+ 限定で Safari・Firefox は未実装＝Baseline 未達。@supports not (interpolate-size: allow-keywords) で必ずフォールバックを用意する。親に overflow: hidden が無いと閉じる途中で中身がはみ出す。高さアニメはレイアウトを走らせる（transform/opacityより高コスト）ので、多数同時展開は要注意。@media (prefers-reduced-motion: reduce) で transition: none にしても開/閉の最終状態は残るので情報は失われない。

ひとことまとめ

『auto への補間』がついにネイティブになった。純CSSで Server Component のまま動く好例だが、当面は対応ブラウザ限定。特徴検出前提で、まずは <details> の開閉から差し込むのが安全な第一歩だ。
