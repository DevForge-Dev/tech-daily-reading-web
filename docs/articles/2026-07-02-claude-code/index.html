Claude Code

Claude Code の既定が Sonnet 5 へ — ネイティブ1Mコンテキストと『落ちない』自動化

7月2日はモデル世代交代の日。Claude Sonnet 5 が Claude Code の新しい既定モデルになり、ネイティブ1Mトークンのコンテキストを標準搭載した（v2.1.197〜）。同時にストリーム監視の既定オン化と、プロセスが停止・再起動しても生き残るバックグラウンド実行が入り、長時間の自動化が現実的に『落ちない』方向へ進んだ。導入価格と上限緩和も合わせて、大きなコードベースを一気に読ませる運用が組みやすくなった。

📅 2026年7月2日 ⏱ 約3分 🔗 出典 4 件

なぜ今これが話題か

この半年、Claude Code は『対話の相棒』から『長時間まわす自動化基盤』へ重心を移してきた。その土台に直接効くのがモデルの世代交代だ。7月2日、Anthropic は Sonnet 系で最も自律的(agentic)とうたう Claude Sonnet 5 を発表し、Claude Code の新しい既定モデルに据えた。推論・ツール利用・コーディング・自律タスク処理が強化され、なによりネイティブ1Mトークンのコンテキストを標準で持つ。大規模リポジトリを分割せず一度に読ませる、という運用のハードルが下がった。

何が新しいか

公式ドキュメント「What's new in Sonnet 5」によれば、1Mコンテキストは『既定かつ最大』で、より小さいコンテキスト版は存在しない。長文脈のためにベータヘッダーは不要、長文脈リクエストも標準価格で課金される。出力は最大128kトークン、適応的思考(adaptive thinking)に対応し、ツールやプラットフォーム機能は Sonnet 4.6 と同等。Claude Code のモデルピッカーで「Sonnet 5 (1M context)」を選ぶと sonnet[1m] にマップされ全幅を使える。導入価格は $2 / $10 per Mtok（8/31まで）、レート上限も引き上げられた（v2.1.197 以降で既定化）。

『落ちない』ための仕組み

7月の Claude Code 更新は、モデルだけでなく耐障害性にも踏み込んだ。ストリームのアイドル監視(watchdog)が全プロバイダで既定オンになり、5分間イベントが来ない応答は中断して自動リトライする。さらに長時間コマンドやワークフローは、セッションのプロセスが停止・再起動・更新されても生き残るようになり、Windows でもバックグラウンドシェルは kill されず引き継がれる。無人で回すスケジュールタスクや CI にとっては、これが体感差になる。

自分が試すなら

  • 大規模リポジトリの横断調査や移行を、分割せず sonnet[1m] の全幅で一気に読ませて挙動を比べる。
  • ただし1M全幅は課金が増える。普段は文脈を絞り、勝負どころだけ全幅、と使い分ける（8/31までの導入価格が効くうちに感触を掴む）。
  • 毎朝の自動処理は、watchdog 既定オン前提で『固まったら自動リトライ』を織り込み、長時間ジョブの生存を確認する。

注意点

1Mは『使えば速い』ではなく『使うほど高い』。標準価格で課金される以上、全幅を常用すると請求が伸びる。カテゴリ別 /usage で自動化枠の消費を棚卸しし、全幅が本当に要る場面に限定したい。

ひとことまとめ

Sonnet 5 の既定化は、Claude Code を『大きな文脈を一度に扱い、落ちずに回す』基盤へ一段進めた。全幅1Mは強力だが正直にコストへ跳ねるので、使い分けが主導権を握る。
