Impeller 移行の完遂——Android の legacy Skia backend を 10+ で削除

2026ロードマップ『Finishing the Job』の柱として Android の Impeller 一本化が完了へ。ラスタライズ約50%高速・120fps 安定・Skia 比 約17%低い電力ドレイン。カスタム fragment shader はビルド時 AOT コンパイルでシェーダージャンクが出ず好相性。安定版は Flutter 3.44.0 / Dart 3.12.2、次の機能リリースは 2026年8月予定。

📅 2026年7月20日 ⏱ 約3分 🔗 出典 3 件

この日のFlutterはImpeller移行の完遂が主役だ。2026ロードマップ『Finishing the Job』の柱としてAndroidのImpeller一本化が完了に向かい、legacy Skia backendをAndroid 10以上で削除、複数年の多段移行が終結する。新レンダラが「もう普通」になり、性能前提を一段引き上げる節目だ。

ロードマップの5本柱と数値

柱はAndroid Impeller完遂・WebのWebAssembly既定化・GenUI/エージェント型アプリ・Full-Stack Dart・AI前提の開発体験再設計で、年4回以上の安定リリース（2/5/8/11月）をコミット。Impellerはラスタライズ約50%高速・高リフレッシュ端末で120fps安定・アニメ多用時にSkia比 約17%低い電力ドレインを、実行時シェーダーコンパイル排除とCPU/GPU同期削減で実現する。安定版はFlutter 3.44.0／Dart 3.12.2、次の機能リリースは2026年8月予定。

カスタムシェーダーで120fps

Impellerは全シェーダーをビルド時（AOT）にコンパイルするため、実行時のシェーダーコンパイルジャンクが出ずカスタムfragment shaderと好相性。実装のコツは重い分岐（if/else）を避けmix/step/clampの数学演算へ置換し、シェーダー実行を4〜5ms以内に収めること（120fpsの予算は8.3ms）。アニメ区間はRepaintBoundaryで囲み、実機（特にミッドレンジAndroid）で検証する。

言語と計測

Dart 3.12はprivate named parameters（先頭_を自動省略）・primary constructors（experimental）・dot shorthandsを追加。計測面ではDevToolsがInspector 2.0へ刷新され、Impeller専用ビューでdraw callのバッチ化やテクスチャのメモリ過多を即可視化する。状態管理の2026ベンチはrebuildがBloc 2.3ms／Riverpod 2.0msとほぼ互角で、選定は性能でなく設計と運用で決まる。

注意点

計測は必ずprofileモード（debug VMは大幅に遅い）。カスタムシェーダーはGPU差（一部Adreno）で挙動が変わるため実機必須。Impeller描画の切り分けは--no-enable-impellerで行う。

ひとことまとめ

「Android legacy Skiaを畳み、Impeller前提の120fps時代へ」。カスタムシェーダーは4〜5ms、計測はprofileモードが鉄則。
