Flutter コアSDKのスリム化 — Material/Cupertino がパッケージ独立へ（material_ui / cupertino_ui）、Dart 3.12 の dot shorthands と private named parameters

安定版は Flutter 3.44.0 / Dart 3.12.2（Google I/O 2026, 2026-05-18, 次は8月）。今日の主役は新ウィジェットではなく、コアSDKそのもののスリム化だ。長年同梱されてきた Material / Cupertino がコード凍結され、独立した pub.dev パッケージ（将来 material_ui / cupertino_ui）へ移行を始めた。3ヶ月のSDKサイクルから切り離して独立更新を可能にし、アプリサイズを縮小する狙いがある。Dart 3.12 の新文法とあわせて整理する。

なぜ今これが話題か

安定版は Flutter 3.44.0 / Dart 3.12.2（Google I/O 2026, 2026-05-18, 次の安定版は8月予定）。今日の主役は新ウィジェットではなくコアSDKそのもののスリム化だ。長年 flutter/flutter に同梱されてきた Material / Cupertino がコード凍結され、リポジトリの外の独立した pub.dev パッケージ（将来 material_ui / cupertino_ui）へ移行を始めた。3ヶ月のSDKサイクルから切り離してUIを独立更新できるようにしつつ、アプリサイズを縮小する狙いがある。

何が変わるか

SDK本体からUIライブラリが分離されると、Material の更新（slider・theming・indicator の対応拡大など）がSDKリリースを待たずに出せるようになり、使わないUIを含めないことでバイナリも小さくなる。開発体験では flutter run が構成ファイルをサポートし、Web の host / port / certification / header に加え proxy 設定も書けるようになった。言語側の Dart 3.12 は dot shorthands（.value の省略記法）で enum/静的メンバ参照を簡潔化、private named parameters と実験的な primary constructors でボイラープレートを削る。さらに Agentic Hot Reload（Dart MCP Server 経由の自動ホットリロード）や Genkit 追加など、AIエージェント連携も前進している。

Dart 3.12 の書き味

dot shorthands は型が分かっている文脈で型名を省ける。enum や静的メンバの参照が短くなり、UIコードの定型句が減る。

// before
setMode(ThemeMode.dark);
final align = MainAxisAlignment.center;

// Dart 3.12: dot shorthands（型が推論できる文脈で型名を省略）
setMode(.dark);
final align = .center;

自分が試すなら

  • 検証ブランチで 3.44 にし、Material/Cupertino の import が将来 material_ui 等へ移る前提で依存を棚卸しする。
  • Dart 3.12 の dot shorthands を enum/定数の多い箇所に部分導入して可読性を測る。
  • flutter run の構成ファイルに proxy を書き、社内プロキシ越しのWeb実行を試す。

注意点

primary constructors は実験プレビューで本番不可。Material/Cupertino のパッケージ分離は移行作業が当面続くため、将来 import パスやバージョン指定の置き換えが発生しうる。状態管理の基本線は変わらず、新規/非同期取得＝Riverpod 3.0、監査・規制・複雑な並行処理＝Bloc 9.0。性能は profile モードで計測し、フレーム予算 60fps=16ms / 120fps=11ms（ラスタライズは各 4ms / 8.3ms 目安）を物差しにする。

ひとことまとめ

3.44 の本質は「コアの軽量化と分離」。UIライブラリをパッケージへ出して独立更新とサイズ削減を進め、Dart 3.12 の新文法で日常コードを整理する。まずは依存の棚卸しと dot shorthands の部分導入から始めるのが現実的だ。
