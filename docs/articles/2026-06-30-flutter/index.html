Flutter 3.44 / Dart 3.12 の主役は AI-Native — Genkit Dart と GenUI（A2UI v0.9）で『LLMが画面を組む』を安全に

Google I/O 2026 で示された Flutter の今年の軸はAIネイティブ。Genkit Dart(preview)はGoogle/Anthropic/OpenAIを統一type-safe APIで扱い、GenUI SDK は A2UI プロトコル上でLLMが生成したJSON UIレシピを『許可済みウィジェットカタログ』から組み立てる。存在しないウィジェットは発明できない安全設計が肝だ。

📅 2026年6月30日 ⏱ 約3分 🔗 出典 3 件

なぜ今これが話題か

安定版は Flutter 3.44 / Dart 3.12（Google I/O 2026 発表、次の安定版は2026年8月予定）。今年の主役は個別ウィジェットの新作ではなく、AIネイティブな開発スタックそのものだ。LLM にUIや処理を委ねる前提で、フレームワーク側が「安全に任せる」仕組みを用意してきた。中心が Genkit Dart と GenUI SDK の2つで、後者の SDK ダウンロードは年初比 +500%と伸びている。

何が新しいか

Genkit Dart（preview）は、モデル戦争に左右されず Google / Anthropic / OpenAI および互換モデルを統一された type-safe API で扱える AI アプリ構築フレームワーク。サーバ〜クライアントを横断するエージェント型 Dart/Flutter アプリを組める。GenUI SDK はオープンな A2UI（Agent-to-UI）プロトコル v0.9 上に構築され、LLM が JSONベースの宣言的UIレシピを生成し、アプリが許可済みの Flutter ウィジェットカタログからその場で画面を組み立てる。モデルが Dart コードを実行するのではなく、カタログに無いウィジェットは発明できないのが安全設計の核だ。あわせて Dart 3.12 は .value の dot shorthands を正式化、Web は CanvasKit/Skwasm の GC 共有や DevTools の WASM 既定化、Widget Previewer 改善でIDEメモリ最大50%削減が入った。採用例として実車販売中の 2026 Toyota RAV4 のマルチメディアUIが Flutter で動く。

仕組みのポイント

GenUI の安全モデルは「生成するのはコードではなくレシピ」という割り切りにある。LLM の出力は宣言的JSONに限定され、実行可能な Dart には触れない。描画できるのは開発者が事前に許可したウィジェットだけなので、未知の挙動や任意コード実行のリスクが構造的に閉じる。Genkit Dart 側は、モデルを差し替えても呼び出しコードが変わらない type-safe な統一APIを提供し、ベンダーロックインを避けつつエージェントの推論部分を組み替えられる。

自分が試すなら

  • 小さなフォームやカード程度の画面で GenUI の許可カタログを定義し、LLM生成レシピがカタログ外を要求したら落ちる挙動を確かめる。
  • Genkit Dart で同じエージェントを Anthropic↔OpenAI に差し替え、呼び出しコード無改修で動くかを検証する。

注意点

Genkit Dart は preview、A2UI も v0.9 と発展途上で、API は変わり得る。GenUIの安全性は「許可カタログの設計」に依存するので、何を許可するかの線引きが開発者の責務になる。描画経路では HCPP（Hybrid Composition++）が Android API34+ で opt-inフラグ段階な点も踏まえて採用する。

ひとことまとめ

Flutter 3.44/Dart 3.12 はAIネイティブが軸。Genkit Dart がモデルを統一APIで束ね、GenUI＋A2UIが「LLMはレシピだけ作り、描画は許可カタログ内」に閉じて安全に画面を生成する。preview段階だが方向性は明快だ。
