安定化に舵 — v2.1.199とSonnet 5既定化、Skills×SubAgentの住み分け

7/6は派手な新機能より“土台固め”。v2.1.199は背景/サブエージェントとネットワーク周りの安定化が中心。並行してSonnet 5がClaude Codeの既定になり100万トークン文脈＋8/31まで$2/$10の優待価格。fallbackModelで予備モデルを最大3つ指定でき、拡張はSkills=マクロ／SubAgent=別文脈ワーカーの住み分けが定説化した。

📅 2026年7月6日 ⏱ 約3分 🔗 出典 5 件

なぜ今これが話題か

ここ数日のClaude Codeは、新機能より「長時間・並列で安定して回す」ための地固めが続いている。7/6はv2.1.199の安定化リリースと、Sonnet 5の既定化という運用面の節目が重なった。

何が新しいか

DevelopersIOによるとv2.1.199（7/2公開）は不具合修正が主体で、バックグラウンドエージェントとサブエージェントの安定性、およびレート制限・ストリーミング・SSLといったネットワーク信頼性を底上げした。直前のv2.1.197〜198で入ったClaude in ChromeのGAや/datavizを、安定運用に載せる位置づけだ。並行してSonnet 5がClaude Codeの既定モデル（Pro/Free含む全プラン）になり、ネイティブ100万トークン文脈に対応、8/31まで入力$2/出力$10（per Mtok）の優待価格で使える。セッションは約967Kトークンで自動コンパクトされる。changelogでは主モデル過負荷時に順に試すfallbackModel（最大3つ）、組織既定モデルのコンソール設定、bashモードのライブパス補完も入った。

拡張の住み分け（Skills×SubAgent）

今日の技術記事群はSkillsとSub-agentsの使い分けで一致している。Skillsは主会話の文脈内でオンデマンドに読み込む「マクロ」、Sub-agentsは.claude/agents/のMarkdownで定義する別コンテキストの「ワーカー」で、広い探索に数万トークン使っても要約だけを主エージェントに返し文脈を汚さないのが最大の差だ。手順や規約の知識はSkillsへ、独立した並列作業や重い探索はSub-agentsへ寄せる。Uravationは並列委譲で3〜6倍速を狙うプロンプト10本を提示している。

自分が試すなら

まずSonnet 5の1M文脈で、これまで分割していた大きめリファクタを一括で投げてコンパクト頻度を見る。定型自動化にはfallbackModelを設定して過負荷での中断を減らし、肥大化したCLAUDE.mdの手順部分をSkillへ、重い横断調査をSubAgentへ切り出す。

注意点

$2/$10は8/31までの期限つき優待。1M文脈は便利だが約967Kで自動コンパクトが走るため、超長セッションは要約で情報が薄まる前提で区切りを設計したい。事業面ではAnthropicがシリーズGで300億ドル調達（ポストマネー3,800億ドル）、法人契約が年初から約4倍と、継続投資の後ろ盾は厚い。

ひとことまとめ

既定はSonnet 5＋1M文脈へ、土台はv2.1.199で安定化。拡張はSkills＝マクロ／SubAgent＝ワーカーの二層で組むのが定石になった。
