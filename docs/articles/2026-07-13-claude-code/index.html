Claude Code

v2.1.207と公式「Steering」ガイド — 決定論と確率論でClaude Codeを操縦する

7/11付v2.1.207がトランスクリプト保護・/doctor診断・約400MBのメモリ削減を実施。同時に公式ブログが CLAUDE.md/hooks（決定論）と skills/subagents（確率論）という制御レイヤーの設計軸を提示した。

📅 2026年7月13日 ⏱ 約3分 🔗 出典 4 件

Claude Codeの設定手段は CLAUDE.md・フック・スキル・サブエージェントと増え続け、「どれに何を書くべきか」が実務の悩みどころになっていた。そこへAnthropic公式ブログが「Steering Claude Code」という使い分けガイドを公開し、同じ週の7月11日にはv2.1.207がリリースされた。運用の指針と足回りの改善が同時に揃った週だ。

まずv2.1.207。トランスクリプト保護と、セットアップ全体を点検する/doctorチェックアップが追加された。PR操作のスマートリンク（既存PRへの編集・マージ・コメント・pushがclaude agentsに自動で紐付く）も入り、エージェントワークフローの追跡性が上がっている。地味に効くのが自動アップデートの改善で、バイナリのダウンロードがディスクへのストリーミング書き込みになり、ピークメモリ使用量が約400MB削減された。長いリストや表のストリーミング中に端末がフリーズする問題も修正済みだ。

Steeringガイドの核心は「決定論的な仕組み（CLAUDE.mdとフック）」と「確率論的な仕組み（スキルとサブエージェント）」の区別を設計判断の軸に置くこと。CLAUDE.mdは200行以内に保ち、超える分はimportで分割するのが公式推奨だ。SmartScopeの上級テクニック集も同じ結論を裏付けていて、「rm -rfを実行しない」といった禁止指示をCLAUDE.mdに書いた場合の遵守率は約70%だが、フックなら100%ブロックできるという。つまり「守られないと困るルール」をプロンプトで頼むのは設計ミスで、フックに移すべきということになる。

自分が試すなら、まず/doctorで現環境を点検し、CLAUDE.mdの中の「絶対に守ってほしい系」の記述をフックへ移植する棚卸しをやりたい。調査系タスクのサブエージェント委譲もコンテキスト節約に直結する。

注意点はふたつ。週次利用上限50%増のプロモーションが本日7月13日で終了し、明日から通常上限に戻る見込みなので、ヘビーユーザーはペース配分の見直しが必要。もうひとつ、Trusted Devices（リモートセッション前のデバイス検証）はTeam/EnterpriseのRemote Control管理者向け機能だ。

ひとことまとめ: 「確実に守らせたいならフック、柔軟に任せたいならスキル/サブエージェント」——公式が示したこの一本の軸で、設定ファイルの置き場所の迷いはほぼ解消する。
