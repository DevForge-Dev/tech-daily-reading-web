Claude Code

Claude Codeの「舵取り」を体系化する — skills / hooks / rules / subagents の役割分担と、入れ子サブエージェント・Artifacts という地ならし

Anthropicは自社の新規プロダクションコードの約8割をClaudeが書くと明かした。これだけ任せるなら、汎用アシスタントを「自分のコードベースの専門家」に寄せる手段が要る。公式ブログが整理した4つの制御手段——skills・hooks・rules・subagents——を、6月の新機能（入れ子サブエージェント、フォールバックモデルチェーン、Artifacts）と合わせて読み解く。

📅 2026年6月29日 ⏱ 約3分 🔗 出典 4 件

なぜ今これが話題か

VentureBeatは、Anthropicが新規プロダクションコードの約80%をClaudeが執筆していると報じた。人間は設計・レビュー・方向づけに比重を移しつつある。これだけエージェントに任せる前提になると、汎用アシスタントをそのまま使うのではなく、自分のコードベースやワークフローの専門家へ寄せる「舵取り（steering）」が効いてくる。Anthropic公式ブログがその手段を体系立てて整理し、SitePointは6月に入った新機能を10項目で俯瞰した。

何が新しい/重要か

公式が挙げる制御手段は4つ。skills は SKILL.md を核にしたフォルダ型の手順パックで、必要時のみ本体が読み込まれるためトークンコストをほぼ増やさない。hooks はイベント発火で処理を決定論的に強制する。rules は常時効くガードレール。subagents は独立コンテキストで副次作業を担い、メインの文脈を汚さない。6月はこれに、親が子を最大3階層まで生成できる入れ子サブエージェント、最大3モデルを順に試すフォールバックチェーン、セッション成果をライブHTMLで共有するArtifactsが加わった。

仕組みのポイント

勘所は「いつ読み込まれ、どれだけ強制力があるか」。skills は長大な参照資料を抱えても、呼ばれるまで本体を読まないのでコストがほぼ発生しない。同じ手順を何度もチャットに貼っているなら、それはSkill化の合図だ。一方 CLAUDE.md に常駐させた知識は毎ターン文脈に乗るため、巨大化するとコストと混乱を招く。hooks は確率的なお願いではなくコードによる強制なので、「必ず通したいゲート」に向く。

自分が試すなら

  • このタスクのような毎朝の自動処理の定型手順を SKILL.md に切り出し、CLAUDE.md から重い手順記述を追い出してトークンを節約する。
  • 調査フェーズを subagent に分離し、メイン文脈を汚さず結論だけ受け取る構成を試す。入れ子は3階層まで。
  • 成果物のダッシュボードを Artifacts でライブ共有し、URL固定・バージョン復元を体験する。

注意点

入れ子サブエージェントやフォールバックチェーンは便利だが、階層とリトライの分だけコストが膨らむ。6月の更新で入ったエージェント単位のコスト按分・スコープ付き権限で、どのエージェントがいくら使ったかを必ず監視する。Artifacts は外部リクエストを遮断する厳格CSPと16MiB上限、Team/Enterpriseの組織内認証メンバー限定という制約も押さえておく。

ひとことまとめ

skills=必要時に読む手順、hooks=必ず通すゲート、rules=常時の規範、subagents=文脈を分ける作業。この4軸で舵を取り、入れ子サブエージェントとArtifactsで作業と共有を底上げする。8割をAIが書く時代の主導権は、こうした制御設計に宿る。
