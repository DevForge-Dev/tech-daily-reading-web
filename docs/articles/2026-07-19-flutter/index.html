Pure Impeller VulkanがAndroid既定に（Flutter 3.44）——シェーダージャンク消滅とラスタライズ50%短縮

Flutter 3.44 でモダンAndroidの既定レンダラが Pure Impeller Vulkan に。全シェーダーをビルド時(AOT)にコンパイルするため初回描画のシェーダーコンパイルジャンクが消え、複雑シーンの平均ラスタライズ時間は約50%短縮。状態管理は Riverpod 3.0 が既定・Bloc 9.0 が監査向け、描画は Build/Layout/Paint/Composite の4フェーズで60fps=16msの予算管理が軸。

📅 2026年7月19日 ⏱ 約3分 🔗 出典 4 件

この日のFlutterは描画エンジンの世代交代が主役。Flutter 3.44 でモダンAndroidの既定レンダラが Pure Impeller Vulkan になり、長年の悩みだった初回描画のシェーダーコンパイルジャンクが消滅した。安定版は Flutter 3.44.0 / Dart 3.12.2、Impeller は Android 既定（10+必須）で、次の機能リリースは2026年8月予定。

何が新しい/重要か

Impeller は全シェーダーをビルド時（AOT）にコンパイルしてランタイムのシェーダー生成ジャンクを排除する。直近テストでは複雑シーンの平均フレームラスタライズ時間を約50%短縮、iOS端末の worst-case フレーム時間も大幅低下。Dart 3.12 では private named parameters（先頭 _ の自動省略）、null-aware elements（? で null要素を条件付き追加）、dot shorthands も入った。視覚/性能問題が出たら --no-enable-impeller でSkiaにフォールバックし切り分ける。

状態管理の三強を選ぶ

2026は Riverpod 3.0（compile-time safety・Mutations・自動リトライ）／Bloc 9.0（監査証跡・event tracing）／GetX（退潮）の三強。パッケージサイズも判断材料になる。

パフォーマンスは4フェーズと予算で

レンダリングは Build / Layout / Paint / Composite の4フェーズ。RepaintBoundary で部分再描画を限定、AutomaticKeepAliveClientMixin でリスト内stateを保持、ValueKey で diff を効率化する。フレーム予算は 60fps=16ms / 90fps=11ms / 120fps=8.3ms（ラスタライズ<4msで60fps）。計測は必ず profile モードで、DevTools の Performance View で UI/Raster スレッドを可視化する。

自分が試すなら／注意点

まず 3.44 に上げて Android 実機で起動→操作のジャンクが減るかを profile モードで確認。アニメは micro 50–200ms／component 200–400ms／Hero 300–600ms を目安に、実装は「まず implicit → 連鎖は flutter_animate → 精密は explicit（dispose必須）→重い演出は Rive/Lottie」。カスタムシェーダーは重い分岐（if/else）を避け mix/step/clamp へ。Impeller由来の描画差が出たら Skia フォールバックで切り分けるのを忘れずに。

ひとことまとめ

「シェーダージャンクが消え、ラスタライズ50%短縮」。土台が速くなった分、再ビルド削減とフレーム予算の基本がそのまま効く年。
