Tailwind CSS v4 のCSS-firstテーマ設計 — @theme でトークンを一本化し、App RouterでSSR安全なダークモードを作る

Tailwind v4 は設定の中心を tailwind.config.js（JS）から CSS内の @theme へ移した。@theme に書いたトークンはそのまま実CSS変数（--color-*）として出力され、ユーティリティ（bg-brand）と素のCSS（var(--color-brand)）が同じ値を参照できる。テーマ切替が変数差し替えだけで完結する一方、darkMode:'class' が効かない等の落とし穴があり、App Router（RSC）では next-themes＋suppressHydrationWarning でちらつき・hydration mismatch を抑える必要がある。

なぜ今これが話題か

v4 はデフォルトで全トークンをCSS変数化するため、ランタイムのテーマ切替にJSクラスの総入れ替えが不要になった。「Tailwindのトークン」と「デザインシステムのトークン」が一本化し、ダーク/ライトやブランド切替が変数の差し替えだけで済む。反面、dark: の挙動が v3 から変わるなど移行上の落とし穴があり、App Router では SSR時にユーザーのテーマが分からずFOUC（ちらつき）や hydration mismatchが起きやすい。

何が新しいか（と最大の落とし穴）

v4 は tailwind.config.js／postcss.config.js／autoprefixer 不要。CSS先頭で @import 'tailwindcss'; し、@theme { --color-brand: oklch(0.62 0.19 255); } のように書くと、トークンは :root のCSS変数として生成され、bg-brand 等のユーティリティも自動生成される。色は知覚的に均一な OKLCH 推奨。最大の罠はダークモードだ。既定の dark: は prefers-color-scheme メディア駆動（OS追従）で、darkMode:'class' は効かない。手動トグルにするには CSS で @custom-variant を明示宣言する必要がある。

/* data属性方式（classと衝突しにくく推奨） */
@custom-variant dark (&:where([data-theme='dark'], [data-theme='dark'] *));

App Router でSSR安全にする

Next.js は v4 を @tailwindcss/postcss 経由で通す。next-themes の <ThemeProvider> は 'use client' のクライアントコンポーネントとして layout.tsx の <body> 内に置き（RSCのままにはできない）、<html lang='ja' suppressHydrationWarning> を付ける。サーバはテーマを知らずに描画→クライアントが localStorage から復元して data-theme を書き換えるため html 要素に差分が出るが、これは想定内なので警告を抑止する（html要素のみ）。attribute='data-theme' は @custom-variant の宣言と一致させる。トグルのアイコン等 useTheme() 依存UIは mounted ガードでマウント後のみ描画して初回不一致を避ける。

自分が試すなら

  • パレットを :root／[data-theme='dark'] に生変数で定義し、@theme inline はそれを指すだけにする二段構えを最小構成で組む。
  • テーマ切替を startViewTransition で包み、対応ブラウザだけ円形リビールにする進行的強化を試す。
  • prefers-reduced-motion: reduce 時は即時切替＋色 transition を無効化し、意味は残して動きだけ消す。

注意点

darkMode:'class' を書いても効かない（CSSで @custom-variant 宣言する）。テーマ依存色を素の @theme に直書きすると属性切替で変わらないので @theme inline＋生変数に。ThemeProvider をServer Componentにする／suppressHydrationWarning 付け忘れは警告とちらつきの元。

ひとことまとめ

v4 はトークンをCSS変数として一本化し、テーマ切替を「変数の差し替え」に落とし込めるのが本質。App Router では client境界・suppressHydrationWarning・mountedガードの3点を押さえれば、SSRでもちらつかないダークモードが作れる。
