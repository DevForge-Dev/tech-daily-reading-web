📅 2026年7月17日 ⏱ 約3分 🔗 出典 4 件

今週のClaude Codeは新機能より足回りの安定化とコスト是正が主役だ。とくに v2.1.211 は課金・セキュリティに関わる回帰を続けて潰した重要リリースで、毎日エージェントを回すユーザーほど恩恵が大きい。

何が新しいか

最大の修正はプロンプトキャッシュの過大請求。Bedrock / Vertex / Mantle / Foundry で、末尾のシステムコンテキストブロックがリクエストごとに新規入力トークンとして課金されていた回帰を直した。あわせて、ユーザーが停止したバックグラウンドエージェントが勝手に再起動して古いプロンプトを再実行する不具合、承認プレビューが紛らわしい文字（confusable characters）で視覚的に偽装できる問題、AutoモードがPreToolUse hookのask判断を上書きする問題も解消している。

サブエージェントの中身が見える

もう一つの目玉が --forward-subagent-text フラグ（環境変数 CLAUDE_CODE_FORWARD_SUBAGENT_TEXT）。stream-json 出力にサブエージェントのテキストと思考（thinking）を含められるようになり、サブエージェントが内部でどうタスクを分解し推論しているかを可視化できる。デバッグや挙動把握に効く。

claude -p "..." --output-format stream-json --forward-subagent-text

実務での使いどころ

週次利用上限を+50%引き上げるキャンペーンが7/19（PT）まで延長された（Pro/Max/Team・レガシー席Enterprise対象、無料/従量課金席は対象外、Claude Code限定）。ヘビーに回すなら期限内に重い検証を寄せると得だ。運用設計では Totalum の指針——「CLAUDE.mdやskillの指示は“お願い”で保証ではない。必ず起こしたい処理はhookに書け」——が刺さる。領域外書込ブロックはPreToolUse、編集後のprettierはPostToolUse、停止時テストはStopへ。

注意点

--forward-subagent-text は出力が増えるぶんログが太る。CIで stream-json をパースしているなら、サブエージェントの thinking が混ざる前提でパーサを見直す。過大請求の修正は更新して初めて効くので、旧版のまま基盤API課金を眺めていた人はまず claude update を。

ひとことまとめ

「黙って更新して、hookで固めて、期限内に回す」週。v2.1.211 へ上げ、コスト観測と hook ガードを整えるのが吉。
