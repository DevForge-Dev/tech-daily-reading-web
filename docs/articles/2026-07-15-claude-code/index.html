Claude Code

Claude Code v2.1系 — スクリーンリーダー対応・企業ラッパー、管理コンソールに費用対効果の可視化

7/14リリースでscreen reader mode（平文レンダリング）・vim挿入モードリマップ・企業向けプロセスラッパーを追加。管理コンソールにはValue/Usageタブが付き、Claude Codeの利用と費用対効果を日次で可視化。週次上限50%増は7/19まで延長。

📅 2026年7月15日 ⏱ 約2分 🔗 出典 4 件

今週のClaude Codeは、単機能の追加というよりアクセシビリティ・企業運用・コスト可視化という運用面がまとめて底上げされた。派手さはないが、チームや企業でエージェント開発を回すうえで効く改善が並ぶ。

何が新しいか

7/14リリースで、スクリーンリーダー向けの平文レンダリング「screen reader mode」が追加された。claude --ax-screen-reader、環境変数CLAUDE_CODE_PROCESS_WRAPPER、またはaxScreenReader設定で有効化する。あわせてvimモードでjj→Escのような2キーの挿入モードリマップ（vimInsertModeRemaps）、全画面メニューのマウス操作、企業ランチャー経由で自己起動を包むプロセスラッパーも入った。fastモードがモデル復帰時に自動復元されない不具合や、バックグラウンドエージェントへの返信消失も修正されている。

管理側では、Claude Enterpriseの管理コンソールにValue（価値）・Usage（利用）の2タブが追加。Usageはアクティブ開発者数・セッション数・組織全体のトップコマンド・受理された行数を日次更新で表示し、Valueは利用とコストを突き合わせてシート当たり価値を推定する。算出式はタブ内に表示され、自社前提に合わせて編集できる。さらに週次利用上限50%増プロモーションが7/19まで延長（Pro/Max/Team/Enterprise対象）。

拡張3レイヤーの決定則

今週読んだ解説ガイドで刺さったのは、Hooks・Subagents・Skillsの使い分けを一言で表す決定則だ。「Skillが手順を教え、Hookがルールを強制し、Subagentが作業を隔離する」。Hooksは25のライフサイクルポイントで発火し、PreToolUse（実行前のセキュリティ関門）やPermissionRequest（自動承認/拒否）が要点。サブエージェントは独自コンテキスト・権限・モデルを持つ隔離インスタンスで、並列委譲すると作業速度が最大3倍とされる。

使いどころと注意点

自分が試すなら、まず管理コンソールのUsageタブで自分の定期タスク群の消費を把握し、上限50%増の期間中に重いリサーチを寄せる。注意点は、Value/Usageの推定式はあくまで概算で、前提を自社に合わせないと過大／過小評価になりうること。プロモーション終了後（7/19以降）は通常上限に戻るので、恒常運用の設計は通常枠を基準にすべきだ。

ひとことまとめ: 機能追加より運用の底上げ。アクセシビリティと企業ラッパー、そして「何にいくら使っているか」が見えるようになった週。
